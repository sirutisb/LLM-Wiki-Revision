Two Types of Uncertainty
Aleatoric Uncertainty (Irreducible)
• Arises from inherent randomness 
in the data-generating process
• Example (coin flip):
• Probability of heads or tails is 0.5
• Flipping the coin more times does not 
reduce this uncertainty
• Cannot be eliminated, only 
modelled probabilistically
19
Epistemic Uncertainty (Reducible)
• Arises from lack of knowledge 
about the system
• Example (coin already flipped):
• The outcome is fixed
• You are uncertain only because you 
do not know the result
• Can be reduced with more 
information or data

Bayesian Inference
• Likelihood: 𝑝(𝒚|𝜃): the probability (or probability density) of the 
observed data 𝒚 =
𝑦1, . . . , 𝑦𝑛, given model parameters 𝜃
• Prior: 𝑝(𝜃): represents uncertainty or beliefs about the parameter θ 
before observing any data.
• Posterior: 𝑝(𝜃|𝒚): updated probability distribution over parameters 
after observing the data which can quantify the uncertainty in the 
parameters 𝜃
• Predictive distribution:  For a new observation 𝑦′, predictions 𝑝(𝑦′|𝒚) 
are obtained by averaging over parameter uncertainty and 
therefore provides uncertainty-aware predictions.

Advantages of posterior distribution:
• Uncertainty-aware prediction (posterior predictive): We can predict a 
new observation y′ via complete posterior predictive distribution i.e. 
𝑝(𝑦′|𝒚), which accounts for both data uncertainty (aleatoric), and 
parameter uncertainty (epistemic).
• Credible intervals for parameters: We can compute the probability that a 
parameter lies in a range.
• Model checking (posterior predictive checks): We can simulate new 
datasets from the fitted model and compare them to the real data.
