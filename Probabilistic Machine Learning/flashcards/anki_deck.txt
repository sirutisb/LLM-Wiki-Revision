What is Bayes' rule?	p(theta | D) = p(D | theta) * p(theta) / p(D). Posterior is proportional to likelihood times prior. The denominator p(D) = integral of p(D|theta)*p(theta) d-theta is the marginal likelihood (evidence).
Name the four components of Bayesian inference and their roles.	(1) Likelihood p(D|theta): how well parameters explain the data. (2) Prior p(theta): beliefs before seeing data. (3) Posterior p(theta|D): updated beliefs after data. (4) Marginal likelihood p(D): normalising constant, usually intractable.
What is the posterior predictive distribution and why is it useful?	p(y'|D) = integral p(y'|theta) * p(theta|D) d-theta. It integrates over all parameter uncertainty, producing calibrated, uncertainty-aware predictions rather than a point estimate.
Why is Bayesian inference preferred over MLE/MAP for small datasets?	MLE/MAP give only point estimates — no uncertainty quantification, no calibrated predictions, vulnerable to overfitting. Bayesian inference maintains a full distribution over parameters, providing credible intervals and principled uncertainty.
What happens to the posterior as the number of observations n → infinity?	The posterior concentrates on the MLE estimate; the prior is washed out. As n → 0, the posterior approaches the prior.
What is the key identity for the unnormalised log-posterior?	log p(theta|D) = log p(D|theta) + log p(theta) + const. The posterior is the product of likelihood and prior, up to normalisation.
Why is the unnormalised posterior p(D|theta)*p(theta) NOT a probability distribution?	It is not normalised — it does not integrate to 1. It cannot be used directly for probabilistic statements. Only the full normalised posterior p(theta|D) is a valid distribution.
What is Maximum Likelihood Estimation (MLE)?	MLE finds the parameter values that maximise the probability (likelihood) of the observed data: theta_hat = argmax_theta sum_i log p(y_i | theta). In practice, maximise the log-likelihood to avoid numerical underflow.
Give the MLE for a Gaussian distribution (both parameters).	mu_hat = (1/N) * sum y_i  (sample mean). sigma^2_hat = (1/N) * sum (y_i - mu_hat)^2  (biased sample variance, divides by N not N-1).
Give the MLE for a Binomial model (y heads in n flips).	theta_hat_MLE = y / n.
Give the MLE for simple linear regression (no intercept).	w_hat_MLE = (sum x_i * y_i) / (sum x_i^2).
Give the Normal Equations for multiple linear regression.	w_hat_MLE = (X^T X)^{-1} X^T y. This is the closed-form solution; derivation via matrix calculus is NOT examinable but the result must be known.
Why does MLE for linear regression equal OLS (ordinary least squares)?	The log-likelihood under a Gaussian noise model is -1/(2*sigma^2) * sum (y_i - w^T x_i)^2. Maximising this is equivalent to minimising the sum of squared errors.
What is a key limitation of MLE?	MLE gives a single point estimate with no uncertainty quantification. It can overfit with small datasets. It is a special case of MAP with a flat (uniform) prior.
Is the Gaussian MLE variance biased or unbiased?	Biased — it divides by N, not N-1. The unbiased estimator divides by N-1 (Bessel's correction), but MLE naturally gives the biased version.
What is MAP estimation?	MAP finds the mode of the posterior: theta_hat_MAP = argmax_theta [log p(D|theta) + log p(theta)]. It is the posterior mode — a point estimate incorporating prior knowledge.
Write the MAP formula for a Gaussian likelihood with Gaussian prior.	mu_hat_MAP = [n*sigma_0^2 / (n*sigma_0^2 + sigma^2)] * y_bar + [sigma^2 / (n*sigma_0^2 + sigma^2)] * mu_0. A weighted average of sample mean y_bar and prior mean mu_0.
How does MAP with a Gaussian prior relate to ridge regression?	With prior w ~ N(0, (1/lambda)*I), MAP = argmin [sum(y_i - w^T x_i)^2 + lambda * ||w||^2]. This is L2-regularised (ridge) regression. The prior variance controls regularisation strength.
What happens to MAP when (a) n=0, (b) n→infinity, (c) strong prior?	(a) n=0: MAP = prior mode. (b) n→infinity: MAP → MLE (prior negligible). (c) Strong prior (small sigma_0^2): MAP strongly pulled toward prior mean mu_0.
What is the key difference between MAP and full Bayesian inference?	MAP is a single point estimate (the posterior mode). Full Bayesian inference uses the entire posterior distribution. MAP cannot provide credible intervals or posterior predictive uncertainty.
Why does dropping p(D) not affect MAP?	p(D) is a constant with respect to theta, so argmax_theta [p(D|theta)*p(theta) / p(D)] = argmax_theta [p(D|theta)*p(theta)]. The normalising constant cancels.
What is a conjugate prior?	A prior p(theta) is conjugate to a likelihood p(D|theta) if the resulting posterior p(theta|D) belongs to the same parametric family as the prior. This gives a closed-form posterior.
State the Beta-Binomial conjugate update.	Prior: theta ~ Beta(alpha, beta). Likelihood: y ~ Binomial(n, theta). Posterior: theta|y ~ Beta(alpha + y, beta + n - y). Add successes to alpha, failures to beta.
What is the posterior mean for the Beta-Binomial model?	E[theta|y] = (alpha + y) / (alpha + beta + n). As n → infinity, this tends to y/n (the MLE). The prior provides pseudo-counts: alpha = prior successes, beta = prior failures.
State the Gamma-Poisson conjugate update.	Prior: lambda ~ Gamma(alpha, beta). Likelihood: y_i ~ Poisson(lambda). Posterior: lambda|y ~ Gamma(alpha + sum y_i, beta + n).
State the Gaussian-Gaussian conjugate update (known variance).	Prior: mu ~ N(mu_0, sigma_0^2). Likelihood: y_i ~ N(mu, sigma^2) known sigma^2. Posterior: mu|y ~ N(mu_hat_MAP, Sigma_n) — also Gaussian.
How do you prove conjugacy for Beta-Binomial?	Write posterior proportional to likelihood * prior: theta^y * (1-theta)^(n-y) * theta^(alpha-1) * (1-theta)^(beta-1) = theta^(alpha+y-1) * (1-theta)^(beta+n-y-1). This is Beta(alpha+y, beta+n-y) — same family as prior.
What is the key advantage of conjugate priors?	Closed-form posterior — avoids numerical integration. Updates are simple hyperparameter updates. No approximation needed. The posterior is in the same family with updated parameters.
State the probabilistic model for simple linear regression.	y_i = w*x_i + epsilon_i, where epsilon_i ~ N(0, sigma^2). Equivalently: y_i | x_i, w ~ N(w*x_i, sigma^2). Assumes Gaussian noise with constant variance (homoscedastic).
Why does MLE for linear regression equal minimising sum of squared errors?	The log-likelihood is -1/(2*sigma^2) * sum (y_i - w*x_i)^2. Maximising this is equivalent to minimising the sum of squared residuals — hence MLE = OLS.
When should you use logistic regression instead of linear regression?	When outputs are binary (not continuous). Linear regression can predict probabilities outside [0,1]. Logistic regression uses the Bernoulli distribution and logit link, ensuring valid probability outputs.
What are the Normal Equations for multiple linear regression?	w_hat_MLE = (X^T X)^{-1} X^T y. This is the closed-form solution obtained by setting the gradient of the log-likelihood to zero.
What is the model for logistic regression?	P(y_i = 1 | x_i) = sigma(w^T x_i) = 1 / (1 + exp(-w^T x_i)). The sigmoid function maps the linear predictor from (-inf, +inf) to (0, 1).
What is the link function for logistic regression and why is it used?	The logit link: log(theta/(1-theta)) = w^T x. It maps probabilities in (0,1) to the real line. The log link is unsuitable because it maps to (0, +inf), not [0,1].
Does logistic regression have a closed-form MLE?	No. The sigmoid is non-linear, so there is no closed-form solution. Requires iterative methods: gradient ascent, Newton-Raphson, or L-BFGS.
What is the log-likelihood for logistic regression?	l(w) = sum_i [y_i * log(theta_i) + (1 - y_i) * log(1 - theta_i)]. This is the binary cross-entropy. Maximising log-likelihood = minimising cross-entropy loss.
Why is the log function not suitable as a link function for logistic regression?	The log link maps the linear predictor to (0, +infinity), not [0, 1]. Probabilities must lie in [0, 1], so we need the logit link which maps the full real line to (0, 1) via the sigmoid.
What is the model for Bayesian linear regression?	Prior: w ~ N(w_0, S_0). Likelihood: y_i | x_i, w ~ N(w^T x_i, sigma^2). Because both are Gaussian (conjugate), the posterior is also Gaussian: w|X,y ~ N(w_N, S_N).
Write the posterior update equations for Bayesian linear regression.	S_N^{-1} = S_0^{-1} + (1/sigma^2) * X^T X. w_N = S_N * (S_0^{-1} * w_0 + (1/sigma^2) * X^T y). The precision matrices (inverses of covariance) add.
What is the predictive distribution for Bayesian linear regression?	p(y* | x*, X, y) = N(w_N^T x*, sigma^2 + x*^T S_N x*). Predictive variance = observation noise sigma^2 (aleatoric) + epistemic uncertainty x*^T S_N x* from the posterior.
How does MAP with Gaussian prior relate to ridge regression in Bayesian LR?	With isotropic prior w ~ N(0, tau^2 * I), the MAP estimate is (X^T X + (sigma^2/tau^2)*I)^{-1} X^T y — ridge regression with lambda = sigma^2/tau^2. The prior variance controls regularisation.
What are the three components of a GLM?	(1) Linear predictor: eta_i = w^T x_i. (2) Link function g: g(mu_i) = eta_i, relates mean to linear predictor. (3) Response distribution: y_i ~ exponential family with mean mu_i.
Give the distribution, link, and inverse link for (a) linear, (b) logistic, (c) Poisson regression.	(a) Gaussian, identity link eta=mu, inverse mu=eta. (b) Bernoulli, logit link eta=log(mu/(1-mu)), inverse mu=sigma(eta). (c) Poisson, log link eta=log(mu), inverse mu=exp(eta).
Why must a link function map (-inf, +inf) to the valid range of the mean?	The linear predictor eta can take any real value. The link must constrain mu to the valid range: [0,1] for probabilities (logit), (0, +inf) for counts/rates (log), (-inf, +inf) for continuous (identity).
How are GLM parameters estimated?	By MLE. Only Gaussian linear regression has a closed-form MLE (Normal Equations). All other GLMs (logistic, Poisson) require iterative gradient-based optimisation.
What is the Naive Bayes assumption?	All features are conditionally independent given the class label: p(x|y) = product_j p(x_j|y). This "naive" assumption rarely holds exactly but often works well in practice.
Write the Naive Bayes classification rule.	y_hat = argmax_y [log p(y) + sum_j log p(x_j|y)]. Equivalently: argmax_y p(y) * product_j p(x_j|y). The denominator p(x) is constant across classes and is dropped.
How are Naive Bayes parameters estimated?	Class prior p(y=c) = n_c / n (fraction of training examples in class c). For discrete features: p(x_j=v|y=c) = count of class-c examples with x_j=v, divided by n_c.
Is Naive Bayes generative or discriminative? What does it model?	Generative. It models the joint distribution p(x, y) = p(y) * product_j p(x_j|y), then uses Bayes' rule to classify. It can generate new data by sampling y then x|y.
What is the difference between a generative and a discriminative model?	Generative: models joint p(x, y) = p(x|y)*p(y); uses Bayes' rule to classify; can generate data. Discriminative: models conditional p(y|x) directly; optimises the decision boundary; cannot generate data.
Give one example each of a generative and discriminative classifier.	Generative: Naive Bayes (models p(y) and p(x|y)). Discriminative: Logistic Regression (models p(y|x) directly).
When does a generative model outperform a discriminative model?	Small training sets (generative models incorporate prior structure), missing data (can marginalise), when data generation is needed. Discriminative models win with large datasets (fewer assumptions → lower asymptotic error).
What does a discriminative model learn vs a generative model?	Discriminative learns p(y|x) — only the decision boundary. Generative learns p(x|y) and p(y) — the full data generating process. Generative models have more parameters but understand the data distribution.
What is the Laplace approximation?	Approximate an intractable posterior with a Gaussian centred at the MAP estimate, using the local curvature of the log-posterior to determine the variance: q(theta) = N(theta_hat_MAP, A^{-1}), where A = -g''(theta_hat).
State the four steps of the 1D Laplace approximation procedure.	(1) Compute g(theta) = log p(D|theta) + log p(theta). (2) Find MAP: theta_hat = argmax g(theta). (3) Compute curvature A = -g''(theta_hat). (4) Approximate posterior: q(theta) = N(theta_hat, A^{-1}).
In the Laplace approximation, why does the first-order Taylor term vanish?	Because theta_hat is the mode of g(theta), so g'(theta_hat) = 0. The second-order Taylor expansion reduces to g(theta) ≈ g(theta_hat) - (A/2)*(theta - theta_hat)^2.
What is the variance of the Laplace approximation?	Variance = A^{-1} = 1/(-g''(theta_hat)) = inverse curvature of the log-posterior at the MAP. Sharper peak (larger A) → smaller variance (more concentrated approximation).
State one use and one limitation of the Laplace approximation.	Use: approximates intractable posteriors cheaply with a Gaussian (just one MAP optimisation + one Hessian). Limitation: only valid near the mode; fails for multimodal, skewed, or heavy-tailed posteriors.
For p(theta|y) proportional to theta^y * (1-theta)^(n-y), find the Laplace approximation.	MAP: theta_hat = y/n. Curvature A = -g''(y/n) = n^3 / (y*(n-y)). Variance = y*(n-y)/n^3. Laplace: q(theta) = N(y/n, y*(n-y)/n^3).
How does Laplace differ from variational inference?	Laplace is local — Gaussian fit only near the MAP, cheap. VI is global — optimises over an entire family of distributions to find the best approximation. VI can handle non-Gaussian posteriors better.
State the BIC formula.	BIC(M) = log p(D | theta_hat, M) - (k/2) * log n, where theta_hat is the MLE, k is the number of parameters, n is the number of data points. Higher BIC = better model.
How is BIC derived from the Laplace approximation?	Apply Laplace to the log marginal likelihood integral. The Hessian term is approximated as (k/2)*log(n), dropping lower-order terms. BIC ≈ log p(D|theta_hat) - (k/2)*log(n).
What does each term in BIC represent?	Data fit term log p(D|theta_hat): favours complex models. Penalty term -(k/2)*log(n): penalises models with many parameters (k). BIC balances fit and complexity — automatic Occam's razor.
Two models: A has k=2, log-likelihood=-10; B has k=5, log-likelihood=-8. n=100. Which does BIC prefer?	BIC_A = -10 - (2/2)*log(100) = -10 - 4.6 = -14.6. BIC_B = -8 - (5/2)*log(100) = -8 - 11.5 = -19.5. Model A preferred (higher BIC).
What is variational inference?	VI approximates an intractable posterior p(theta|D) by solving an optimisation problem: find q*(theta) in a tractable family Q that minimises KL(q || p(theta|D)). Equivalent to maximising the ELBO.
Why is reverse KL (used in VI) called mode-seeking?	Reverse KL = KL(q||p). If q(theta) > 0 but p(theta) ≈ 0, the log ratio blows up — q is heavily penalised for placing mass where p is near zero. So q "seeks" one mode of p, potentially underestimating spread.
Why is forward KL mass-covering?	Forward KL = KL(p||q). Expectation taken under p — q is penalised if q ≈ 0 where p > 0. q must cover all mass of p. Result: q overestimates spread. Also intractable (requires p, which is unknown).
Write the key relationship between log p(D), ELBO, and KL divergence.	log p(D) = ELBO(q) + KL(q || p(theta|D)). Since KL ≥ 0 and log p(D) is constant: maximising ELBO ≡ minimising KL(q || p(theta|D)).
How does VI compare to MCMC and Laplace approximation?	Laplace: local Gaussian near MAP, cheap, fails for multimodal posteriors. VI: global optimisation over a family, deterministic, scalable, but biased (approximating family may not contain true posterior). MCMC: asymptotically exact but slow, produces correlated samples.
What is the ELBO (Evidence Lower Bound)?	ELBO = E_q[log p(D, theta)] - E_q[log q(theta)] = E_q[log p(D|theta)] - KL(q(theta) || p(theta)). It is a lower bound on log p(D), used as the VI objective.
Derive the ELBO from the KL divergence definition.	KL(q || p(theta|D)) = E_q[log q(theta)] - E_q[log p(D,theta)] + log p(D). Rearranging: log p(D) = ELBO + KL, where ELBO = E_q[log p(D,theta)] - E_q[log q(theta)]. Since KL ≥ 0, ELBO ≤ log p(D).
Interpret the two terms of the ELBO: E_q[log p(D|theta)] - KL(q || p(theta)).	Term 1 (expected log-likelihood): measures data fit under q — higher is better. Term 2 (KL to prior): measures how far q deviates from the prior — penalises complexity. Maximising ELBO balances fit and regularisation.
When does the ELBO equal log p(D)?	When q(theta) = p(theta|D) — i.e., when the approximation perfectly matches the true posterior. The KL gap is zero. In practice this rarely happens since the true posterior is outside the variational family.
State the two equivalent forms of the ELBO that must be known.	Form 1: ELBO = E_q[log p(D,theta)] - E_q[log q(theta)]. Form 2: ELBO = E_q[log p(D|theta)] - KL(q(theta) || p(theta)). Both are equivalent and examinable.
Define the KL divergence KL(q||p).	KL(q||p) = integral q(x) * log(q(x)/p(x)) dx = E_q[log q(x) - log p(x)]. Measures the extra information needed to encode samples from p using a code optimised for q.
State two key properties of KL divergence.	(1) Non-negative: KL(q||p) ≥ 0, with equality iff q = p. (2) Asymmetric: KL(q||p) ≠ KL(p||q). It is not a true distance metric (violates symmetry and triangle inequality).
Prove that KL divergence is non-negative using Jensen's inequality.	KL(q||p) = E_q[-log(p(x)/q(x))] ≥ -log(E_q[p(x)/q(x)]) = -log(1) = 0. The inequality follows because -log is convex and Jensen gives f(E[X]) ≤ E[f(X)] for convex f.
How are KL divergence, entropy, and cross-entropy related?	KL(p||q) = H(p,q) - H(p), where H(p,q) = -sum p(x)*log q(x) is cross-entropy and H(p) = -sum p(x)*log p(x) is entropy. Cross-entropy = entropy + KL.
What does it mean for KL divergence to be "mode-seeking" vs "mass-covering"?	Reverse KL(q||p): q avoids regions where p ≈ 0 → mode-seeking, underestimates uncertainty. Forward KL(p||q): q must cover all where p > 0 → mass-covering, overestimates uncertainty. VI uses reverse KL.
What is the mean-field assumption in variational inference?	q(theta) = product_j q_j(theta_j). The approximating distribution fully factorises — all parameters are assumed independent under q. This ignores posterior correlations.
State the CAVI (Coordinate Ascent VI) update rule for factor q_j.	log q_j*(theta_j) = E_{q_{-j}}[log p(D, theta)] + const, where the expectation is over all factors except j. Normalise to obtain q_j*. Iterate over all j until ELBO converges.
What is the key limitation of mean-field VI?	It ignores posterior correlations — each factor is updated independently. This leads to underestimation of posterior variance (due to mode-seeking reverse KL). The factorised family may not contain the true posterior.
How does CAVI relate to Gibbs sampling?	Both iterate over conditional updates one variable at a time. Gibbs sampling: draws samples from exact conditionals. CAVI: optimises each factor q_j to maximise ELBO. CAVI is deterministic; Gibbs is stochastic.
What is MCMC and what problem does it solve?	MCMC (Markov Chain Monte Carlo) constructs a Markov chain whose stationary distribution equals the target posterior, allowing posterior expectations to be estimated via sample averages. Solves intractable posterior inference without needing the normalisation constant.
What is the detailed balance condition and why does it matter?	p*(theta) * T(theta'|theta) = p*(theta') * T(theta|theta'). If satisfied, p* is the stationary distribution of the Markov chain. MH is designed to satisfy this condition.
What is burn-in in MCMC?	The initial samples in a Markov chain depend on the starting point theta^(0) before the chain reaches stationarity. Burn-in = the number of initial samples to discard. After burn-in, samples approximate the target distribution.
Compare MCMC with variational inference.	MCMC: asymptotically exact (converges to true posterior), samples are correlated, slow, no distributional assumptions. VI: fast, deterministic, differentiable, but approximate (biased by choice of variational family).
In MCMC, why does the normalisation constant Z cancel?	Only the ratio p*(theta')/p*(theta) is needed for acceptance decisions. Z = integral of p*(theta) d-theta appears in both numerator and denominator and cancels: [p~(theta')/Z] / [p~(theta)/Z] = p~(theta')/p~(theta).
State the Monte Carlo integration estimator.	I_hat = (1/S) * sum_{s=1}^S f(theta_s), where theta_1,...,theta_S ~ p(theta). Approximates E_p[f(theta)] = integral f(theta)*p(theta) d-theta.
What is the convergence rate of Monte Carlo integration?	O(1/sqrt(S)) — standard error = std(f)/sqrt(S). This rate is independent of the dimension of the problem, making MC the method of choice for high-dimensional integrals.
Why does MC integration beat numerical quadrature in high dimensions?	Quadrature requires exponentially many grid points as dimension increases. Monte Carlo's O(1/sqrt(S)) convergence rate does not depend on dimension — the same number of samples works regardless of how many dimensions there are.
Describe the rejection sampling algorithm.	(1) Find constant M such that p~(theta) ≤ M*q(theta) for all theta. (2) Sample theta' ~ q(theta). (3) Sample u ~ Uniform(0,1). (4) Accept theta' if u ≤ p~(theta')/(M*q(theta')); otherwise reject and repeat.
Why does rejection sampling fail in high dimensions?	The bounding constant M must cover the entire support of p~. In high dimensions, the proposal q has most of its mass in regions where p* has little mass. Acceptance rate drops exponentially with dimension.
What is the acceptance rate for rejection sampling?	Acceptance rate = Z/M, where Z = integral p~(theta) d-theta. A tighter bound (M close to Z) gives higher acceptance rate. Larger M → more proposals rejected → wasteful.
What is importance sampling?	Estimate E_{p*}[f(theta)] by sampling from a proposal q(theta) and reweighting: mu_hat = (1/S) * sum f(theta_s) * w(theta_s), where w(theta_s) = p*(theta_s)/q(theta_s) is the importance weight.
What is the self-normalised importance sampling estimator?	mu_hat = [sum_s f(theta_s) * w~(theta_s)] / [sum_s w~(theta_s)], where w~(theta_s) = p~(theta_s)/q(theta_s). Does not require knowing the normalisation constant Z.
Why does importance sampling fail in high dimensions?	The proposal q covers regions where p* has almost no mass — a few samples get very large weights, the rest get near-zero weights. Effective sample size (ESS) collapses exponentially. Weight degeneracy makes estimates unreliable.
How does importance sampling differ from rejection sampling?	Rejection sampling: accepts or rejects samples; accepted samples are independent and unweighted. Importance sampling: keeps all samples but reweights them; no samples discarded but weights can degenerate. Both fail in high dimensions.
State the Metropolis-Hastings acceptance ratio.	A = min(1, [p~(theta') * q(theta^{t-1} | theta')] / [p~(theta^{t-1}) * q(theta' | theta^{t-1})]). Accept theta' with probability A; else keep theta^{t-1}.
What is the difference between MH and the Metropolis algorithm?	Metropolis is a special case of MH where the proposal is symmetric: q(theta'|theta) = q(theta|theta'). The correction term q(theta|theta')/q(theta'|theta) = 1, so A = min(1, p~(theta')/p~(theta^{t-1})).
Why does MH work even when we don't know the normalisation constant?	The acceptance ratio only requires p~(theta') / p~(theta^{t-1}) — the normalisation constant Z cancels in the ratio. We only need the unnormalised posterior p~(theta) = p(D|theta)*p(theta).
What happens if the proposal step size is too small vs too large in MH?	Too small: high acceptance rate but slow exploration — chain gets stuck locally. Too large: proposals jump far but are often rejected (low acceptance rate) — chain barely moves. Optimal acceptance rate ≈ 23% in high dimensions, ≈ 44% in 1D.
What is Gibbs sampling?	MCMC algorithm that samples from the joint p(theta_1,...,theta_d) by iteratively sampling each variable from its full conditional p(theta_j | theta_{-j}, D), keeping all others fixed. Always accepts (acceptance rate = 1).
Why does Gibbs sampling always accept (A=1)?	It samples from the exact full conditional distribution p(theta_j | theta_{-j}, D). This is a special case of MH where the proposal is the exact conditional — the acceptance ratio evaluates to exactly 1.
What is required for Gibbs sampling to work?	Tractable full conditionals p(theta_j | theta_{-j}, D). In conjugate models, these often simplify to known distributions (Gaussian, Beta, Dirichlet). If conditionals are intractable, Gibbs cannot be used.
Compare Gibbs sampling to Metropolis-Hastings.	Gibbs: samples exact conditionals, acceptance = 1 always, no proposal to tune, requires tractable conditionals. MH: general proposal distribution, accept/reject step, acceptance < 1, works even without tractable conditionals.
Define discrete entropy.	H(X) = -sum_x p(x) * log p(x) = E_p[-log p(X)]. Units: bits (log base 2) or nats (natural log). Measures average surprise/uncertainty of the distribution.
What distribution maximises entropy for a discrete variable with K outcomes?	The uniform distribution, with H_max = log K. Minimum entropy is H=0 for a deterministic variable (one outcome with probability 1).
Define differential entropy for a continuous variable.	h(X) = -integral p(x) * log p(x) dx. Unlike discrete entropy, differential entropy can be negative. It depends on scale and is not invariant to reparameterisation.
What is the differential entropy of a Gaussian N(mu, sigma^2)?	h(X) = (1/2) * log(2*pi*e*sigma^2). Depends only on sigma^2, not mu. Larger variance → higher entropy → more uncertainty. Two Gaussians with the same variance have equal entropy regardless of their means.
Which Gaussian has larger entropy: N(0,1) or N(0, 6.25)?	N(0, 6.25) — larger variance sigma^2 = 6.25 > 1. Entropy of Gaussian depends only on sigma^2. Mean is irrelevant to entropy.
What is the information-theoretic interpretation of entropy?	Shannon's Noiseless Coding Theorem: entropy = minimum average code length. Optimal code length for symbol x is -log_2 p(x) bits. Average code length = E[-log p(X)] = H(X).
Define cross-entropy H(p, q).	H(p, q) = -sum_x p(x) * log q(x) = E_p[-log q(x)]. The expected number of bits needed to encode data from p using a code optimised for q.
State the decomposition of cross-entropy into entropy and KL divergence.	H(p, q) = H(p) + KL(p || q). Cross-entropy = entropy of p + extra bits wasted by using q instead of the optimal code. Since KL ≥ 0: H(p,q) ≥ H(p), with equality iff p = q.
What is the connection between minimising cross-entropy and MLE?	Minimising cross-entropy H(p_data, q) over model q is equivalent to minimising KL(p_data || q), which is equivalent to maximising sum_i log q(x_i) — the log-likelihood. MLE = minimum cross-entropy.
Write the binary cross-entropy loss for classification.	L = -(1/n) * sum_i [y_i * log(p_hat_i) + (1-y_i) * log(1-p_hat_i)]. This is the negative log-likelihood of the Bernoulli model. Minimising it = maximising the Bernoulli log-likelihood.
Define mutual information I(X;Y).	I(X;Y) = sum_{x,y} p(x,y) * log(p(x,y) / (p(x)*p(y))) = KL(p(x,y) || p(x)p(y)). Measures reduction in uncertainty about X given knowledge of Y.
Express mutual information in terms of entropies.	I(X;Y) = H(X) - H(X|Y) = H(Y) - H(Y|X) = H(X) + H(Y) - H(X,Y). The reduction in uncertainty about X when Y is observed.
State three properties of mutual information.	(1) Non-negative: I(X;Y) ≥ 0. (2) Zero iff independent: I(X;Y) = 0 iff X ⊥ Y. (3) Symmetric: I(X;Y) = I(Y;X). Upper bound: I(X;Y) ≤ min(H(X), H(Y)).
What is the conditional entropy H(X|Y)?	H(X|Y) = -sum_{x,y} p(x,y) * log p(x|y) = sum_y p(y) * H(X|Y=y). Average uncertainty about X when Y is known.
State the maximum entropy principle.	Given known constraints on a distribution (e.g. mean, variance), choose the distribution with the highest entropy — making the fewest additional assumptions beyond the stated constraints.
What distribution has maximum entropy given fixed mean and variance?	The Gaussian N(mu, sigma^2) has maximum entropy among all continuous distributions with fixed mean mu and variance sigma^2. This justifies using Gaussians as default distributions.
Complete the table: MaxEnt distribution given (a) support [0,1]; (b) mean mu, support [0,inf); (c) mean mu and variance sigma^2.	(a) Uniform(0,1). (b) Exponential(1/mu). (c) Gaussian N(mu, sigma^2).
Why does the maximum entropy principle produce the "least informative" distribution?	Adding extra structure beyond known constraints implies knowledge we don't have — it reduces entropy. Maximum entropy makes no assumptions beyond the given constraints, representing maximum ignorance consistent with the known facts.
Define an HMM and its three components.	HMM lambda = (A, B, pi): A = transition matrix (a_ij = P(s_t=j | s_{t-1}=i), rows sum to 1). B = emission matrix (b_j(o) = P(o_t=o | s_t=j), rows sum to 1). pi = initial state distribution (pi_i = P(s_1=i), sums to 1).
State the two key assumptions of an HMM.	(1) Markov property: P(s_t | s_1,...,s_{t-1}) = P(s_t | s_{t-1}) — state depends only on previous state. (2) Observation independence: P(o_t | s_1,...,s_T, o_1,...) = P(o_t | s_t) — observation depends only on current state.
What are the three fundamental problems of HMMs?	(1) Evaluation/Likelihood: compute P(O|lambda) → Forward algorithm. (2) Decoding: find argmax_S P(S|O,lambda) → Viterbi algorithm. (3) Learning: estimate lambda from O → Baum-Welch (NOT examinable).
Write the joint distribution of an HMM.	p(s_{1:T}, o_{1:T}) = pi_{s_1} * product_{t=2}^T a_{s_{t-1},s_t} * product_{t=1}^T b_{s_t}(o_t). Initial state probability * all transition probabilities * all emission probabilities.
Why is dynamic programming needed for HMM inference instead of direct summation?	Naive summation over all N^T state sequences is exponential in T. Dynamic programming stores intermediate probabilities alpha_t(i) at each time step and reuses them, reducing complexity from O(N^T) to O(N^2 * T).
Define the forward variable alpha_t(i).	alpha_t(i) = P(o_1, o_2,..., o_t, s_t = i | lambda). Probability of observing the first t outputs AND being in state i at time t.
State all three steps of the forward algorithm.	Initialisation: alpha_1(j) = pi_j * b_j(o_1). Recursion: alpha_t(j) = [sum_i alpha_{t-1}(i) * a_ij] * b_j(o_t). Termination: P(O|lambda) = sum_i alpha_T(i).
What is the time complexity of the forward algorithm?	O(N^2 * T) — N states, T time steps. Compare to naive O(N^T). For N=5, T=100: forward uses ~2500 operations vs 10^70.
In the Weather HMM (N=2, T=3): alpha_1(R)=0.06, alpha_1(S)=0.24, alpha_2(R)=0.0552, alpha_2(S)=0.0486, alpha_3(R)=0.02904, alpha_3(S)=0.004572. What is P(O|lambda)?	P(O|lambda) = alpha_3(R) + alpha_3(S) = 0.02904 + 0.004572 = 0.0336.
What is the key difference between the forward algorithm and the Viterbi algorithm?	Forward uses sum (marginalises over all paths): alpha_t(j) = [sum_i alpha_{t-1}(i)*a_ij] * b_j(o_t) → total likelihood P(O|lambda). Viterbi uses max (finds best path): v_t(j) = [max_i v_{t-1}(i)*a_ij] * b_j(o_t) → most likely state sequence.
Define the Viterbi variable v_t(j).	v_t(j) = max_{s_1,...,s_{t-1}} P(s_1,...,s_{t-1}, s_t=j, o_1,...,o_t | lambda). Probability of the best (most likely) partial path ending in state j at time t.
State all steps of the Viterbi algorithm including backtracking.	Init: v_1(j) = pi_j * b_j(o_1). Recursion: v_t(j) = [max_i v_{t-1}(i)*a_ij] * b_j(o_t); backpointer psi_t(j) = argmax_i v_{t-1}(i)*a_ij. Termination: s_T* = argmax_j v_T(j). Backtrack: s_{t-1}* = psi_t(s_t*) for t=T,...,2.
What are backpointers in the Viterbi algorithm?	psi_t(j) = argmax_i v_{t-1}(i) * a_ij. Stores which previous state i gave the maximum score when transitioning to state j at time t. Used in backtracking to recover the full optimal state sequence.
In the Weather HMM: v_3(R)=0.01344, v_3(S)=0.002592, psi_3(R)=R, psi_2(R)=S. What is the most likely sequence?	s_3* = argmax{v_3} = R. Backtrack: s_2* = psi_3(R) = R. s_1* = psi_2(R) = S. Most likely sequence: Sunny → Rainy → Rainy.
What is the most common pitfall when confusing Forward and Viterbi?	Forward uses SUM (to compute total likelihood by marginalising). Viterbi uses MAX (to find the single best path). They have identical initialisation but different recursion operators.
What is an autoencoder?	A neural network trained to reconstruct its input through a low-dimensional bottleneck. Architecture: x → encoder f_theta → z → decoder g_phi → x_hat. Trained to minimise ||x - g_phi(f_theta(x))||^2.
What is the latent space like in a standard autoencoder?	Deterministic: z = f_theta(x) is a fixed point in latent space for each x. No probabilistic structure — no prior on z. Points between encoded examples may decode to nonsense. Cannot generate new data.
Why can't a standard autoencoder generate new data?	There is no structured prior on the latent space. You cannot sample a meaningful z without first encoding a real x. The latent space may be irregular and discontinuous — random z values decode to nonsense.
Compare AE vs VAE across: latent z, objective, generation ability, latent structure.	AE: z deterministic (f_theta(x)); reconstruction loss only; cannot generate; irregular latent space. VAE: z probabilistic (z ~ q_phi(z|x)); ELBO = reconstruction - KL; can generate by sampling z ~ N(0,I); smooth structured latent space.
Describe the generative model of a VAE.	z ~ p(z) = N(0, I). x ~ p_theta(x|z) — decoder neural network. Marginal likelihood p_theta(x) = integral p_theta(x|z)*p(z) dz is intractable. The encoder q_phi(z|x) approximates the true posterior p_theta(z|x).
Write the VAE ELBO (training objective).	ELBO = E_{q_phi(z|x)}[log p_theta(x|z)] - KL(q_phi(z|x) || p(z)). Term 1 (reconstruction): log-likelihood of data given latent code. Term 2 (regulariser): forces latent distribution toward prior N(0,I).
What is the VAE encoder?	q_phi(z|x) = N(mu_phi(x), sigma^2_phi(x) * I). A neural network that outputs mean mu_phi(x) and variance sigma^2_phi(x) of the approximate posterior over z given x.
What does the KL term in the VAE ELBO prevent?	It forces q_phi(z|x) to stay close to the prior N(0,I). Without it, the encoder could map each x to a point mass (z deterministic), making the decoder overfit and the latent space irregular. KL regularisation ensures a smooth, structured latent space.
How do you generate new samples from a trained VAE?	Sample z ~ N(0, I) (the prior). Pass z through the decoder p_theta(x|z). The structured latent space ensures these random z values produce plausible samples.
Why is the reparameterization trick needed in VAE training?	Training requires gradients of the ELBO with respect to phi (encoder parameters). But z ~ q_phi(z|x) is a sampling operation — non-differentiable. Standard backpropagation cannot flow through a stochastic node.
State the reparameterization trick for a Gaussian encoder.	Instead of z ~ N(mu_phi(x), sigma^2_phi(x)*I), write z = mu_phi(x) + sigma_phi(x) ⊙ epsilon, where epsilon ~ N(0, I). Now z is a deterministic function of phi and independent noise epsilon — gradients can flow through mu_phi and sigma_phi.
What requirement must a distribution satisfy to use the reparameterization trick?	It must be reparameterisable: expressible as z = g(phi, epsilon) for a deterministic function g and noise epsilon with a fixed (phi-independent) distribution. Works for Gaussian, Uniform, and other location-scale families.
Name the six core components of a reinforcement learning problem.	Agent (learner), Environment (everything external), State s (current situation), Action a (agent's choice), Reward r (scalar feedback), Policy pi(a|s) (agent's decision rule — maps states to actions).
What is the RL objective?	Maximise expected cumulative discounted reward: G_t = sum_{k=0}^inf gamma^k * r_{t+k}, where gamma in [0,1) is the discount factor. gamma=0 is myopic (only immediate reward), gamma≈1 is long-sighted.
What is the exploration vs exploitation trade-off?	Exploitation: choose the best known action to maximise reward now. Exploration: try unknown actions to learn if they might be better. Too much exploitation: miss better options. Too much exploration: waste time on known-bad options.
What is the difference between model-free and model-based RL?	Model-based: learns transition model P(s'|s,a) and reward R(s,a) explicitly, then plans. Model-free: learns Q-values directly from experience without learning a model (e.g. Q-learning).
What is the multi-armed bandit problem?	Stateless RL: K arms (actions), each yielding rewards from an unknown distribution. Agent repeatedly chooses an arm to maximise total reward. No state transitions — purely the exploration/exploitation trade-off.
Define regret in the bandit setting.	Instantaneous regret: l_t = V* - Q(a_t), where V* = Q(a*) is the optimal arm value. Total regret: L_T = sum_{t=1}^T (V* - Q(a_t)). Goal: minimise total regret = maximise cumulative reward.
Describe the epsilon-greedy algorithm for bandits.	With probability (1-epsilon): choose argmax_a Q_hat_t(a) (exploit best estimate). With probability epsilon: choose a random action (explore). Epsilon typically small (e.g. 0.1). All arms eventually explored.
How is the sample-average estimate Q_hat_t(a) computed?	Q_hat_t(a) = (total reward received from arm a so far) / (number of times arm a has been chosen). As t → infinity with fixed epsilon, Q_hat_t(a) → Q(a) for all a (law of large numbers).
At a given time step, how can you tell if epsilon-greedy definitely explored (not exploited)?	If the selected action is NOT the greedy action (not argmax_a Q_hat_t(a)) at that time step, then epsilon-greedy must have explored. If the selected action equals the greedy action, it could be either.
Define an MDP as a tuple and explain each component.	MDP = (S, A, P, R, gamma). S: state space. A: action space. P(s'|s,a): transition probability. R(s,a): reward function. gamma in [0,1): discount factor. The agent interacts with this environment to maximise discounted return.
State the Markov property for MDPs.	P(s_{t+1} | s_t, a_t, s_{t-1}, a_{t-1},...) = P(s_{t+1} | s_t, a_t). The future depends only on the current state and action, not the history. This memoryless property makes MDPs tractable.
Define the state-value function V^pi(s) and action-value function Q^pi(s,a).	V^pi(s) = E_pi[sum_{k=0}^inf gamma^k * r_{t+k} | s_t=s] — expected return from state s under policy pi. Q^pi(s,a) = E_pi[sum_{k=0}^inf gamma^k * r_{t+k} | s_t=s, a_t=a] — expected return from taking action a in state s then following pi.
What is the optimal policy in an MDP?	pi*(s) = argmax_a Q*(s,a). The greedy policy with respect to the optimal Q-function. At each state, take the action with the highest Q-value.
State the Bellman optimality equation for Q*(s,a).	Q*(s,a) = R(s,a) + gamma * sum_{s'} P(s'|s,a) * max_{a'} Q*(s',a'). For deterministic transitions: Q*(s,a) = R(s,a) + gamma * max_{a'} Q*(s',a').
State the Bellman optimality equation for V*(s).	V*(s) = max_a [R(s,a) + gamma * sum_{s'} P(s'|s,a) * V*(s')]. The optimal value of a state = best immediate reward + discounted optimal value of next state.
What does the Bellman equation recursively say about Q*(s,a)?	The optimal Q-value of (s,a) = immediate reward r(s,a) + discounted value of the best possible next action. It is self-consistent: knowing Q* everywhere lets you compute Q* at each (s,a).
What is the temporal difference (TD) error?	TD(s,a) = [R(s,a) + gamma * max_{a'} Q(s',a')] - Q(s,a). Measures how far Q(s,a) is from satisfying the Bellman equation. Q-learning drives TD error toward zero. TD > 0: action better than expected; TD < 0: worse.
State the Q-learning update rule.	Q_t(s,a) = Q_{t-1}(s,a) + alpha * [r(s,a) + gamma * max_{a'} Q_{t-1}(s',a') - Q_{t-1}(s,a)]. Where alpha is the learning rate, gamma is the discount factor, r is the reward, s' is the next state.
What does it mean for Q-learning to be model-free and off-policy?	Model-free: does not learn P(s'|s,a) or R(s,a) explicitly — updates Q-values directly from observed (s,a,r,s') transitions. Off-policy: the update uses max_{a'} Q(s',a') — the optimal action — regardless of which action was actually taken next.
State the Q-learning algorithm steps.	(1) Initialise Q(s,a)=0 for all s,a. (2) Observe state s. (3) Choose action a (e.g. epsilon-greedy). (4) Execute a, observe r and s'. (5) Update: Q(s,a) ← Q(s,a) + alpha*[r + gamma*max_{a'}Q(s',a') - Q(s,a)]. (6) Repeat.
Example: alpha=0.6, gamma=0.4. Transition: s3→a1, r=5. What is Q(s3,a1) after one update from Q=0?	Q(s3,a1) = 0 + 0.6 * [5 + 0.4 * max_{a'} Q(s',a') - 0] = 0.6 * [5 + 0] = 3.0.
What does Q-learning converge to, given sufficient exploration?	Q-learning converges to Q*(s,a) — the optimal action-value function satisfying the Bellman optimality equation. The optimal policy is then pi*(s) = argmax_a Q*(s,a).
Compare MLE, MAP, and Bayesian inference on these dimensions: uncertainty, overfitting, computation.	MLE: point estimate, no uncertainty, prone to overfitting (small data), cheap. MAP: point estimate with prior, no uncertainty quantification, less overfitting (prior regularises), cheap. Bayesian: full posterior, uncertainty quantified, most principled, expensive (integration).
Compare Laplace approximation, Variational Inference, and MCMC.	Laplace: local Gaussian near MAP, cheap (1 MAP + Hessian), fails for multimodal/skewed. VI: global deterministic optimisation, scalable, fast, biased (variational family). MCMC: asymptotically exact, samples posterior faithfully, slow, correlated samples.
Compare rejection sampling and importance sampling.	Rejection: accepts/rejects proposals, accepted samples are independent, requires bound M, discards samples (wasteful). Importance: reweights all samples by w=p*/q, no samples discarded, but weight degeneracy in high dimensions. Both fail in high dimensions.
Compare Naive Bayes and Logistic Regression.	Naive Bayes: generative, models p(y)*prod p(x_j|y), conditional independence assumption, fewer data needed, more parameters. Logistic Regression: discriminative, models p(y|x) directly, no independence assumption, needs more data, lower asymptotic error.
What are the three approximation methods for intractable posteriors and when would you use each?	Laplace: cheap, good for roughly Gaussian/unimodal posteriors, use when MAP is available. VI: when scalability matters, large datasets, need deterministic approximation. MCMC: when accuracy is critical, model is complex, asymptotic correctness needed.
Compare the Forward Algorithm and the Viterbi Algorithm side by side.	Forward: goal=P(O|lambda), operator=sum, no backpointers, init=pi_j*b_j(o_1), recursion=sum_i(alpha_{t-1}(i)*a_ij)*b_j(o_t). Viterbi: goal=best state sequence, operator=max, stores backpointers psi, same init, recursion=max_i(v_{t-1}(i)*a_ij)*b_j(o_t).
Bandit vs MDP — what is the key structural difference?	Bandit: stateless (one state), no transitions, purely exploration/exploitation. MDP: multiple states, transition function P(s'|s,a), sequential decisions with state dependence. Q-learning extends bandit Q-value updates to the full MDP setting.
What connects MLE training of logistic regression to minimising cross-entropy?	The log-likelihood for Bernoulli data is sum[y_i*log(p_i) + (1-y_i)*log(1-p_i)]. Maximising this = minimising negative log-likelihood = minimising binary cross-entropy. Cross-entropy loss IS the negative log-likelihood.
How does the ELBO appear in both variational inference and the VAE?	Both use the same ELBO structure: log p(D) ≥ E_q[log p(D|theta)] - KL(q||prior). In VI: theta are model parameters, q is the approximate posterior. In VAE: theta=decoder parameters, q_phi(z|x) is the encoder (approximate posterior over latent z).

State Bayes' rule from memory (no formula sheet).	p(theta|D) = p(D|theta) * p(theta) / p(D). Equivalently: p(theta|D) ∝ p(D|theta) * p(theta). Must know this — it is not provided.
State the MLE for Gaussian (mu and sigma^2) from memory.	mu_hat = (1/N)*sum y_i. sigma^2_hat = (1/N)*sum (y_i - mu_hat)^2. Divides by N (biased), not N-1.
State the MLE for Binomial from memory.	theta_hat = y/n (number of successes divided by number of trials).
State the simple linear regression MLE from memory.	w_hat = (sum x_i * y_i) / (sum x_i^2). Derived by differentiating the log-likelihood and setting to zero.
State the Bellman equation for Q* from memory.	Q*(s,a) = R(s,a) + gamma * max_{a'} Q*(s', a'). For stochastic transitions: Q*(s,a) = R(s,a) + gamma * sum_{s'} P(s'|s,a) * max_{a'} Q*(s',a').
State the Q-learning update from memory.	Q(s,a) ← Q(s,a) + alpha * [r + gamma * max_{a'} Q(s',a') - Q(s,a)]. Learning rate alpha ∈ (0,1], discount factor gamma ∈ [0,1).
