Self-Organising Maps
Aim:
Learn to map points from a high-dimensional space to a
low-dimensional (discrete) that preserves topological properties
Uses:
Visualisation and regularity discovery in data
Self-organised:
Map emerging from local interactions
(competition and co-operation) between data points
Neural network?
There are aspects of brain organisation and learning that are based on
self-organisation (e.g. the brain is naturally organised into functional
areas for dealing with language, vision etc)
Very different from MLPs – no activation functions, no linear
combination of inputs, no backpropagation

Working assumptions
SOM as unsupervised learning must learn the underlying structure in the
data with no desired target signal
It assumes that
Input data that belong to the same class or cluster share common
features
SOM will be able to identify these key features across a number of
data points
SOM will be able to meaningfully organise/order the input data
according to a given low-dimensional structure

Learning algorithm
1 Initialise network
Set weights to small random values
Define (large) neighbourhood size
2 Sample random input (xt) from the data and input to the network
3 Compute the Euclidean distance of each output node j from the
current input using
δ(xt, wj) =
sX
i
(xi −wij)2
4 Identify the node closest to the input (the BMU)
5 Adjust the weights of the BMU and its neighbours with
wij(t + 1) = wij(t) + η(t)(xij(t) −wij(t))
6 Repeat from step (2) until the map converges

Initialisation problems
Unfavourable initialisation
The network can unfold badly if badly
initialised
Learning rate or neighbourhood radius
too slow or too quickly collapsing
exacerbates
Map exhibits a knot – difficult to untwist
Better approaches to initialisation
Random initialisation with data range awareness
Principal component initialisation
Sample-based initialisation
Pre-training (e.g. k-means) to provide initial weight estimates
