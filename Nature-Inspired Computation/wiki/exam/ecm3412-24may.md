### Answer

#### Part (a) [5 marks]

Nature-inspired techniques offer the following advantages:

1. **No gradient required** — they work on any fitness landscape, including non-differentiable, discontinuous, or noisy objective functions where classical optimisation (gradient descent, branch-and-bound) fails.
2. **Escape local optima** — population-based approaches explore multiple regions simultaneously; stochastic variation prevents permanent trapping in local optima.
3. **Flexible and general** — they can be applied to any problem expressible as a fitness evaluation; the same algorithmic framework adapts to radically different problem domains with minimal modification.
4. **Handle complexity** — effective on NP-hard combinatorial problems where exact methods are computationally infeasible.
5. **Emergent solutions** — can discover solutions that human designers would not think of (e.g. the NASA-evolved antenna geometry).

**Example:** [[genetic-algorithms]] applied to the Travelling Salesman Problem (TSP) — a permutation-encoded chromosome represents a tour; crossover and mutation operators generate new tours; the algorithm finds near-optimal tours on instances that are computationally intractable for exact methods.

Another example: [[ant-colony-optimization]] applied to job scheduling — ants construct feasible schedules guided by pheromone reinforcement; no explicit mathematical formulation of the optimum is needed.

#### Part (b) [5 marks]

**Fitness Proportionate (Roulette Wheel) Selection — mechanism:**

Each individual $i$ in a population of size $P$ is assigned a selection probability proportional to its fitness:

$$P(i) = \frac{f_i}{\sum_{k=1}^{P} f_k}$$

Conceptually, imagine a roulette wheel where each individual occupies a sector whose arc-length is proportional to its fitness. Spinning the wheel selects an individual: fitter individuals have larger sectors and are more likely to be selected. The process is repeated until enough parents have been chosen.

See [[selection]] for the full comparison of selection methods.

**Disadvantages:**

1. **Superfit dominance:** if one individual has a very high fitness (e.g. $f = 100$) while all others have $f \approx 0.1$–$0.5$, that individual dominates almost all selections. The population converges prematurely to one solution with no diversity.
2. **Requires positive fitness values:** does not work directly for minimisation problems (low fitness = large probability, which is the wrong direction) or problems where fitness can be negative. Requires transformation (e.g. $f_{\max} - f_i$) which introduces additional design choices.
3. **Scaling sensitivity:** adding a constant to all fitness values changes relative selection probabilities. For example, if all fitnesses increase from the range $[1,2]$ to $[101,102]$, selection becomes almost uniform — the algorithm loses selection pressure.
4. **Not commonly used today** — tournament selection and rank-based selection have superseded roulette wheel because they avoid all three problems above.

#### Part (c) [6 marks]

**Application 1: Hyperparameter optimisation in machine learning**

Optimisation algorithms are used to search for the best hyperparameters (learning rate, number of layers, regularisation coefficient, etc.) for a machine learning model — values that are not learned by gradient descent but must be set before training.

- **Example 1:** [[evolutionary-algorithms]] / [[genetic-algorithms]] to search the hyperparameter space of a neural network — each chromosome encodes a set of hyperparameters; fitness = validation accuracy after training with those settings.
- **Example 2:** [[pso]] applied to SVM hyperparameter tuning — particles move through the (C, γ) parameter space; gbest converges to the combination that minimises cross-validation error.

**Application 2: Neural Architecture Search (NAS) / feature selection**

Optimisation algorithms are used to select which features (input variables) to include in a model, or to find the topology of the neural network itself — decisions that cannot be made by gradient descent alone.

- **Example 1:** [[genetic-algorithms]] for feature selection — binary chromosome where each bit indicates whether a feature is included; fitness = model accuracy on validation set with that subset of features.
- **Example 2:** [[evolutionary-algorithms]] / [[spiking-neural-networks]] — bi-level EA optimisation for neural architecture search (NAS), simultaneously evolving network structure (upper level) and weights (lower level), used particularly for SNNs where backpropagation is not straightforward.

#### Part (d) [4 marks]

| | Genetic Algorithm (GA) | Genetic Programming (GP) |
|---|---|---|
| **What the chromosome represents** | A vector of parameters (binary, integer, or real) — a fixed-length solution | A tree-structured program or function |
| **What is being optimised** | The parameters of a solution to a specific problem | The algorithm or function that solves the problem |
| **Fitness evaluation** | Direct computation: decode chromosome and evaluate $f(\mathbf{x})$ numerically | **Execute the program** on a set of test cases; compare output to expected output; aggregate errors |
| **Fitness cost** | Cheap (one calculation) | Expensive (run the program for every training case) |

In a GA, the fitness function directly computes a score from the decoded chromosome values (e.g. route length for TSP, network error for ANN weight optimisation).

In GP, the "chromosome" is an executable program. To evaluate fitness you must run the evolved program on all training examples and measure how well its outputs match the targets (e.g. sum of absolute errors across all training data points for symbolic regression). See [[genetic-programming]] and [[genetic-algorithms]].

#### Part (e) [4 marks]

In [[pso]], every particle maintains two memory values:

**Personal best ($p_{ij}$, "pbest"):** the best position (highest fitness) that particle $i$ has visited in its own history. It encodes the particle's individual experience — where the particle itself found the best solution so far.

**Global best ($g_j$, "gbest"):** the best position visited by *any* particle in the entire swarm across all time. It encodes collective social knowledge — the best solution the group has collectively discovered.

**Effect of using both:**
- Using only pbest would make particles individually hill-climb around their personal history — the algorithm would converge slowly and might miss the globally best region.
- Using only gbest would pull all particles toward one location immediately — the swarm would converge too quickly (loss of diversity, premature convergence).
- Combining both balances **exploration** (personal memory keeps diversity) and **exploitation** (social memory drives convergence to the best-known region).

**How information is combined — the velocity update:**

$$v_{ij}(t+1) = v_{ij}(t) + c_1 z_1 (p_{ij} - x_{ij}(t)) + c_2 z_2 (g_j - x_{ij}(t))$$

The three terms are:
1. **Inertia** $v_{ij}(t)$: momentum from the previous velocity
2. **Cognitive component** $c_1 z_1 (p_{ij} - x_{ij})$: pull toward personal best (scaled by $c_1$ and random $z_1 \sim U(0,1)$)
3. **Social component** $c_2 z_2 (g_j - x_{ij})$: pull toward global best (scaled by $c_2$ and random $z_2 \sim U(0,1)$)

Position update: $x_{ij}(t+1) = x_{ij}(t) + v_{ij}(t+1)$

#### Part (f) [4 marks]

In [[multi-objective-optimization]], when all objectives are to be **minimised**, solution $\mathbf{a}$ **dominates** solution $\mathbf{b}$ (written $\mathbf{a} \succ \mathbf{b}$) if and only if:

1. $\mathbf{a}$ is **at least as good as** $\mathbf{b}$ on **every** objective: $f_m(\mathbf{a}) \leq f_m(\mathbf{b})$ for all $m$, AND
2. $\mathbf{a}$ is **strictly better than** $\mathbf{b}$ on **at least one** objective: $f_m(\mathbf{a}) < f_m(\mathbf{b})$ for at least one $m$

If neither $\mathbf{a} \succ \mathbf{b}$ nor $\mathbf{b} \succ \mathbf{a}$, the solutions are **mutually non-dominating** — each is better on at least one objective and worse on at least one other. Neither can be considered objectively superior.

**Using dominance for selection quality:**
- Solutions that are not dominated by any other solution in the population form the **Pareto front** (rank 0) — these are the best trade-offs available.
- In [[nsga-ii]], the dominance relation is applied in non-dominated sorting: rank 0 = Pareto front; rank 1 = Pareto front after removing rank 0; and so on. Lower rank = higher quality. When two solutions share the same rank, **crowding distance** (sparsity in objective space) acts as a tiebreaker.

#### Part (g) [4 marks]

**Early stopping** is a regularisation technique used when training [[neural-networks]] (and other ML models) to prevent overfitting.

**Principle:**

```
Training error ↓                   Validation error ↓ then ↑
              |                                       |
              |      UNDERFITTING → GOOD FIT → OVERFITTING
              |________________________|_______________|
                                  ↑
                         Stop training HERE
                         (point of minimum validation error)
```

The model is trained on a training set; at regular intervals, its error is evaluated on a separate **validation set** (not used for weight updates). Training is halted when the validation error stops decreasing and begins to increase, even though training error continues to fall.

**Diagram (conceptual):**

```
Error
  |
  |  \  Training error (always decreasing)
  |   \_____________________________________________
  |
  |      \  Validation error
  |       \___________/‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾
  |                   ↑
  |             Best generalisation
  |             (early stopping point)
  |_________________________________________________
                                              Epochs
```

**Why it is important:**
- Prevents **overfitting**: a model that memorises training data loses the ability to generalise to unseen examples.
- Acts as a free regulariser: no extra hyperparameters beyond the patience threshold (number of epochs to wait).
- Saves computation: training stops before convergence on the training set, avoiding wasted computation in the overfitting regime.

#### Part (h) [4 marks]

In a [[self-organising-maps]] (SOM), weight update proceeds in three stages for each input data point $\mathbf{x}$:

**Step 1 — Competition (find the Best Matching Unit, BMU):**
Each neuron $i$ has a weight vector $\mathbf{w}_i$ of the same dimensionality as the input. The BMU is the neuron whose weight vector is closest to the input:

$$\text{BMU} = \arg\min_i \|\mathbf{x} - \mathbf{w}_i\|$$

**Step 2 — Cooperation (define a neighbourhood):**
A neighbourhood function $h(i, \text{BMU}, t)$ is centred on the BMU. It is typically a **Gaussian** that decreases with distance from the BMU on the map grid and also shrinks over training time $t$:

$$h(i, \text{BMU}, t) = \exp\left(-\frac{d(i, \text{BMU})^2}{2\sigma(t)^2}\right)$$

where $d(i, \text{BMU})$ is the grid distance between neuron $i$ and the BMU, and $\sigma(t)$ is the neighbourhood radius (decreases over time).

**Step 3 — Adaptation (update weights):**
The BMU and all neurons in its neighbourhood move their weight vectors toward the input:

$$\mathbf{w}_i \leftarrow \mathbf{w}_i + \eta(t) \cdot h(i, \text{BMU}, t) \cdot (\mathbf{x} - \mathbf{w}_i)$$

where $\eta(t)$ is the learning rate (also decreases over time). The BMU moves the most; neighbours move proportionally less according to $h$.

**Result:** Over many iterations, nearby neurons on the 2D grid come to represent similar input patterns — topographic organisation is preserved.

#### Part (i) [4 marks]

In a [[cellular-automata]], the state transition at time $t \to t+1$ works as follows:

1. **Snapshot the current states:** at time $t$, every cell has a defined state (e.g. on/off, 0/1). This full grid is the input to the transition.

2. **Apply the rule simultaneously to every cell:** for each cell $c$ in the grid:
   - Identify the states of all cells in $c$'s **neighbourhood** at time $t$ (e.g. Moore neighbourhood = 8 surrounding cells + the cell itself in 2D).
   - Look up (or compute) the **transition rule**: given this combination of neighbourhood states, what is $c$'s new state?
   - Record the new state for $c$.

3. **Simultaneity is critical:** all cells are updated **at the same time** using the state snapshot from time $t$. No cell's update at time $t+1$ depends on another cell's already-updated state — all transitions see the same $t$ snapshot.

4. **Replace:** after all new states are computed, replace the entire grid with the new states to give the configuration at time $t+1$.

**Example — Conway's Game of Life** (Moore neighbourhood, binary states):
- Dead cell + exactly 3 live neighbours → becomes alive
- Live cell + 2 or 3 live neighbours → stays alive
- All other live cells → die

The application of these two simple local rules simultaneously to every cell at every timestep produces all of the Game of Life's complex emergent behaviour (gliders, oscillators, etc.).

### Answer

#### Part (a) [6 marks]

[[swarm-intelligence]] algorithms share these core properties:

1. **Simple individual agents:** each agent follows a small, fixed rule set. No individual has a global view of the problem.
2. **Local interactions only:** agents interact only with immediate neighbours or with the local environment — no global communication or central coordinator.
3. **No central control / decentralisation:** every agent is identical and equal; no leader gives instructions. The system is self-organising.
4. **Emergent collective behaviour:** the group-level intelligent behaviour arises bottom-up from local rules. The swarm can do what no individual can.
5. **Positive and negative feedback:** positive feedback (pheromone reinforcement; global-best attraction) amplifies good solutions. Negative feedback (evaporation; velocity noise) prevents premature convergence and maintains diversity.
6. **Robustness:** the system degrades gracefully when agents fail — no single point of failure.

**Two approaches:**

- [[ant-colony-optimization]] (ACO) — ants deposit pheromone trails on graph edges; indirect stigmergic communication guides the colony to find short paths. Canonical for discrete/combinatorial problems.
- [[pso]] (Particle Swarm Optimisation) — particles in continuous space update velocity using personal best and global best positions; direct sharing of gbest. Canonical for continuous optimisation.

---

#### Part (b)(i) [6 marks]

**Constructing the ACO construction graph for multi-machine job scheduling:**

**Nodes:** represent **jobs** (A–H) plus a **start node** and, optionally, an **end node** for each machine.

**Edges:** a directed edge from job $u$ to job $v$ (for the same machine) represents "perform job $v$ immediately after job $u$ on this machine". The construction graph allows any ordering of the unscheduled jobs.

**Structure:**
- There are **M = 2 machines**. Each ant must construct M sequences (one per machine) that together cover all J = 8 jobs with no job assigned more than once.
- The pheromone matrix $\tau_{ij}$ encodes how desirable it is to schedule job $j$ immediately after job $i$ on a given machine.
- The heuristic $\eta_{ij}$ can encode urgency — e.g. $\eta_{ij} = 1/\text{deadline}(j)$ so jobs with tighter deadlines are preferred next.

**Construction process:** At each step, an ant selects the next unassigned job for the currently active machine using the standard ACO transition rule:

$$P_{ij}^k = \frac{[\tau_{ij}]^\alpha \cdot [\eta_{ij}]^\beta}{\sum_{l \in \text{allowed}} [\tau_{il}]^\alpha \cdot [\eta_{il}]^\beta}$$

The ant switches to the other machine after deciding to (or when one machine's capacity is full).

**Constraints the ants must respect:**

1. **No duplicate assignment:** each job is assigned to exactly one machine and appears exactly once in the solution. Once a job is added to any machine's sequence, it is removed from the "allowed" set for both machines.
2. **Capacity constraint:** the total processing time assigned to each machine cannot exceed 15 hours. Jobs whose addition would exceed remaining capacity cannot be placed on that machine.
3. **Sequential execution:** jobs on the same machine are executed one after another (no parallelism on a single machine). The start time of job $j$ on machine $m$ = finish time of the previous job on machine $m$.
4. **All jobs must be scheduled:** the ant's solution is only valid once all 8 jobs are assigned (to one of the two machines).

#### Part (b)(ii) [3 marks]

**Proposed fitness function:**

The **lateness** of a job $j$ is:

$$L_j = \max(0,\ C_j - d_j)$$

where $C_j$ is the completion time of job $j$ (in hours from 9am) and $d_j$ is its deadline (in hours from 9am). Lateness is zero if the job finishes before or on its deadline.

A fitness function to **minimise** over all jobs:

$$F = \sum_{j=1}^{J} L_j = \sum_{j=1}^{J} \max(0,\ C_j - d_j)$$

This is the **total tardiness** — the sum of lateness across all jobs. A lower value is better; $F = 0$ means all jobs meet their deadlines. Since ACO typically maximises, we can use $1/(1 + F)$ as the fitness to be maximised, or equivalently minimise $F$ directly.

#### Part (c) [7 marks]

**Principle of graceful degradation:**

**Graceful degradation** (also called **fault tolerance**) is the property whereby a system continues to function — with reduced but non-catastrophic performance — when some of its components are damaged or removed. The system degrades gradually rather than failing suddenly.

In [[neural-networks]], this arises because knowledge is **distributed** across all weights in the network rather than being localised in any single neuron or weight. Removing one neuron or connection partially disrupts the learned representations, causing a small performance drop, but no single unit encodes any single piece of information completely — so total failure requires removing a large fraction of the network.

This contrasts sharply with:
- **Conventional computers (von Neumann architecture):** a single corrupted bit in a program address or instruction can cause complete failure.
- **Symbolic AI (expert systems):** removing a single rule from a rule-base can break the system's ability to handle entire classes of inputs.

**Proposed experiment to demonstrate graceful degradation in an ANN:**

1. **Train** a multi-layer perceptron (MLP) on a classification task (e.g. classifying handwritten digits — MNIST) until it achieves good accuracy (say ≥ 95% on a held-out test set).

2. **Baseline:** record test accuracy with the full network.

3. **Ablation test — progressive neuron removal:**
   - Randomly select a fraction $p$ of the hidden-layer neurons and disable them (set their outputs to zero or remove them).
   - Evaluate test accuracy on the same held-out set.
   - Repeat for $p \in \{5\%, 10\%, 20\%, 30\%, 50\%, 70\%, 90\%\}$.

4. **Plot** accuracy vs. fraction of neurons removed.

**Expected result demonstrating graceful degradation:** accuracy decreases smoothly and gradually as more neurons are removed. The system still achieves reasonable performance even with 30–50% of neurons disabled. Only at very high removal rates (e.g. >70–80%) does accuracy fall sharply toward chance level.

**Contrast with a fragile system:** a rule-based system (expert system) tested with the same proportion of its rules deleted would show abrupt, unpredictable failures for specific input classes even at low deletion rates.

This experiment directly demonstrates that the network's knowledge is encoded distributedly across many weights, not in isolated components.
