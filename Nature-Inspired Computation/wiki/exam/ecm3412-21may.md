## Question 1 — Representations, Operators, and Exploration vs Exploitation

### Part (a)

The following describes three representations that might be used within an evolutionary algorithm:

(1) Permutation representation using the letters from A to L
(2) Binary representation of length 12
(3) K-Ary representation of length 12 and a K of 8

**(i) Provide an example of a single random solution for each of the representation types.**

**(6 marks)**

**(ii) Name and briefly describe the most appropriate mutation operators for each representation.**

**(6 marks)**

**(iii) Which representation would be most useful for the following problems? Explain your answer.**

A) The knapsack problem, where decisions must be made as to which items to pack into a knapsack, maximising value within weight constraints

B) The travelling salesman problem, where a route must be devised that visits all cities once and doesn't revisit any city

C) Water distribution network optimisation, where a set of pipe diameters are selected from a discrete set

**(6 marks)**

**(iv) Is standard single-point crossover likely to work well for all these representations? Explain your answer.**

**(4 marks)**

### Answer

#### Part (a)(i) [6 marks]

**(1) Permutation representation (A–L):** each letter appears exactly once; a random solution is any permutation of the 12 letters, e.g.:

```
E, B, J, A, L, G, C, I, F, K, D, H
```

**(2) Binary representation of length 12:** each gene is 0 or 1, e.g.:

```
0, 1, 1, 0, 0, 1, 0, 1, 1, 0, 0, 1
```

**(3) K-Ary representation of length 12, K=8:** each gene is an integer in $[0, 7]$, e.g.:

```
3, 7, 0, 5, 2, 6, 1, 4, 7, 3, 0, 5
```

See [[representations]] for the full taxonomy of encoding types.

#### Part (a)(ii) [6 marks]

**(1) Permutation — Swap mutation:** choose two positions at random and swap the elements they hold. This is the correct operator because simple single-gene random replacement would create duplicates (breaking the permutation constraint). E.g. swapping positions 2 and 5 in `E,B,J,A,L,G,...` gives `E,L,J,A,B,G,...`.

**(2) Binary — Bit-flip mutation:** choose one gene (bit) at random and flip it (0→1 or 1→0). This is the canonical mutation for binary encodings and is the smallest meaningful perturbation.

**(3) K-Ary — Single-gene mutation:** choose one gene at random and replace it with a new value drawn uniformly from $[0, K-1]$ (i.e. from $\{0,1,2,3,4,5,6,7\}$). The new value should be different from the current one. This operator is natural for integer/categorical encodings where any value in the range is valid.

See [[crossover-mutation]] for the full operator reference.

#### Part (a)(iii) [6 marks]

**A) Knapsack problem → Binary representation (2)**

Each gene corresponds to one item: 1 means "pack this item", 0 means "leave it out". The chromosome directly encodes an include/exclude decision for each of the 12 items. A binary string of length 12 naturally represents all $2^{12}$ possible subsets. The permutation and k-ary representations would be unnecessarily complex for a binary include/exclude decision.

**B) Travelling salesman problem → Permutation representation (1)**

TSP requires visiting every city exactly once. A permutation directly encodes the order in which cities are visited and structurally guarantees that each city appears exactly once. Binary or k-ary representations would require external repair mechanisms to ensure all cities are visited without revisiting — the permutation encoding makes validity structurally guaranteed.

**C) Water distribution network optimisation → K-Ary representation (3)**

Pipe diameters are selected from a discrete set of possible sizes. If there are up to 8 possible diameter choices (K=8), each gene of the k-ary chromosome represents one pipe's chosen diameter as an index into the discrete set. This maps naturally to the problem. Binary could work (encoding each index in binary) but k-ary is more direct and avoids decoding overhead.

#### Part (a)(iv) [4 marks]

Standard single-point crossover works well for the **binary (2)** and **k-ary (3)** representations but **not** for the **permutation representation (1)**.

- **Binary and k-ary:** single-point crossover produces valid children — the prefix from parent 1 and suffix from parent 2 are both valid values, and the resulting child has exactly 12 genes each with a valid value. No constraints are violated.

- **Permutation:** single-point crossover will almost certainly produce **invalid children** — genes that appear twice (from both parents) and genes that are absent. For example, if parent 1 contributes `E,B,J,A` and parent 2 contributes `G,A,C,E,F,K,D,H`, the child contains A and E twice and is missing some letters entirely. A repair step (such as Order Crossover / OX) is required to restore validity for permutation representations.

See [[crossover-mutation]] for the OX operator details.

### Part (b)

Nature-inspired methods attempt to balance *exploration* against *exploitation* during the search process. This balance can often be altered through the modification of the algorithm's parameters. For each of the parameters shown below, write down a value that would promote exploration and a value that would promote exploitation and explain why these values produce these effects.

**(i) Tournament size (with a population size of 100).**

**(4 marks)**

**(ii) Population size for an evolutionary algorithm.**

**(4 marks)**

**(iii) c2.**

**(4 marks)**

### Answer

#### Part (b)(i) [4 marks] — Tournament size

See [[selection]] for the full tournament selection explanation.

| | Value | Why |
|--|-------|-----|
| **Exploration** | Tournament size = 2 | With a small tournament (e.g. 2), any individual in the population has a reasonable chance of being selected, even weaker ones. This maintains diversity: many different solutions breed, the search explores many landscape regions simultaneously. |
| **Exploitation** | Tournament size = 20 (or larger, e.g. 50) | A large tournament almost always picks the best individual in the sampled group. Selection pressure is high: only the fittest breed, the population converges rapidly toward the current best region of the landscape. |

Extreme: tournament size = 1 → pure random selection (no pressure at all); tournament size = 100 (= population size) → always picks the single best individual (maximum exploitation, no diversity).

#### Part (b)(ii) [4 marks] — Population size

See [[evolutionary-algorithms]] for the parameter effects table.

| | Value | Why |
|--|-------|-----|
| **Exploration** | Large population (e.g. 500–1000) | More individuals means greater diversity — many different regions of the search space are sampled simultaneously. Crossover can combine solutions from distant landscape regions, and there is less risk of premature convergence to a local optimum. |
| **Exploitation** | Small population (e.g. 10–20) | With few individuals, selection pressure causes rapid convergence. The small number of solutions are all focused in a narrow region of the landscape, refining local solutions quickly. However, diversity is low and premature convergence is a significant risk. |

#### Part (b)(iii) [4 marks] — $c_2$ (social coefficient in PSO)

$c_2$ is the **social (global) acceleration coefficient** in [[pso]]. It scales the pull of each particle toward the swarm's global best position (**gbest**):

$$v_{ij}(t+1) = v_{ij}(t) + c_1 z_1 (p_{ij} - x_{ij}) + c_2 z_2 (g_j - x_{ij})$$

| | Value | Why |
|--|-------|-----|
| **Exploitation** | High $c_2$ (e.g. $c_2 = 4.0$) | Particles are strongly pulled toward **gbest**, the best solution found so far by any particle. The swarm converges quickly toward this single point — effective exploitation of the currently best-known region. Risk: premature convergence if gbest is a local optimum. |
| **Exploration** | Low $c_2$ (e.g. $c_2 = 0.5$) | Weak social pull means particles rely more on their own personal history ($c_1$) and inertia. They spread out more independently through the search space, exploring diverse regions. |

Note: setting $c_2 = 0$ entirely removes social learning — particles only follow their personal best, becoming isolated hillclimbers with no information sharing.

### Part (c)

Multi-layer perceptrons and self-organising maps are both methods inspired by the brain. Briefly explain three main differences between these two neural techniques.

**(6 marks)**

### Answer

See [[neural-networks]] and [[self-organising-maps]] for full details.

| Dimension | Multi-Layer Perceptron (MLP) | Self-Organising Map (SOM) |
|-----------|------------------------------|---------------------------|
| **1. Learning type** | **Supervised** — requires labelled training data (input–target pairs). The network learns to map inputs to known desired outputs. | **Unsupervised** — requires only input data, no targets or labels. The network discovers structure and patterns without guidance. |
| **2. Training algorithm** | **Backpropagation** — error is computed at the output layer and propagated backwards through the network to update weights, using the chain rule and the derivative of the activation function. | **Competitive learning (BMU + neighbourhood update)** — the winning neuron (Best Matching Unit) and its spatial neighbours have their weight vectors moved toward the current input. No error signal or backpropagation involved. |
| **3. Output and purpose** | Produces a **class label or numeric value** — it is a function approximator/classifier solving a specific prediction task. Output is a single value or class probability. | Produces a **2D map position** — it is a topology-preserving dimensionality reduction. The output is which neuron in the 2D grid "wins" for a given input, used for clustering and visualisation, not prediction. |

Additional differences worth noting: MLPs use activation functions (sigmoid, ReLU) and compute weighted sums of inputs; SOMs use Euclidean distance to find the BMU and have no activation functions. MLPs have directed connections between layers; SOMs have a flat 2D grid architecture with no inter-neuron connections (only weight vectors).

## Question 2 — Lecture Timetabling Design Problem

The University has asked you to develop a new lecture timetabling solution based on a nature-inspired algorithm of your choice. The algorithm should allocate lectures for a set of modules to a finite set of lecture rooms with varying capacity within the University. The number of students on each module will be provided to you, along with the capacity of each room and the staff members required to teach and their availability. Students would prefer not to have lectures at 8.30am and 5.30pm.

**(a) Describe, and appropriately label, the hard and soft constraints of this problem.**

**(5 marks)**

**(b) Describe how you would formulate the objective function(s) for this problem.**

**(5 marks)**

**(c) Describe and justify your choice of algorithm to solve this problem and describe the representation you would use.**

**(8 marks)**

**(d) Describe the parameter settings you would use and what experiments you would conduct to determine the efficacy of your choices.**

**(7 marks)**

**(e) Explain what 'success' would look like on completion of this project.**

**(5 marks)**

### Answer

#### Part (a) [5 marks] — Hard and soft constraints

**Hard constraints** (must not be violated — any solution that violates these is infeasible):

1. No two lectures can be scheduled in the same room at the same time slot.
2. A lecturer cannot be required to teach two different modules at the same time.
3. The capacity of the assigned room must be at least as large as the number of students enrolled on the module (room not overcrowded).
4. A lecturer can only be assigned to a time slot they are available for (respects staff availability).
5. Every module must be assigned a time slot and a room (all lectures are scheduled).

**Soft constraints** (desirable but not mandatory — violations reduce solution quality):

1. Students would prefer not to have lectures at 8.30am or 5.30pm (unpopular times).
2. Lectures for the same module should ideally be spread across the week rather than clustered.
3. Students should not have two lectures simultaneously (student clash minimisation).
4. Room utilisation should be reasonable — excessively large rooms for small classes are wasteful.

#### Part (b) [5 marks] — Objective function formulation

The objective function penalises violations and should be minimised. A common approach for constrained timetabling uses a **weighted penalty function**:

$$F(\mathbf{x}) = W_H \cdot \text{HardViolations}(\mathbf{x}) + W_S \cdot \text{SoftViolations}(\mathbf{x})$$

where $W_H \gg W_S$ to ensure hard constraints dominate.

**Hard constraint penalty:** count the total number of hard constraint violations (e.g. number of room double-bookings + number of lecturer clashes + number of over-capacity assignments). Each violation adds a fixed large penalty.

**Soft constraint penalty:** count the number of lectures assigned to 8.30am or 5.30pm, weighted by module size (more students affected = higher penalty). Also add a penalty for student clashes if student timetable data is available.

A perfectly feasible schedule has HardViolations = 0. At that point the algorithm optimises purely for soft constraint satisfaction. A two-phase approach can be used: first minimise hard violations, then (once feasible solutions are found) minimise soft violations. See [[evolutionary-algorithms]] for penalty function approaches in constrained optimisation.

#### Part (c) [8 marks] — Algorithm choice and representation

**Algorithm: Genetic Algorithm (GA)** — see [[genetic-algorithms]] and [[evolutionary-algorithms]].

**Justification:**

Timetabling is a classic combinatorial optimisation problem with a large, discrete search space and many constraints. A GA is well-suited because:

1. The search space is too large for exact methods (exponential in the number of lectures × time slots × rooms).
2. The landscape is multimodal (many locally good timetables) — the GA's population-based search and crossover help avoid local optima.
3. The penalty-function objective is straightforward to evaluate, and the GA's flexible fitness evaluation accommodates it.
4. GAs have strong literature support for timetabling problems specifically.

**Representation:**

An **integer vector (direct encoding)**: one gene per module-lecture, where the gene value encodes both the assigned time slot and room as a single integer index (or alternatively two separate integers: one for the time slot and one for the room). With $T$ time slots and $R$ rooms, there are $T \times R$ possible assignments per lecture.

Example: for 20 modules, 10 time slots, and 5 rooms, the chromosome has 20 genes each in range $[0, 49]$, where gene value $k = t \times R + r$ encodes time slot $t$ and room $r$.

- **Mutation:** single-gene mutation — randomly reassign one lecture to a new (time slot, room) combination.
- **Crossover:** single-point or uniform crossover — combine assignment decisions from two parent timetables.

An **indirect encoding** is an alternative: genes encode a priority ordering of modules; a deterministic decoder places each module in the best available slot. This reduces the number of hard-constraint violations structurally but requires domain-specific decoder logic.

#### Part (d) [7 marks] — Parameter settings and experiments

**Initial parameter choices (starting point):**

| Parameter | Initial setting | Rationale |
|-----------|----------------|-----------|
| Population size | 200 | Sufficient diversity for a complex landscape |
| Tournament size | 5 | Moderate selection pressure |
| Mutation rate | 0.05 (5%) | Enough perturbation without destroying good solutions |
| Crossover rate | 0.8 | High recombination as primary operator |
| Generations | 5,000 | Adequate budget for convergence |
| $W_H$ | 1,000 | Hard violations dominate |
| $W_S$ | 1 | Soft violations secondary |

**Experiments to determine efficacy:**

1. **Multiple independent runs:** Run the GA ≥30 times with different random seeds to account for stochasticity. Report mean, standard deviation, best, and worst solution quality. This tests robustness.

2. **Parameter sensitivity analysis:** Vary one parameter at a time (tournament size ∈ {2, 5, 10, 20}; population size ∈ {50, 100, 200, 500}; mutation rate ∈ {0.01, 0.05, 0.1}). Measure mean final fitness and convergence speed to identify parameter sensitivity.

3. **Convergence plots:** Plot best fitness vs. generation to detect premature convergence (flat early) or slow convergence (still improving at generation limit). Adjust parameters accordingly.

4. **Comparison against a baseline:** Compare GA results against a simple greedy heuristic (assign each lecture to the first feasible slot) to demonstrate added value.

5. **Statistical testing:** Use a Wilcoxon signed-rank test or t-test to confirm that performance differences between parameter settings are statistically significant rather than due to chance.

#### Part (e) [5 marks] — Definition of success

Success has multiple layers:

1. **Feasibility:** The algorithm consistently produces timetables that satisfy all hard constraints (zero hard violations) — no room double-bookings, no lecturer clashes, all rooms appropriately sized, all staff availability respected.

2. **Quality:** Among feasible solutions, the timetable minimises soft constraint violations — very few or no lectures scheduled at 8.30am/5.30pm, minimal student clashes, good room utilisation.

3. **Reliability:** The algorithm finds a feasible solution in the majority (e.g. ≥95%) of independent runs, within a time budget acceptable for operational use (e.g. runs within 10 minutes on standard hardware).

4. **Improvement over baseline:** The GA produces measurably better timetables than the current manual or greedy approach, in terms of soft constraint satisfaction.

5. **Stakeholder acceptance:** The generated timetable is accepted by the university scheduling team as practically usable, with no obvious flaws that algorithmic metrics do not capture (e.g. geographic spread of rooms across campus).

## Question 3 — Perceptron and MLP

You have been given a classification problem to solve with a perceptron with an activation function, $\varphi$ that takes the value 1 when the input is greater than or equal to 0, and 0 otherwise. The training set for the problem is defined as follows:

| Input ($x_1$) | Input ($x_2$) | Target ($y$) |
|----------------|----------------|---------------|
| 0              | 3              | 0             |
| -1             | 2              | 0             |
| -1             | 0              | 1             |
| 1              | 2              | 1             |

**(a) Sketch a decision boundary that separates the two different classes in y and specify its equation.**

**(6 marks)**

**(b) Design a single-neuron perceptron to solve this classification problem by drawing a diagram and writing down a possible solution of weights and biases that would lead to the behaviour above.**

**(8 marks)**

**(c) Test your solution with all four input patterns, writing down your calculations.**

**(8 marks)**

**(d) Describe the changes that would be necessary to convert this perceptron into a multi-layer perceptron (MLP).**

**(4 marks)**

**(e) Describe the main steps of training the MLP on training data.**

**(4 marks)**

### Answer

#### Part (a) [6 marks] — Decision boundary

Plotting the four points in the $(x_1, x_2)$ plane:

- Class 0 (target = 0): $(0, 3)$ and $(-1, 2)$ — these are the "upper" points.
- Class 1 (target = 1): $(-1, 0)$ and $(1, 2)$ — these are the "lower-right" points.

A linear decision boundary that separates class 0 from class 1 must place $(0,3)$ and $(-1,2)$ on one side, and $(-1,0)$ and $(1,2)$ on the other.

One valid boundary is the line:

$$x_1 - x_2 + 1.5 = 0 \quad \Longleftrightarrow \quad x_2 = x_1 + 1.5$$

Check:
- $(0, 3)$: $0 - 3 + 1.5 = -1.5 < 0$ → class 0. Correct.
- $(-1, 2)$: $-1 - 2 + 1.5 = -1.5 < 0$ → class 0. Correct.
- $(-1, 0)$: $-1 - 0 + 1.5 = 0.5 > 0$ → class 1. Correct.
- $(1, 2)$: $1 - 2 + 1.5 = 0.5 > 0$ → class 1. Correct.

All four points are correctly classified by this boundary. The boundary is the line $x_2 = x_1 + 1.5$.

#### Part (b) [8 marks] — Perceptron design

A single-neuron perceptron computes:

$$\text{net} = w_1 x_1 + w_2 x_2 + b$$
$$\text{output} = \varphi(\text{net}) = \begin{cases} 1 & \text{if net} \geq 0 \\ 0 & \text{otherwise} \end{cases}$$

From the decision boundary $x_1 - x_2 + 1.5 = 0$, we can read off:

$$w_1 = 1, \quad w_2 = -1, \quad b = 1.5$$

This means the neuron fires (output = 1) when $x_1 - x_2 + 1.5 \geq 0$, i.e. when the point is on the class 1 side of the boundary.

**Diagram (described textually):**

```
x1 ──[w1 = 1.0 ]──┐
                   ├──[ Σ + bias b=1.5 ]──[ φ: output 1 if ≥0 ]──► output y
x2 ──[w2 = -1.0]──┘
```

The neuron takes two weighted inputs plus a bias, sums them, and passes the result through the step activation function $\varphi$.

#### Part (c) [8 marks] — Testing the solution

Using $w_1 = 1$, $w_2 = -1$, $b = 1.5$:

**Pattern 1:** $x_1 = 0,\ x_2 = 3$, target $y = 0$

$$\text{net} = (1)(0) + (-1)(3) + 1.5 = 0 - 3 + 1.5 = -1.5$$
$$\varphi(-1.5) = 0 \quad \checkmark \quad (\text{matches target } 0)$$

**Pattern 2:** $x_1 = -1,\ x_2 = 2$, target $y = 0$

$$\text{net} = (1)(-1) + (-1)(2) + 1.5 = -1 - 2 + 1.5 = -1.5$$
$$\varphi(-1.5) = 0 \quad \checkmark \quad (\text{matches target } 0)$$

**Pattern 3:** $x_1 = -1,\ x_2 = 0$, target $y = 1$

$$\text{net} = (1)(-1) + (-1)(0) + 1.5 = -1 + 0 + 1.5 = 0.5$$
$$\varphi(0.5) = 1 \quad \checkmark \quad (\text{matches target } 1)$$

**Pattern 4:** $x_1 = 1,\ x_2 = 2$, target $y = 1$

$$\text{net} = (1)(1) + (-1)(2) + 1.5 = 1 - 2 + 1.5 = 0.5$$
$$\varphi(0.5) = 1 \quad \checkmark \quad (\text{matches target } 1)$$

All four patterns produce the correct output. The perceptron correctly classifies the entire training set.

#### Part (d) [4 marks] — Converting to an MLP

See [[neural-networks]] for the full MLP description.

The following changes are required to convert the single-neuron perceptron into a Multi-Layer Perceptron:

1. **Add one or more hidden layers:** introduce a layer of neurons between the input layer and the output neuron. Each hidden neuron receives weighted inputs from all input nodes and produces an output via its own activation function.

2. **Replace the step activation function with a differentiable one:** the step function used above ($\varphi$) is not differentiable, which prevents backpropagation. Replace it with a smooth, differentiable activation function such as the **sigmoid** $\sigma(a) = 1/(1+e^{-a})$ or ReLU. This is required for training via gradient descent.

3. **Add more weight connections:** every input node is connected to every hidden neuron, and every hidden neuron is connected to every output neuron. With $I$ inputs, $H$ hidden neurons, and $O$ outputs, there are $I \times H + H \times O$ weights plus biases.

4. **The output layer** may now contain multiple neurons if the problem requires multi-class classification.

These changes allow the MLP to solve non-linearly separable problems (such as XOR) that a single neuron cannot handle.

#### Part (e) [4 marks] — Training the MLP

See [[neural-networks]] for the backpropagation equations.

The main steps are:

1. **Initialise weights** to small random values (e.g. drawn from $\mathcal{N}(0, 0.1)$).

2. **Forward pass:** for each training pattern, feed the input through the network layer by layer, computing the weighted sum and activation at each neuron, until an output is produced.

3. **Compute error:** compare the network output to the target label using an error measure (e.g. sum of squared errors, or cross-entropy for classification).

4. **Backward pass (backpropagation):** propagate the error signal backwards from the output layer to the hidden layers. For each weight $w_{ij}$, compute the gradient of the error with respect to that weight using the chain rule:
   $$\Delta w_{ij} = \eta \cdot \delta_{pj} \cdot o_{pj}$$
   where $\delta_{pj}$ is the error signal at node $j$, $o_{pj}$ is the output of node $j$, and $\eta$ is the learning rate. For output units: $\delta_{pj} = f'(o_{pj})(t_{pj} - o_{pj})$; for hidden units: $\delta_{pj} = f'(o_{pj})\sum_k \delta_{pk} w_{pk}$.

5. **Update weights:** adjust all weights in the direction that reduces the error.

6. **Repeat** over all training patterns (one epoch) and iterate over many epochs until the error converges to an acceptable level or a maximum number of epochs is reached.
