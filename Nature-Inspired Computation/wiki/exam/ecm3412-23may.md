### Answer

#### Part (a) [6 marks]

**Exact algorithms** guarantee finding the optimal solution by systematically exploring the entire (or provably sufficient) search space. They are complete and optimal, but computationally infeasible for large or NP-hard problems because their runtime grows exponentially or combinatorially with problem size.

- Example: **dynamic programming** / **branch-and-bound** for the Travelling Salesman Problem (TSP). For $n$ cities, an exhaustive search examines $(n-1)!/2$ possible tours.

**Approximate algorithms** (heuristics and metaheuristics) sacrifice the guarantee of optimality in exchange for finding good (not necessarily optimal) solutions in tractable time. They are designed for problems where exact methods are too slow.

- Example: a **Genetic Algorithm (GA)** ([[genetic-algorithms]]) applied to TSP. It finds a near-optimal tour quickly, but cannot certify it is the shortest possible.

Key distinction: exact algorithms always find the best solution given enough time; approximate algorithms find solutions that are "good enough" in practical time.

#### Part (b) [7 marks]

See [[representations]] for full detail.

**Direct encoding:** each gene in the chromosome maps directly to a variable or decision in the problem.

**Indirect encoding:** each gene encodes a parameter for a constructive heuristic or rule, which then builds the solution. The chromosome does not directly represent the solution.

| Dimension | Direct encoding | Indirect encoding |
|-----------|-----------------|-------------------|
| **What the gene represents** | A decision variable directly (e.g. exam in time slot 4) | A rule/parameter of a constructive procedure (e.g. "use 4th clash-free slot for this exam") |
| **Invalid solutions** | Common — must penalise fitness or use repair | Rare — heuristic builds only valid solutions by construction |
| **Search space size** | Larger (includes infeasible regions) | Smaller (infeasible solutions removed structurally) |
| **Landscape smoothness** | Often smoother | Can be more rugged (indirect mapping may be discontinuous) |
| **Decoding speed** | Fast — chromosome is read directly | Slower — must run the constructive heuristic |
| **Domain knowledge required** | Not required | Required to design the heuristic |
| **Advantage** | Simple to implement; broadly applicable | Fewer wasted evaluations on invalid solutions; exploits problem structure |
| **Disadvantage** | Wasted search effort on invalid regions; constraint handling needed | Requires domain expertise; landscape harder to navigate |

**Key insight:** Indirect encodings shrink the effective search space by embedding problem constraints into the representation itself, at the cost of complexity and domain knowledge.

#### Part (c) [6 marks]

See [[swarm-intelligence]] for full detail.

**Properties of the swarm intelligence (SI) algorithm family:**

1. **Simple individual agents** — each agent follows a small, local rule set. No single agent is intelligent.
2. **Local interactions only** — agents interact only with immediate neighbours or with their local environment; there is no global broadcast.
3. **No central controller** — the system is fully decentralised; every agent is equal. No leader gives instructions.
4. **Emergent collective behaviour** — the intelligent group-level capability arises bottom-up from local rules and interactions. The swarm can do what no individual can.
5. **Stigmergy (in ACO-type algorithms)** — indirect communication via environmental modification (e.g. pheromone trails).

**Two swarm intelligence algorithms:**

1. **Ant Colony Optimisation (ACO)** ([[ant-colony-optimization]]) — ants communicate via pheromone trails on a graph; collectively find shortest paths.
2. **Particle Swarm Optimisation (PSO)** ([[pso]]) — particles navigate a continuous space guided by their own best-known position (pbest) and the swarm's best-known position (gbest).

#### Part (d) [7 marks]

See [[pso]] for full detail.

**Main algorithmic elements of PSO:**

**Population initialisation:** $N$ particles are initialised with random positions $x_i$ and random velocities $v_i$ in the search space.

**Personal best (pbest):** Each particle $i$ stores the best position it has personally visited, $p_i$.

**Global best (gbest):** The best position found by any particle across the entire swarm, $g$.

**Velocity update:**
$$v_{ij}(t+1) = v_{ij}(t) + c_1 \cdot z_1 \cdot (p_{ij} - x_{ij}(t)) + c_2 \cdot z_2 \cdot (g_j - x_{ij}(t))$$

Three components:
- **Inertia** $v_{ij}(t)$: momentum from the previous velocity
- **Cognitive** $c_1 z_1 (p_{ij} - x_{ij})$: pull toward the particle's own best
- **Social** $c_2 z_2 (g_j - x_{ij})$: pull toward the swarm's global best

**Position update:**
$$x_{ij}(t+1) = x_{ij}(t) + v_{ij}(t+1)$$

**Parameters and feasible settings:**

| Parameter | Typical value | Effect of increasing |
|-----------|--------------|---------------------|
| Swarm size $N$ | 20–50 | Better global search; slower per iteration |
| Cognitive coefficient $c_1$ | 2.0 | Particles trust personal history more; slower social convergence |
| Social coefficient $c_2$ | 2.0 | Particles follow gbest more strongly; faster convergence, less diversity |
| $z_1, z_2$ | $\sim U(0,1)$, resampled each step | Randomness prevents premature convergence |

Termination: maximum iterations reached, or swarm radius ≈ 0 (converged), or no improvement for $k$ iterations.

#### Part (e) [6 marks]

See [[flocking-boids]] for full detail.

**Three examples of flocking behaviour and their benefits:**

1. **V-formation flight in birds (e.g. geese)**
   - Description: birds fly in a V shape, each positioned slightly behind and to the side of the one ahead.
   - Benefit: the leading bird's wingtips create upwash that reduces drag for the bird behind. Geese in V-formation travel up to **70% further** on the same energy compared to flying alone.

2. **Fish schooling (e.g. sardines, herring)**
   - Description: fish form tightly coordinated shoals and perform synchronised manoeuvres such as flash expansion (the school suddenly opens outward) and the fountain effect (parting before a predator and rejoining behind it).
   - Benefit: **predator confusion** — the coordinated motion makes it very difficult for a predator to isolate and target an individual. Also, fish cast off slime that reduces water resistance for neighbours, lowering energy expenditure.

3. **Murmurations of starlings (or antelope herding / insect swarms)**
   - Description: hundreds of thousands of starlings move in complex 3D formations with no apparent leader, shifting direction instantaneously.
   - Benefit: **predator evasion** — the unpredictable, rapidly shifting mass confuses and intimidates raptors. The group also benefits from **many-eyes effect**: more individuals scanning for threats, reducing the vigilance cost per individual.

All three are enabled by Reynolds' three local rules (separation, alignment, cohesion) — no central control is needed. See [[flocking-boids]].

#### Part (f) [8 marks]

**Artificial Immune Systems (AIS)** draw inspiration from the biological immune system's ability to detect, recognise, and respond to pathogens, applying these mechanisms to computer security (intrusion detection, anomaly detection, virus detection).

**Four properties:**

1. **Uniqueness (Self/Non-self discrimination)**
   The immune system distinguishes the host's own cells ("self") from foreign material ("non-self"). In AIS for computer security, "self" is defined as the normal, legitimate pattern of system behaviour or network traffic. Anything deviating sufficiently from self is flagged as anomalous. This enables detection of previously unseen attacks, unlike signature-based systems that only detect known threats.

2. **Diversity (Immune cell repertoire)**
   The immune system maintains a highly diverse repertoire of T-cells and B-cells with different receptors, ensuring it can recognise a wide variety of pathogens. In AIS, a diverse population of detectors is evolved or generated — no single detector is responsible for all threats. This diversity makes the system robust: even if one detector type is evaded, others may still detect the threat. It mirrors the immune strategy of never relying on a single line of defence.

3. **Adaptability / Learning (Clonal selection and affinity maturation)**
   When the immune system encounters a pathogen, B-cells that best bind to it are clonally expanded and hypermutated to produce cells with even higher binding affinity — the response improves over time. AIS applies this as a learning mechanism: detectors are selected, replicated, and mutated toward better coverage of the threat space. The system can adapt to new types of attack by generating new detectors dynamically.

4. **Memory (Immunological memory)**
   After successfully defeating a pathogen, the immune system retains long-lived memory cells that allow a faster, stronger response to the same pathogen on re-exposure. In AIS, memory detectors are retained after a successful detection event, improving future response speed. This is analogous to a signature database, but generated by the system rather than manually compiled.

### Answer

#### Part (a) [8 marks]

See [[fitness-landscapes]] for full detail.

**Hillclimbing algorithm:**

Hillclimbing is a single-solution, local search algorithm:

```
1. Generate a random starting solution c; evaluate f(c)
2. Repeat until termination:
   a. Create a mutant m by applying a mutation operator to c
   b. Evaluate f(m)
   c. If f(m) >= f(c): replace c with m  (keep mutant if no worse)
   d. Otherwise: discard m (never accept a worse solution)
3. Return c
```

The name derives from the metaphor of climbing a fitness landscape: always move uphill, never downhill. The algorithm is fast and simple but critically, **it only accepts improvements** — once it reaches a local optimum (no nearby improvement exists), it is trapped there permanently.

**Limitation:** On multimodal landscapes (many local optima), the algorithm gets stuck at the first local optimum it reaches, which is very unlikely to be the global optimum. The final solution quality depends strongly on the random starting point.

---

**Two improvements:**

**Improvement 1: Monte Carlo (Random Restart) Search**
Accept a worse solution with some small fixed probability $p$ (e.g. $p = 0.1$). This allows the algorithm to occasionally "jump off" a local optimum and explore new regions. However, the acceptance probability is fixed and does not guide the search strategically. A more sophisticated version is **Simulated Annealing**, where the acceptance probability decreases over time (cooling schedule) — high early on (explores freely), low later (focuses on good solutions found).

**Improvement 2: Tabu Search**
Instead of evaluating only one mutant per step, evaluate *all* neighbours of the current solution and move to the best, **even if it is worse** than the current solution. To prevent cycling back to already-visited solutions, recently visited solutions are placed on a **tabu list** and are forbidden from being revisited for a number of steps. This allows the algorithm to escape local optima by accepting controlled downhill moves while the memory structure prevents it from going in circles.

Both improvements allow the search to escape local optima, something pure hillclimbing cannot do.

#### Part (b) [9 marks]

See [[selection]] for full detail.

**Tournament selection procedure:**

```
To select one parent:
1. Randomly draw t individuals from the population (with replacement)
2. Return the individual with the highest fitness among the t drawn
```

The parameter $t$ is the **tournament size**. The larger $t$, the more likely the best individual in the pool is the global best, and the higher the **selection pressure** — the degree to which fitter individuals are preferentially chosen.

---

**Case 1: Tournament size 5 in population size 5000**

With 5000 individuals, drawing 5 at random gives a very low chance of any single draw landing on the best or near-best individuals in the population. There is a wide spread of fitness across the population and substantial genetic diversity. Tournament size 5 out of 5000 ($t/N = 0.1\%$) represents a **very low selection pressure**:

- The winner of each tournament is likely to be a moderately above-average individual, not necessarily a top-rank individual.
- This keeps diversity high and slows convergence.
- The algorithm explores the search space broadly but may converge slowly.
- Effect: good for escaping local optima, maintaining population diversity; slow to exploit good solutions found.

**Case 2: Tournament size 5 in population size 20**

With only 20 individuals, drawing 5 at random samples **25% of the entire population** in each tournament. The probability that the globally best individual is in the tournament is $1 - (15/20)^5 \approx 76.7\%$. This represents a **very high selection pressure**:

- The best individual dominates selection — it is very frequently the tournament winner.
- The population converges rapidly toward the best known solution.
- Diversity collapses quickly; genetic variation is lost.
- Effect: fast convergence but high risk of premature convergence to a local optimum. The algorithm exploits good solutions aggressively but explores poorly.

**Summary comparison:**

| Case | $t/N$ ratio | Selection pressure | Diversity | Convergence speed | Risk |
|------|------------|-------------------|-----------|-------------------|------|
| $t=5$, $N=5000$ | 0.1% | Low | High | Slow | Under-convergence |
| $t=5$, $N=20$ | 25% | Very high | Low | Very fast | Premature convergence |

The key insight is that **tournament size alone does not define selection pressure** — it must be considered relative to population size.

### Answer

#### Part (a) [6 marks]

See [[multi-objective-optimization]] for full detail.

**Pareto front (definition):**

In multi-objective optimisation, we simultaneously optimise two or more conflicting objectives with no single solution that is best on all of them. A solution $\mathbf{a}$ **dominates** solution $\mathbf{b}$ if:
- $\mathbf{a}$ is at least as good as $\mathbf{b}$ on **every** objective, AND
- $\mathbf{a}$ is strictly better than $\mathbf{b}$ on **at least one** objective.

The **Pareto front** is the set of all **non-dominated** solutions — those for which no other feasible solution dominates them. A solution on the Pareto front cannot be improved on any one objective without worsening at least one other objective. These represent the full set of optimal trade-offs.

**Example — offshore wind farm layout:**

Two conflicting objectives:
- Minimise cost (fewer turbines is cheaper)
- Maximise power output (more turbines generate more power)

| Solution | Cost | Power | Dominated? |
|----------|------|-------|------------|
| A: 10 turbines | Low | Low | No |
| B: 20 turbines | Medium | Medium | No |
| C: 30 turbines | High | High | No |
| D: 20 turbines | Medium | Low | Yes (B dominates D: same cost, more power) |

Solutions A, B, C form the Pareto front — you cannot do better on power without increasing cost, and vice versa. Solution D is dominated and not on the front.

An MOEA such as [[nsga-ii]] aims to approximate the full Pareto front in a single run, giving the decision-maker a complete picture of all trade-offs.

#### Part (b) [5 marks]

See [[self-organising-maps]] for full detail.

**Unsupervised learning** means training on data that has **no labels** or **no target outputs**. The algorithm receives only input vectors $\mathbf{x}$ and must discover structure, patterns, clusters, or relationships in the data on its own — without being told what the correct answer is for any input.

**Supervised learning** uses a labelled training set: each input $\mathbf{x}$ is paired with a target output $t$. The learning algorithm adjusts its parameters to minimise the error between its predictions and the provided targets. Examples: classification (predict class label), regression (predict a continuous value).

**Comparison:**

| Aspect | Supervised learning | Unsupervised learning |
|--------|--------------------|-----------------------|
| Training data | Inputs + labelled targets | Inputs only (no targets) |
| Learning signal | Error between prediction and target | Internal structure/distances |
| Goal | Learn a mapping $\mathbf{x} \to t$ | Discover regularities in $\mathbf{x}$ |
| Examples | Classification, regression | Clustering, dimensionality reduction |
| SOM? | No | Yes |

In the context of **SOMs**: the network receives input vectors and organises itself so that similar inputs activate nearby neurons on the map — it learns the topology of the input space without ever being told which class or category an input belongs to. There is no target output and no error signal from an external teacher.

#### Part (c)(ii) [7 marks]

See [[neural-networks]] for background.

**Perceptron diagram:**

The single-neuron perceptron has:
- Two inputs: $x_1$ and $x_2$
- A bias input (constant 1): allows the decision boundary to be offset from the origin
- Weights $w_1$, $w_2$, and bias weight $w_0$
- A weighted sum: $a = w_0 \cdot 1 + w_1 \cdot x_1 + w_2 \cdot x_2$
- An activation function producing output $y$

```
         w1
x1 ------\
          \
  1 -----> [Σ] --> activation f(a) --> y
  (bias)  /
x2 ------/
         w2
```

**Activation function** — step function (Heaviside):
$$y = f(a) = \begin{cases} 1 & \text{if } a \geq \theta \\ 0 & \text{if } a < \theta \end{cases}$$

where $\theta$ is a threshold. With a bias weight, we set $\theta = 0$ and absorb the threshold into the bias:

$$a = w_0 + w_1 x_1 + w_2 x_2$$

**Finding weights** using the decision boundary $x_2 = x_1 + 1$, or equivalently $x_1 - x_2 + 1 = 0$:

The perceptron fires ($y=1$) when $a \geq 0$, i.e.:
$$w_0 + w_1 x_1 + w_2 x_2 \geq 0$$

Matching with $x_1 - x_2 + 1 \geq 0$ (class 1 region, where $x_2 < x_1 + 1$):

Choose:
$$w_0 = 1, \quad w_1 = 1, \quad w_2 = -1$$

(These give $a = 1 + x_1 - x_2$. Fire when $a \geq 0$, i.e. when $x_2 \leq x_1 + 1$.)
