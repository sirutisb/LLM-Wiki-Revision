#### Part (a) [8 marks]

See [[pso]], [[flocking-boids]].

**PSO Pseudocode:**

```
INPUT:
  N           = swarm size                                (e.g. 20–50)
  c_1, c_2    = cognitive and social acceleration coeffs (e.g. 2.0 each)
  v_max       = velocity clamp                            (e.g. 10–20% of search range)
  T_max       = maximum iterations                        (e.g. 100–1000)
  D           = dimensionality of the problem
  fitness f() = objective function

1. INITIALISE
   For each particle i = 1..N:
       Initialise position x_i ~ U(x_min, x_max)^D
       Initialise velocity v_i ~ U(-v_max, v_max)^D
       Set personal best:  pbest_i = x_i
       Evaluate fitness f(x_i)
   Set global best:        gbest = argmax_i f(pbest_i)

2. MAIN LOOP
   For t = 1..T_max:
       For each particle i = 1..N:
           For each dimension j = 1..D:
               z_1, z_2 ~ U(0,1)
               v_ij = v_ij + c_1 * z_1 * (p_ij - x_ij)
                            + c_2 * z_2 * (g_j  - x_ij)
               Clamp v_ij to [-v_max, v_max]
               x_ij = x_ij + v_ij
           Evaluate f(x_i)
           If f(x_i) better than f(pbest_i):
               pbest_i = x_i
           If f(x_i) better than f(gbest):
               gbest = x_i

3. TERMINATE when t = T_max, an acceptable solution is found,
   no improvement for N iterations, or swarm radius ≈ 0.

4. RETURN gbest
```

**Velocity update equation:**
$$v_{ij}(t+1) = v_{ij}(t) + c_1 z_1 (p_{ij} - x_{ij}(t)) + c_2 z_2 (g_j - x_{ij}(t))$$

**Position update:**
$$x_{ij}(t+1) = x_{ij}(t) + v_{ij}(t+1)$$

**Feasible parameter settings:**

| Parameter | Symbol | Typical value | Role |
|-----------|--------|---------------|------|
| Swarm size | $N$ | 20–50 | Larger → more diversity, more expensive per iteration |
| Cognitive coefficient | $c_1$ | $\approx 2.0$ | Pull toward particle's personal best |
| Social coefficient | $c_2$ | $\approx 2.0$ | Pull toward swarm's global best |
| Inertia weight (optional) | $w$ | $\approx 0.7$ | Coefficient on previous velocity if used |
| Random terms | $z_1, z_2$ | $\sim U(0,1)$ | Independent random draws each iteration |
| Velocity clamp | $v_{\max}$ | ~10–20% of range | Stops explosive particle movement |
| Max iterations | $T_{\max}$ | 100–1000 | Computational budget |

Three components of the velocity update: **inertia** (previous velocity), **cognitive** (pull toward pbest), and **social** (pull toward gbest). The random factors $z_1, z_2$ prevent premature convergence.

#### Part (b) [5 marks]

See [[fitness-landscapes]], [[metaheuristics-critique]].

**Exhaustive search** systematically enumerates and evaluates every candidate solution in the search space and returns the best.

**Benefits:**

1. **Guaranteed optimality:** Because every solution is checked, exhaustive search is guaranteed to find the global optimum. There is no risk of getting trapped in a local optimum.
2. **Simple to implement:** No parameter tuning, no stochastic operators — just iterate over all solutions.
3. **Deterministic and reproducible:** No randomness, so results are identical on every run; this is useful for verification and debugging.
4. **No assumptions about the landscape:** Works equally well on smooth, rugged, deceptive, or multimodal landscapes — the search procedure does not depend on landscape structure.

**Drawbacks:**

1. **Computationally infeasible for large problems:** The size of the search space grows combinatorially. For example, the TSP with $n$ cities has $(n-1)!/2$ tours — even $n=20$ gives $\approx 6 \times 10^{16}$ tours. NP-hard problems are out of reach.
2. **No use of problem structure:** Even when good solutions cluster in identifiable regions, exhaustive search wastes effort enumerating the entire infeasible/poor regions.
3. **Not adaptive:** Cannot exploit fitness information learnt along the way — every solution is evaluated regardless of past results.
4. **Continuous problems are impossible:** For continuous decision variables, the search space is infinite; exhaustive enumeration requires discretisation, which itself sacrifices optimality.

**Conclusion:** Exhaustive search is only practical for very small or low-dimensional problems. Nature-inspired methods trade the guarantee of optimality for the ability to find very good solutions in tractable time on problems where exhaustive search is impossible.

#### Part (c)(i) [5 marks] — Direct vs Indirect Representation

See [[representations]], [[sources/lecture05-encodings]].

**Direct representation:** Each gene of the chromosome directly encodes a **problem variable**. The chromosome literally is the solution (or trivially maps to it), so decoding is a simple read-off.

- *Example — Exam timetabling (direct):* Chromosome = `[4, 2, 1, 4, 3, 1, 2, 5, 3, 2]`, where gene $i$ specifies the time slot assigned to exam $i$ directly. The chromosome corresponds exactly to a candidate timetable.
- Other direct encodings: pipe diameters in a water network (k-ary), antenna wire coordinates (real vector), neural network weights (real vector), TSP tour (permutation).

**Indirect representation:** Each gene encodes a **parameter to a constructive heuristic**, not a solution value. A **decoder** uses these heuristic parameters together with domain knowledge to build the solution.

- *Example — Exam timetabling (indirect):* Chromosome = `[3, 1, 2, 4, 1, 3, 2, 1, 4, 2]`, where gene $i$ says "use the $k$th **clash-free** slot for exam $i$". The decoder iterates through exams, and for each one finds the $k$th slot in which no already-placed clashing exam is sitting. The chromosome no longer directly specifies the slots, but the decoded timetable is guaranteed clash-free.
- Other indirect encodings: priority rules for job scheduling (gene = priority of each job, decoder builds the schedule); seed orderings for greedy heuristics.

**Key difference:** the direct encoding represents the *what*; the indirect encoding represents the *how* — a recipe the decoder follows to build the solution.

---

#### Part (c)(ii) [4 marks]

See [[representations]].

**Advantage — Reduced/feasible search space:**
The decoder typically only produces **valid** (feasible) solutions. In the timetabling example, the indirect encoding cannot represent a clash — every chromosome decodes to a clash-free timetable. The EA therefore spends no effort exploring large infeasible regions of the direct search space, and the fitness function does not need a penalty term for invalid solutions. This is an advantage because it focuses the search budget on solutions that matter and removes the need for awkward penalty engineering.

**Disadvantage — Rugged / discontinuous fitness landscape:**
Because the decoder applies sequential, conditional logic (e.g. "find the next clash-free slot"), a tiny change to one gene can cause cascading large changes in the decoded solution. Solutions whose genotypes are similar may decode to very different phenotypes, so the fitness landscape becomes **rugged and high in epistasis**. This is a disadvantage because mutation no longer makes small smooth changes to the solution, so the EA's local-search behaviour is disrupted. Additional disadvantages: indirect encodings require **domain knowledge** to design the decoder, and decoding is **slower** than reading a direct chromosome (each fitness evaluation runs the decoder).

#### Part (d) — ACO symbols

See [[ant-colony-optimization]].

**(i) $\rho$ — Pheromone evaporation rate [2 marks]**

$\rho \in [0, 1]$ is the **evaporation rate** in the pheromone update equation:
$$\tau_{ij}(t+1) = (1-\rho)\,\tau_{ij}(t) + \sum_k \Delta\tau_{ij}^k$$

At each iteration, every pheromone trail is reduced by a fraction $\rho$. Its purpose is **forgetting**: it prevents pheromone from accumulating irreversibly on early-discovered (possibly mediocre) paths, ensuring exploration continues. Low $\rho$ → trails persist; algorithm exploits aggressively but risks premature convergence. High $\rho$ → trails decay rapidly; encourages exploration but may lose convergence pressure.

**(ii) $\alpha$ — Pheromone weight (exponent) [2 marks]**

$\alpha$ is the exponent on the pheromone term in the transition rule:
$$P_{ij}^k \propto [\tau_{ij}]^\alpha \cdot [\eta_{ij}]^\beta$$

$\alpha$ controls **how strongly the ants follow accumulated pheromone evidence**. When $\alpha = 0$, ants ignore pheromone entirely (purely heuristic, like a greedy randomised search). Large $\alpha$ makes the ants strongly favour edges with high pheromone — strong exploitation of learnt experience, but a risk of locking onto a suboptimal trail.

**(iii) $\beta$ — Heuristic weight (exponent) [2 marks]**

$\beta$ is the exponent on the heuristic term $\eta_{ij}$ (e.g. $1/d_{ij}$ for TSP) in the transition rule. It controls **how strongly the ants are biased by problem-specific domain knowledge**. When $\beta = 0$, ants use only pheromone (no greedy preference for short edges in TSP). Large $\beta$ makes the algorithm increasingly greedy — almost always picks the locally best edge — which speeds up convergence on simple landscapes but may miss globally better paths whose edges are individually less attractive.

#### Part (e) — Cellular automata terms

See [[cellular-automata]], [[sources/lecture20-alife]].

**(i) Localism [2 marks]**

**Localism** is the property that each cell's next state depends **only on its current state and the states of its immediate neighbours**, not on any global information about the grid. Every rule is defined over a small local neighbourhood (e.g. von Neumann: cell + 4 orthogonal neighbours; Moore: cell + 8 surrounding cells).

*Example:* In Conway's Game of Life, a cell decides whether to live or die based only on its 8 Moore neighbours; it has no knowledge of cells two steps away, of the global grid pattern, or of any centralised state. Complex global behaviour such as gliders or spaceships nevertheless **emerges** from these purely local rules.

**(ii) Parallelism [2 marks]**

**Parallelism** is the property that **all cells update their state simultaneously** at each time step. There is no sequential sweep through the grid — every cell looks at its neighbourhood at time $t$ and computes its new state at time $t+1$, and the new state of one cell does not influence the inputs used by another cell in the same step.

*Example:* In Game of Life, the entire grid is updated in lockstep — a cell about to die at this step still contributes "live" to its neighbour's count. This corresponds to massively parallel computation, with the CA naturally suited to parallel hardware.

**(iii) Homogeneity [2 marks]**

**Homogeneity** is the property that **every cell in the grid uses the same update rule**. There are no specialist cells with different behaviour — all cells are identical. Any heterogeneity in the resulting pattern arises from differences in the **initial states and neighbourhood configurations**, not from variation in the rules themselves.

*Example:* In Game of Life, every cell follows the same birth/survival/death rules. Yet emerging patterns can be highly varied — gliders moving through empty space, oscillators flipping in place, still lifes — because they sit in different local configurations on top of the homogeneous rule set.

Together, **localism + parallelism + homogeneity** explain why CAs are a canonical example of *emergence*: complex global behaviour from simple, identical, locally applied rules.

#### Part (f) — Perceptron learning [6 marks]

See [[neural-networks]].

A perceptron computes a weighted sum and applies a threshold/step activation:
$$\hat{y} = \varphi\!\left(\sum_{i=1}^{n} w_i x_i + b\right), \qquad \varphi(a) = \begin{cases} 1 & a \geq 0 \\ 0 & a < 0 \end{cases}$$

After computing $\hat{y}$ on a training pattern with target $y$, the **error** is $e = y - \hat{y}$. All three rules adjust the weights to drive this error toward zero.

**(i) The standard perceptron learning rule [2 marks]**

For each input pattern $(\mathbf{x}, y)$, the weight update is proportional to the error and the input:

$$w_i^{\text{new}} = w_i^{\text{old}} + (y - \hat{y}) \cdot x_i$$
$$b^{\text{new}} = b^{\text{old}} + (y - \hat{y})$$

- If the perceptron classifies correctly ($\hat{y} = y$): the error is zero, no update happens.
- If the perceptron under-fires (target 1, predicted 0): error $= +1$, weights are increased for the active inputs (those with $x_i = 1$).
- If the perceptron over-fires (target 0, predicted 1): error $= -1$, weights are decreased for the active inputs.

This is the original Rosenblatt rule (1958). It converges for any **linearly separable** problem but produces correction magnitudes that can be unstably large for some inputs.

**(ii) Perceptron learning rule with a learning rate [2 marks]**

A scalar **learning rate** $\eta \in (0, 1]$ is added to scale every update:

$$w_i^{\text{new}} = w_i^{\text{old}} + \eta \, (y - \hat{y}) \cdot x_i$$
$$b^{\text{new}} = b^{\text{old}} + \eta \, (y - \hat{y})$$

- Small $\eta$ (e.g. 0.1) → slow but stable convergence; the weights change gradually each pattern, averaging over presentations.
- Large $\eta$ (e.g. 1.0) reproduces the standard rule.
- Too-large $\eta$ → overshooting and oscillation around the correct decision boundary.

The learning rate gives explicit control over the step size in weight space and is essential when the magnitudes of inputs or errors are uneven.

**(iii) Widrow-Hoff (delta) learning rule [2 marks]**

The Widrow-Hoff (also known as the **delta rule** or LMS — Least Mean Squares) rule modifies (ii) by computing the error on the **net input $a$** (the linear activation, before thresholding), rather than the thresholded output. Equivalently, the error is the *real-valued* difference between target and pre-activation:

$$w_i^{\text{new}} = w_i^{\text{old}} + \eta \, (y - a) \cdot x_i$$
$$b^{\text{new}} = b^{\text{old}} + \eta \, (y - a)$$

where $a = \sum_i w_i x_i + b$ is the net input (pre-activation).

This is a **gradient descent** on the mean-squared error $E = \tfrac{1}{2}(y - a)^2$:
$$\frac{\partial E}{\partial w_i} = -(y - a)\,x_i \quad \Rightarrow \quad \Delta w_i = -\eta \frac{\partial E}{\partial w_i} = \eta(y - a) x_i$$

**Key advantages over the standard perceptron rule:**
- Defined for continuous-valued outputs (no need for a binary threshold), so it works for **regression** as well as classification.
- Continues to make weight changes even on correctly classified patterns whose net input is barely above/below zero, **widening the margin** and improving robustness.
- It is the basis for the generalised **delta rule** used in **backpropagation** for multilayer perceptrons.

#### Part (a) [15 marks] — Sejnowski & Rosenberg (1987): NETtalk

See [[neural-networks]].

> **Note:** Sejnowski & Rosenberg's NETtalk paper is not part of the current (2025/26) syllabus reading list — the contemporary equivalent paper-discussion questions tend to focus on other applications (e.g. AntNet, F1 GA). The following answer is reconstructed from standard NIC literature.

**Problem and motivation:**
NETtalk addresses the problem of **text-to-speech conversion** — given a string of English letters (a word or sentence), produce the correct sequence of phonemes (sounds) and stress patterns. The standard approach at the time was DECtalk, a hand-engineered symbolic system containing thousands of pronunciation rules and exception entries written by linguists. Sejnowski & Rosenberg asked whether a neural network could **learn** to perform the same mapping from training examples instead of being programmed.

**Network architecture (representation):**

NETtalk is a feed-forward **multilayer perceptron** with three layers:

| Layer | Units | Role |
|-------|-------|------|
| Input | 7 letter positions × 29 units each (letters A–Z plus 3 punctuation/space) = **203 units** | A sliding window of 7 letters centred on the letter currently being pronounced |
| Hidden | typically **80 units** | Learned distributed representation |
| Output | **26 units** | 21 articulatory phonological features (e.g. voiced/unvoiced, vowel height, place of articulation) + 5 stress/syllable-boundary features |

**Local (one-hot) input encoding:** for each of the 7 positions, exactly one of the 29 units is active, indicating which letter is in that position. The window slides one letter at a time across the text; the network's output is the phoneme + stress to be produced for the centre letter, with the 6 surrounding letters providing context.

**Distributed output encoding:** rather than a single output unit per phoneme, the network outputs articulatory features. A phoneme is selected by computing the output vector and finding the phoneme whose feature pattern is closest.

**Activations and training:**
- Sigmoid activation functions on all hidden and output units.
- Trained by **backpropagation** using a corpus of ~1000 words of running text, with each input letter paired with its correct phoneme/stress feature vector. Some experiments used a phonetic dictionary (isolated words) as training data.

$$w_{ij}^{(t+1)} = w_{ij}^{(t)} + \eta\,\delta_{pj}\,o_{pj}$$

**Experimental findings:**

1. **Rapid acquisition:** Within a few passes through the training corpus, the network's output was **intelligible** ("babbling" sounds quickly became recognisable phonemes). The progression mimicked the developmental stages observed in children learning to read aloud.
2. **Performance:** NETtalk achieved roughly **90% accuracy on the training set** and **~78% on held-out (novel) text**, showing genuine generalisation rather than memorisation. Performance was competitive with the hand-engineered DECtalk system despite NETtalk being learned from scratch.
3. **Generalisation:** The network correctly pronounced words it had never seen, including made-up but plausible words — evidence that it had learnt **regularities** of English pronunciation (e.g. the rules for hard/soft *c*, silent *e* lengthening) rather than memorising the dictionary.
4. **Distributed representation:** Analysing the hidden-unit activations revealed that **clusters of hidden units encoded phonological features** (vowel/consonant, voiced/unvoiced) — these features were not hand-designed but **emerged** during training. This was held up as evidence for the **connectionist** view that high-level structure can arise spontaneously in a neural network.
5. **Graceful degradation:** Damaging the network (zeroing some hidden units or weights) caused **gradual** performance loss rather than catastrophic failure — analogous to brain-injury studies of dyslexia. Re-training recovered performance.
6. **Cognitive-science significance:** NETtalk became one of the iconic demonstrations of connectionism in the 1980s, showing that a simple connectionist model could learn a task previously solved only by elaborate symbolic rule systems, and providing a candidate model for human reading and language development.

**Why it is significant for NIC:** NETtalk demonstrates **learning from examples** (rather than programming), **graceful degradation**, **distributed internal representations**, and **generalisation** — properties contrasting strongly with symbolic AI and motivating the connectionist approach used throughout this module.

#### Part (b)(i) [5 marks] — Unsupervised vs supervised learning

See [[self-organising-maps]], [[sources/lecture19-som]].

In machine learning the word **unsupervised** refers to learning from training data in which **no target outputs / labels are provided**. The learner sees only the inputs $\mathbf{x}_1, \ldots, \mathbf{x}_N$ and must discover regularities, structure, or groupings in them on its own.

By contrast, **supervised** learning is given training data as input–target pairs $(\mathbf{x}_i, t_i)$. The learner's objective is to find a function $f$ such that $f(\mathbf{x}_i) \approx t_i$ — i.e. it tries to mimic the labelled mapping.

| Aspect | Supervised | Unsupervised |
|--------|-----------|--------------|
| Training data | Input + target pairs $(\mathbf{x}, t)$ | Inputs only $\mathbf{x}$ |
| Objective | Approximate the labelled mapping | Discover structure / groupings / topology |
| Typical tasks | Classification, regression | Clustering, dimensionality reduction, density modelling |
| Error signal | Target − prediction | None — implicit from data statistics |
| Example algorithm | MLP trained by backpropagation | SOM, k-means, autoencoder |

**For a SOM specifically:** the inputs are presented one at a time; the network adjusts its internal weights so that *similar* inputs end up activating *nearby* neurons on the 2-D map. There is no teacher pointing the SOM at correct map positions — the organisation emerges from competition between neurons.

#### Part (b)(ii) [5 marks] — SOM learning equations

See [[self-organising-maps]].

A SOM consists of a 2-D grid of neurons, where each neuron $i$ has a **weight vector** $\mathbf{w}_i \in \mathbb{R}^d$ of the same dimension as the input data. For each input $\mathbf{x}$ presented at time $t$, learning proceeds in three steps:

**1. Competition — find the Best Matching Unit (BMU):**

The BMU is the neuron whose weight vector is closest (smallest Euclidean distance) to the input:

$$c = \arg\min_i \|\mathbf{x}(t) - \mathbf{w}_i(t)\|$$

**2. Cooperation — neighbourhood function:**

Define a neighbourhood kernel $h_{ci}(t)$ around the BMU $c$, typically a Gaussian centred on the BMU on the 2-D map:

$$h_{ci}(t) = \exp\!\left(-\frac{\|\mathbf{r}_c - \mathbf{r}_i\|^2}{2\sigma(t)^2}\right)$$

where $\mathbf{r}_c, \mathbf{r}_i$ are the positions of the BMU and neuron $i$ on the map grid, and $\sigma(t)$ is the neighbourhood radius at time $t$. Neurons close to the BMU on the map get a value near 1; far-away neurons get values near 0.

**3. Adaptation — weight update:**

Every neuron's weight vector is moved a small step toward the input, scaled by the learning rate $\eta(t)$ and the neighbourhood value $h_{ci}(t)$:

$$\mathbf{w}_i(t+1) = \mathbf{w}_i(t) + \eta(t)\,h_{ci}(t)\,(\mathbf{x}(t) - \mathbf{w}_i(t))$$

**Effect on the map:**
- The BMU's weights are pulled strongly toward $\mathbf{x}$ — that neuron becomes even more tuned to inputs like $\mathbf{x}$.
- Neurons in the neighbourhood are pulled less strongly in the same direction → topologically nearby neurons become tuned to similar inputs.
- After many presentations, **similar inputs activate nearby neurons** — the map preserves the topology of the input distribution.

#### Part (b)(iii) [5 marks] — Variables that change over time

See [[self-organising-maps]].

Two key variables decrease (typically exponentially or linearly) over the course of training:

**1. Learning rate $\eta(t)$:**

$\eta(t)$ scales the magnitude of every weight update. It begins large (e.g. $\eta(0) \approx 0.1$–$0.5$) and decays toward 0:

$$\eta(t) = \eta(0)\,\exp(-t/\tau_\eta) \quad \text{or} \quad \eta(t) = \eta(0)(1 - t/T_{\max})$$

- **Early in training**: large $\eta$ causes large weight changes — the map undergoes rapid global reorganisation, with weight vectors moving substantial distances to find their place in the input distribution.
- **Late in training**: small $\eta$ causes tiny adjustments — the map performs fine-tuning, settling into a precise topological organisation without disrupting what has already been learnt.
- *Effect on learning*: gives the **ordering / convergence** distinction. Without decay, the map would never settle (perpetual disruption); without an initial high $\eta$, the map would never globally re-organise (it would only fine-tune random initial positions).

**2. Neighbourhood radius $\sigma(t)$:**

The Gaussian width $\sigma$ controls how many neurons around the BMU are updated. It begins large (often spanning most of the map) and shrinks toward 0:

$$\sigma(t) = \sigma(0)\,\exp(-t/\tau_\sigma)$$

- **Early in training**: large $\sigma$ → almost every neuron is updated toward the input. This produces **coarse global ordering** — large regions of the map align with the broad structure of the input space.
- **Late in training**: small $\sigma$ → only the BMU and immediate neighbours update. This produces **local fine-tuning** — each neuron specialises in its own input region without disturbing neighbours.
- *Effect on learning*: allows the **topology** to first form globally and then be refined locally. This two-phase behaviour (ordering followed by convergence) is essential for the SOM to produce a meaningful topology-preserving map.

**Joint effect:** Both decays together implement **simulated annealing**-like behaviour in the SOM: large initial perturbations rapidly explore configurations, while shrinking updates settle into a stable, topology-preserving organisation. If either variable were kept constant the SOM would either fail to organise (always coarse) or fail to settle (constant disruption).

#### Part (a) [15 marks] — Koza's Five Preparatory Steps

See [[genetic-programming]], [[sources/lecture06-gp]].

Before running a GP system one must specify five things (Koza, 1992). For the **mpg regression problem** (5 input features → 1 mpg value, 100 labelled training cars):

**1. Terminal set $T$ — the leaves (variables and constants):**

The terminals are the inputs the program can read and any constants it may use. For this problem:

$$T = \{\text{weight}, \text{seats}, \text{gears}, \text{engine\_size}, \text{cd}, \mathcal{R}\}$$

where $\mathcal{R}$ is a generator of **ephemeral random constants** (real-valued numbers sampled when a node is first created, e.g. from a uniform $[-10, 10]$ distribution). Constants are essential for symbolic regression because the discovered formula will almost certainly need numeric coefficients (e.g. $-2.5 \cdot \text{weight} + 30$).

**2. Function set $F$ — the internal nodes (operators):**

For a regression problem, arithmetic operators are essential. For safety we use **protected division** (returns 1 if divisor = 0) to ensure every tree evaluates:

$$F = \{+, -, \times, \div_{\text{protected}}, \sqrt{\cdot}, \log, \exp\}$$

A minimal set $\{+, -, \times, \div_{\text{protected}}\}$ is also reasonable and is closer to standard symbolic regression practice. One might exclude `exp/log` if they cause numerical instability. **Closure** (every function must accept any subtree as input and return a numeric value) must hold across all combinations — this is why protected division and bounded log/exp are used.

**3. Fitness measure:**

Each candidate program is run on every training car (input = the 5 features; output = predicted mpg). Compare against the true mpg and aggregate the error. A standard choice is **sum of absolute errors** or **mean squared error** across the 100 training cars:

$$\text{fitness}(p) = \frac{1}{N} \sum_{i=1}^{100} \big(\text{mpg}_i - p(\text{car}_i)\big)^2$$

This is **minimised**. (Equivalently fitness = $-$ MSE if the EA expects a maximisation objective.) Optionally a **parsimony penalty** can be added (see part (c)) to discourage large trees. Cross-validation across the 100 cars should be used to monitor generalisation.

**4. Parameters:**

| Parameter | Suggested value |
|-----------|-----------------|
| Population size | 500–1000 |
| Maximum initial tree depth | 4–6 (used during initialisation, e.g. Koza's ramped half-and-half) |
| Maximum allowed depth | 17 (Koza's default) — caps bloat |
| Crossover rate | 0.85–0.9 |
| Mutation rate | 0.05–0.1 |
| Selection method | Tournament selection, $t = 5$–7 |
| Elitism | Retain best 1 individual per generation |
| Generations | 50–200 |

**5. Termination criterion:**

End the run when **any** of:
- A program is found with fitness below a target error (e.g. mean absolute mpg error < 1.0).
- The best fitness has not improved for $k$ generations (e.g. 25).
- The maximum number of generations is reached.

After termination, return the best-of-run program, validate it on a held-out test set, and check it makes physical sense (e.g. mpg is decreasing in weight and increasing in aerodynamic efficiency).

**Summary table:**

| Step | Choice for this problem |
|------|------------------------|
| Terminal set $T$ | {weight, seats, gears, engine_size, cd, ephemeral constants} |
| Function set $F$ | {+, −, ×, ÷_protected, √, log, exp} |
| Fitness measure | Mean squared (or absolute) error over 100 training cars |
| Parameters | Pop 500–1000; tournament $t=5$; crossover 0.9; mutation 0.05; max depth 17 |
| Termination | Error threshold OR plateau of $k$ generations OR max generations |

#### Part (c) [6 marks] — Bloat

See [[genetic-programming]].

**(i) Behaviour resulting from bloat [3 marks]**

Bloat is the tendency for GP individuals to grow in size (number of nodes / depth) over generations **without a corresponding improvement in fitness**. Observed behaviours:

1. **Stagnant fitness, exploding size:** A graph of best-of-generation fitness becomes essentially flat while a graph of average tree size keeps rising. The population is "running on the spot" but consuming more memory.
2. **Slower evaluation per generation:** Each fitness evaluation requires walking the tree; doubling the average tree size roughly doubles evaluation time. Generations get longer and longer to complete; total run time inflates.
3. **Introns / dead code:** Most of the new nodes are **semantically inert** — e.g. `(× X 1)`, `(+ subtree 0)`, `(− X X)`, or branches whose result is multiplied by 0. They do not change the program's output but provide "padding" that protects useful code from being disrupted by crossover.
4. **Reduced search progress:** Crossover and mutation become less effective because most of the chromosome is dead code; useful changes happen rarely.
5. **Loss of interpretability:** The final program, the point of symbolic regression in this problem, becomes unreadable — buried inside an enormous tree of identity operations — even if its predictions are accurate.
6. **Memory and stack-overflow risk:** Without a depth cap, trees can grow past system limits and crash the evaluator.

**(ii) Measures to reduce bloat [3 marks]**

A combination of the following is standard:

1. **Hard depth (or size) cap:** Reject any child whose depth exceeds a fixed maximum (Koza's classic value is depth 17 for trees, or a node-count cap). This is the simplest, most common safeguard.
2. **Parsimony pressure (size penalty in fitness):** Modify the fitness function to penalise size:
   $$\text{fitness}'(p) = \text{error}(p) + \lambda \cdot \text{size}(p)$$
   The size penalty $\lambda$ trades accuracy against compactness. Equivalent multi-objective formulations evolve fitness and size jointly (e.g. NSGA-II on (error, size)).
3. **Tournament selection by size as tie-breaker (lexicographic parsimony):** If two individuals tie on fitness, pick the smaller; this gradually selects against bloat without distorting the fitness landscape.
4. **Tarpeian method (Poli):** With probability $p_T$, **automatically assign the worst possible fitness** to any program whose size exceeds the current population's median. This actively kills oversized individuals while leaving the rest of the search unaffected.
5. **Size-fair / homologous crossover:** Modify crossover so that the subtree being inserted has roughly the same size as the subtree being removed, preventing systematic growth.
6. **Subtree mutation that prefers replacing large subtrees with small ones (shrink mutation):** Periodically replace a random subtree with a single terminal — this directly reduces tree size.
7. **Editing / simplification:** Periodically apply algebraic simplifications (e.g. $x \cdot 1 \to x$, $x + 0 \to x$) to remove introns. Useful for interpretability but adds computational cost.

In practice, a **depth cap + parsimony pressure** is the default combination, with size-fair crossover added for difficult problems.

## Question 4 — Population-Based Search Comparison [30 marks]

*Evolutionary algorithms (EA), ant colony optimisation (ACO) and particle swarm optimisation (PSO) are all examples of population based search.*

**(a) Describe the advantages that population-based search brings to search and optimisation tasks over methods that use a single point strategy.**

**(5 marks)**

**(b) Describe the additional algorithmic steps that an EA must conduct due to the existence of a population.**

**(5 marks)**

**(c) Explain how communication between population members differs between ACO and PSO.**

**(5 marks)**

*These three algorithms have different methods of varying the extent to which they exploit or explore during search and optimisation. For each algorithm, describe the parameters that are modified to achieve different exploitation and exploration capability and explain how varying these alters the behaviour of the algorithm.*

**(15 marks)**

**(Total 30 marks)**

### Answer

#### Part (a) [5 marks] — Advantages of population-based search

See [[evolutionary-algorithms]], [[fitness-landscapes]], [[swarm-intelligence]].

A **single-point** search (hillclimbing, simulated annealing, tabu search) maintains and updates a single candidate solution; a **population-based** search (EA, PSO, ACO) maintains many simultaneously. The advantages are:

1. **Parallel exploration of multiple landscape basins:** Different population members can lie in different basins of attraction simultaneously, so the algorithm probes several regions of the search space at once. A single-point hillclimber can only ever explore one basin per run.
2. **Reduced risk of premature convergence to local optima:** Because the population maintains diversity, even when some members are trapped in a local optimum, others may still be exploring elsewhere. The population as a whole is far less likely to be permanently stuck than a single point that has reached a local peak.
3. **Recombination becomes possible:** Two known-good parents from different parts of the search space can be combined (crossover in EA, or implicitly via gbest in PSO and pheromone reinforcement in ACO) to form a child that is potentially better than either — a move that single-point algorithms cannot make.
4. **Implicit landscape sampling and statistical robustness:** A population provides a statistical sample of the landscape, allowing methods (rank-based selection, niching, archive maintenance) to estimate distributions of good solutions and concentrate search adaptively.
5. **Natural parallelism:** Fitness evaluations of population members are independent, so a population-based algorithm parallelises trivially across cores or machines — a single-point algorithm is fundamentally sequential.
6. **Robustness to noise:** With a population, the algorithm averages over many evaluations and is more robust to noisy fitness functions than a single-point method, which can be misled by one bad evaluation.

---

#### Part (b) [5 marks] — Additional algorithmic steps in an EA due to the population

See [[evolutionary-algorithms]], [[selection]].

Compared to a single-solution hillclimber, an EA must perform several extra population-management steps each generation:

1. **Initialisation of the entire population:** Generate (and evaluate) $N$ random candidate solutions at the start, rather than just one. This is an upfront cost in fitness evaluations proportional to population size.
2. **Selection of parents:** With multiple individuals available, the EA must choose *which* members of the population produce offspring. This requires a **selection mechanism** (tournament, rank-based, or roulette-wheel — see [[selection]]) that biases choices toward fitter individuals while preserving diversity. A hillclimber has no analogue — there is only one parent.
3. **Bookkeeping / maintaining a list of $N$ individuals:** The algorithm must store all $N$ chromosomes and their fitness values, not just a single current solution. Data structures (e.g. sorted by fitness for elitism) add overhead.
4. **Crossover (recombination) operator:** Because two parents are now available, EAs can apply crossover — combining genetic material from two individuals to make a child. This is a new operator that a single-point algorithm does not have, and which must be designed appropriately for the encoding (see [[crossover-mutation]]).
5. **Replacement strategy:** New offspring must be inserted into the population, displacing existing members. The algorithm must decide which: *generational* replacement (replace whole population, often with elitism to preserve best), *steady-state* (replace weakest), or other strategies. This is an additional design decision absent from single-point methods.
6. **Diversity maintenance:** Particularly with strong selection pressure, the population can collapse to identical clones. The EA must therefore monitor and (often) actively maintain diversity through niching, fitness sharing, restart strategies, or moderated selection pressure — none of which exist for single-solution methods.

#### Part (c) [5 marks] — Communication: ACO vs PSO

See [[ant-colony-optimization]], [[pso]], [[comparisons/aco-vs-pso]] (referenced), [[swarm-intelligence]].

| Aspect | ACO | PSO |
|--------|-----|-----|
| **Communication channel** | **Indirect**: via the shared pheromone matrix on the environment (the construction graph) | **Direct**: each particle reads the swarm's global best (gbest) and the topology of pbest information |
| **Mechanism** | **Stigmergy** — ants modify the environment by depositing pheromone; other ants respond to that modification | A shared/broadcast variable: gbest (or local-best in neighbourhood PSO) is read directly by every particle each step |
| **Persistence** | Cumulative and decaying: pheromone integrates contributions from many ants over many iterations, modulated by evaporation $\rho$ | Snapshot-based: gbest is overwritten whenever a better solution is found |
| **What is shared** | A landscape of edge desirabilities $\tau_{ij}$ over the whole construction graph | A single best position (gbest) plus each particle's personal memory (pbest) |
| **Timing** | Pheromone is updated at the end of each iteration after all ants have constructed tours | gbest can be updated within the iteration as soon as a particle's new position exceeds it |
| **Biological metaphor** | Real ants depositing trail pheromone (Deneubourg double-bridge experiment) | Bird flocks tracking a "cornfield vector" — the direction of the best food smell |

**Summary:** ACO communication is **indirect** (mediated by the environment, with no individual knowing what any specific other ant did), persistent, and integrates over time via the pheromone matrix. PSO communication is **direct** (each particle is informed of the swarm's current best position and steered toward it) and snapshot-based. ACO is therefore naturally suited to **discrete construction problems** where partial decisions can be reinforced, whereas PSO is suited to **continuous spaces** where the best-known location can be pursued geometrically by every particle.

---

#### Final part [15 marks] — Exploration/exploitation parameters for EA, ACO, PSO

See [[evolutionary-algorithms]], [[ant-colony-optimization]], [[pso]], [[selection]], [[crossover-mutation]].

Each of the three algorithms has its own parameters controlling the **exploration–exploitation balance**. The general principle is the same: increasing the bias toward currently best information accelerates convergence (exploitation) but risks premature convergence; reducing that bias and/or adding randomness preserves diversity (exploration) but slows convergence.

##### EA (5 marks)

| Parameter | Effect of *increasing* | Mechanism |
|-----------|------------------------|-----------|
| **Mutation rate** ($p_m$, e.g. 0.001–0.1) | Higher → more **exploration** (random perturbation, large jumps) | Larger fraction of offspring receive new genetic material; can disrupt good solutions if too high |
| **Mutation step size** ($\sigma$ for real-valued / Gaussian mutation) | Higher → larger jumps → more **exploration**; lower → fine-tuning → **exploitation** | Controls the radius of the local search around a parent |
| **Crossover rate** ($p_c$, e.g. 0.6–0.95) | Higher → more recombination of existing good material (a form of moderated exploitation by mixing parents); lower → more mutation-only offspring (more pure exploration) | Crossover exploits good partial solutions by recombining them |
| **Tournament size $t$** | Higher → stronger selection pressure → **exploitation** (population converges quickly on best individuals); lower → weaker pressure → more **exploration** | $t=1$ is random selection (pure exploration); $t = N$ is deterministic (pure exploitation of best) |
| **Population size $N$** | Higher → more **exploration** (more diverse coverage); lower → faster, more focused **exploitation** | Larger populations sample the landscape more broadly |
| **Elitism count** | Higher → more **exploitation** (top individuals always preserved); lower → more **exploration** (any individual can be lost) | Elitism guarantees the best are not destroyed by stochastic variation |

**Key levers:** mutation rate/step size control low-level exploration; tournament size $t$ controls selection pressure / how fast the population converges. The interplay between these two is the central design choice in an EA — too much pressure with too little mutation → premature convergence; too little pressure or too much mutation → no progress.

##### ACO (5 marks)

| Parameter | Effect of *increasing* | Mechanism |
|-----------|------------------------|-----------|
| **Pheromone weight $\alpha$** | Higher → ants more strongly follow pheromone trails → **exploitation** (reinforcement of established trails) | Pheromone term $[\tau_{ij}]^\alpha$ in transition rule becomes dominant |
| **Heuristic weight $\beta$** | Higher → ants more strongly follow domain heuristic (e.g. shorter edges in TSP) → greedy **exploitation** | Heuristic term $[\eta_{ij}]^\beta$ becomes dominant; behaviour approaches nearest-neighbour heuristic |
| **Evaporation rate $\rho$** | Higher → pheromone forgets faster → more **exploration** (old trails don't dominate); lower → trails persist → **exploitation** | $\tau_{ij}\leftarrow (1-\rho)\tau_{ij}$ in update rule |
| **Number of ants $m$** | More ants → broader sampling of the graph per iteration → **exploration**; fewer ants → faster convergence (less coverage) → **exploitation** | Each ant constructs an independent tour each iteration |
| **ACO variant (AS / Elitist AS / MMAS)** | Standard AS: all ants update (most exploration). Elitist: only top $n$ update. MMAS: only best ant updates AND $\tau \in [\tau_{\min}, \tau_{\max}]$ → strongest **exploitation** while bounding pheromone prevents stagnation | Restricts which ants reinforce trails |
| **$Q$ (deposit constant)** | Higher → bigger contributions per ant → bigger differences between trails after one update → faster **exploitation** | $\Delta\tau^k_{ij} = Q/L^k$ |

**Key levers:** $\alpha$ vs $\beta$ trade learnt experience against domain heuristic; $\rho$ is the **primary diversity mechanism** — without evaporation pheromone accumulates irreversibly on early discoveries (premature convergence). Tuning is typically: moderate $\alpha$ ($\approx 1$), $\beta$ slightly larger ($\approx 2$–5 in TSP), and $\rho \approx 0.1$–0.5 depending on problem.

##### PSO (5 marks)

| Parameter | Effect of *increasing* | Mechanism |
|-----------|------------------------|-----------|
| **Cognitive coefficient $c_1$** | Higher → particles pulled more strongly toward their **own** best position → more **exploration** (each particle stays near its own discovered region) | Cognitive term $c_1 z_1 (p_{ij} - x_{ij})$ in velocity update |
| **Social coefficient $c_2$** | Higher → particles pulled more strongly toward swarm's gbest → **exploitation** (swarm collapses onto current best) | Social term $c_2 z_2 (g_j - x_{ij})$ dominates |
| **Inertia weight $w$ (when included)** | Higher → particles keep their momentum → wide-ranging **exploration**; lower → particles decelerate → **exploitation** | $v_{ij}(t+1) = w \cdot v_{ij}(t) + \ldots$; many implementations decay $w$ from $\approx 0.9$ to $\approx 0.4$ |
| **Velocity clamp $v_{\max}$** | Higher → particles can take large steps → **exploration**; lower → small steps → **exploitation** | Caps the magnitude of each velocity update |
| **Neighbourhood size / topology** | **Global best (full neighbourhood)** → fast convergence → **exploitation**. **Local best (ring or other restricted neighbourhood)** → slower information propagation → more **exploration** | Determines whether the social term uses gbest or a local nbest |
| **Swarm size $N$** | Higher → more diverse initial coverage → **exploration**; smaller → faster, more focused → **exploitation** | Number of independent search points |

**Key levers:** the balance of $c_1$ vs $c_2$ controls individual vs collective behaviour — high $c_2$ collapses the swarm rapidly to gbest (premature convergence); high $c_1$ keeps particles dispersed near their personal bests. The random factors $z_1, z_2 \sim U(0,1)$ are essential stochastic perturbations that prevent deterministic premature convergence. A widely used rule is $c_1 = c_2 = 2.0$ with a decaying inertia weight $w$ to give a natural exploration-to-exploitation transition over the run.

**Common pattern across all three algorithms:**
- There is always a parameter that **controls how strongly current best information is followed** ($t$ for EA, $\alpha$ for ACO, $c_2$ for PSO).
- There is always a parameter that **injects fresh diversity** (mutation rate for EA, evaporation $\rho$ for ACO, $c_1$ + random terms for PSO).
- Balancing the two is the central tuning task; a decaying schedule (more exploration early, more exploitation late) often works well.
