#### Part (a) [8 marks]

See [[concepts/ant-colony-optimization]].

**Pseudocode (Basic Ant System on TSP):**

```
1. Initialise pheromone τ_ij on all edges (e.g. small uniform value)
   Set parameters α (pheromone weight), β (heuristic weight),
                  ρ (evaporation rate), Q (deposit constant),
                  m (number of ants)
2. Repeat until termination condition met:
   2.1 For each ant k = 1..m:
       a. Place ant on a random start node
       b. Initialise tour memory M_k = {start node}
       c. While not all nodes visited:
            - Compute transition probability to each allowed
              (unvisited) node j from current node i:
                P_ij^k = (τ_ij)^α (η_ij)^β  /  Σ_{l ∈ allowed} (τ_il)^α (η_il)^β
            - Sample next node j according to P_ij^k
            - Move ant to j; append j to M_k
       d. Compute tour length L^k
   2.2 EVAPORATE pheromone on every edge:
            τ_ij ← (1 − ρ) · τ_ij
   2.3 DEPOSIT pheromone according to tour quality:
            For each ant k and each edge (i,j) used by ant k:
                τ_ij ← τ_ij + Q / L^k
3. Return the best tour seen across all iterations
```

**Features supporting exploration, exploitation and convergence:**

| Feature | Role |
|---------|------|
| **Stochastic transition rule** (probabilistic, not deterministic) | **Exploration** — ants do not always pick the strongest pheromone edge, so new edges and tours are continually sampled. The random draw makes diverse tours possible even when pheromone is uneven. |
| **Heuristic factor $\eta_{ij}$** (e.g. $1/d_{ij}$) | **Exploration of good regions via domain knowledge** — biases search toward locally promising moves without locking onto any specific tour. |
| **Pheromone deposition $\Delta\tau \propto 1/L^k$** | **Exploitation** — better tours leave more pheromone, reinforcing their edges so subsequent ants are biased toward retracing what has been learned. |
| **Pheromone weight exponent $\alpha$** | Tunes the balance: higher $\alpha$ → stronger exploitation of learned trails; lower $\alpha$ → behaviour closer to a random heuristic walk (more exploration). |
| **Pheromone evaporation $(1-\rho)\tau_{ij}$** | **Diversity / negative feedback** — prevents old pheromone from accumulating indefinitely, allowing the colony to "forget" sub-optimal early choices and continue exploring. |
| **Positive feedback loop** (good edges accumulate pheromone faster than they evaporate) | **Convergence** — over many iterations one or a few edges dominate the pheromone matrix; the transition rule then almost always picks them, so all ants converge to (near-)identical tours. |

The interplay of positive feedback (deposition) and negative feedback (evaporation), modulated by the stochastic transition rule, is what gives ACO its characteristic exploration→exploitation→convergence trajectory.

#### Part (b) [6 marks]

See [[concepts/ant-colony-optimization]], [[sources/lecture08-aco-detail]].

A **construction graph** is the generalisation of the TSP graph that lets ACO be applied to problems whose solutions are not naturally cities-and-edges. The graph encodes the *construction* of a solution as a path that an ant walks through a set of decision nodes.

**Structure (labelled diagram):**

```
                       (Start)
                       /  |  \
                      /   |   \
                  τ,η  τ,η   τ,η
                    /     |     \
                 [A1]   [A2]   [A3]      ← Decision layer 1
                 / | \   / | \   / | \
              τ,η τ,η τ,η ...
                [B1]   [B2]   [B3]      ← Decision layer 2
                  ...
                [Z1]   [Z2]   [Z3]      ← Decision layer L
                       \  |  /
                       (End)            (optional)
```

**Labels:**
- **Nodes:** each node corresponds to *one decision* the ant must make (e.g. "assign job J to machine 1", "include feature F in the subset", "place item I in bin B").
- **Layers:** consecutive layers represent the sequence of decisions needed to construct a complete solution. An ant walks from `Start` through one node per layer down to the end.
- **Edges:** edges connect nodes that are compatible decisions; each carries a **pheromone level $\tau_{ij}$** (learned desirability) and a **heuristic $\eta_{ij}$** (problem-specific desirability, e.g. job urgency, item-bin fit).
- **Start / end nodes:** an explicit Start node ensures the very first decision also receives pheromone reinforcement; an End node can be added for open-path problems (e.g. scheduling — no return to start as in TSP).

**Why it is needed:**

1. **ACO is fundamentally a graph-walking algorithm.** Its transition rule and pheromone update both assume that solutions are constructed as sequences of edge traversals. A problem that is not naturally a graph (e.g. bin packing, feature selection, job scheduling) must be mapped onto one before ACO can be applied.
2. **Encodes the structure of valid solutions.** By laying out decisions as layers and only connecting compatible nodes, the construction graph guarantees that any complete walk corresponds to a feasible solution — invalid solutions are excluded by construction rather than by penalty.
3. **Provides edges for pheromone to live on.** Pheromone is associated with edges (decisions in context), not with isolated objects. Without a construction graph there is nothing for $\tau_{ij}$ to be indexed by.
4. **Enables problem-specific heuristics.** $\eta_{ij}$ can encode any domain knowledge (e.g. due-date proximity, item size, feature relevance) and is naturally embedded in the same edges.

**Examples:**

| Problem | Nodes | Layers | Heuristic $\eta$ |
|---------|-------|--------|-----------------|
| TSP | Cities | Position in tour | $1/d_{ij}$ |
| Job scheduling | Jobs | Position in schedule | $1/\text{deadline}$ |
| Bin packing | (Item, bin) pairs | Items in order | item-bin fit score |
| Feature selection | Features | Include / exclude decisions | feature relevance |

#### Part (c)(i) [2 marks]

See [[concepts/selection]].

**Tournament selection:**

1. Randomly sample $t$ individuals from the population (with or without replacement; usually without).
2. Compare their fitnesses.
3. Return the individual with the best fitness as the selected parent.

The procedure is repeated each time a parent is required. The parameter $t$ is the **tournament size** and controls selection pressure.

---

#### Part (c)(ii) [4 marks]

As $t$ increases, **selection pressure increases**.

**Why:**

- When $t = 1$ a single random individual is returned — selection probability is uniform across the population (no pressure at all).
- When $t = 2$ the best of two random individuals wins, so fitter individuals win more often than not — moderate pressure.
- As $t$ grows, the probability that the tournament contains one of the very best individuals approaches 1, and when it does the best is always returned. The fittest individual is therefore selected with near-certainty on every draw.
- At $t = \text{popsize}$ the tournament always contains the population's single best individual, so selection becomes deterministic.

The practical consequence is that increasing $t$ concentrates reproduction on a smaller elite, accelerates convergence, and reduces diversity — risking premature convergence on a local optimum. Decreasing $t$ flattens selection pressure and preserves diversity at the cost of slower convergence.

---

#### Part (c)(iii) [4 marks]

Tournament selection is preferred to roulette wheel selection for several reasons:

1. **Immune to superfit dominance.** In roulette wheel, $P(i) = f_i / \sum_k f_k$, so one individual with much higher fitness than the rest dominates the wheel and is selected almost every time — diversity collapses and the EA converges prematurely. Tournament selection only ever compares the $t$ individuals in a tournament, bounding any one individual's influence.
2. **Handles negative fitness and minimisation problems.** Roulette wheel requires strictly positive fitness values and is designed for maximisation; minimisation or negative fitness needs awkward transformations (e.g. $f_{\max} - f_i$). Tournament selection uses only *relative* fitness comparisons and works unchanged for any fitness scale or sign.
3. **Tunable selection pressure via a single parameter.** $t$ provides direct, intuitive control over the exploration–exploitation trade-off. Roulette wheel has no analogous knob without ad-hoc fitness-scaling tricks (sigma-scaling, ranking, etc.).
4. **Computationally efficient.** Tournament selection is $O(t)$ per parent and requires no global sorting or cumulative-fitness sums. Roulette wheel needs the sum of fitnesses over the whole population for every selection.

These advantages explain why tournament selection (typically $t = 2$ to $5$) is the modern default in EAs.

#### Part (d) [6 marks]

See [[concepts/flocking-boids]].

Craig Reynolds' Boids (1987) — created originally for realistic 3D computer animation, not optimisation — generates complex emergent flocking behaviour from just three simple local rules applied identically by every agent (boid) at every time step:

| Rule | Description | Effect |
|------|-------------|--------|
| **1. Separation** (short-range repulsion) | Steer to avoid crowding *local* flockmates: each boid moves away from any neighbour that comes too close within its perception radius. | Prevents collisions and maintains personal space inside the flock. |
| **2. Alignment** | Steer toward the *average heading* (direction of travel / velocity) of local flockmates. | Synchronises direction of motion; gives the flock its coherent forward sweep. |
| **3. Cohesion** | Steer toward the *average position* (centre of mass) of local flockmates. | Keeps the group together; isolated boids are pulled back into the flock. |

Each boid is **homogeneous** (no leaders), has only **local omni-directional perception** within a fixed neighbourhood radius, and responds **immediately** with no memory of past states. The complex behaviours — spontaneous polarisation, synchronised direction changes, flash expansion if startled, flock merging and splitting — all *emerge* from these three rules; none is explicitly programmed. (PSO is the optimisation algorithm derived from this insight: if the food source is the optimum, the same kind of velocity update converges the flock on it.)

#### Part (e)(i) [2 marks]

See [[concepts/swarm-intelligence]].

**Emergence** in the context of collectives refers to the phenomenon whereby complex, structured, often intelligent group-level behaviour *arises bottom-up* from many simple agents following local rules, despite the fact that no individual agent has the capability or knowledge to produce that behaviour on its own and no central controller orchestrates it. The whole exhibits properties that are more than the sum of its parts (e.g. an ant colony finds the shortest path even though no single ant can perceive or reason about the path).

---

#### Part (e)(ii) [2 marks]

See [[concepts/pso]], [[concepts/swarm-intelligence]].

PSO depends fundamentally on emergence. No individual particle has any view of where the global optimum lies; each particle merely follows a simple velocity rule combining its own personal best (cognitive) and the swarm's best-known position (social). The swarm-level ability to *locate good optima* in a continuous search space emerges from the interaction of many such simple updates. Without emergence, the algorithm would degenerate into many independent randomised hillclimbers and would lose its key strength — collective search and convergence on solutions that no individual particle could find alone.

#### Part (f)(i) [2 marks] — Perceptron vs MLP

See [[concepts/neural-networks]].

The (single-layer) **perceptron** (McCulloch–Pitts / Rosenblatt) is a single neuron with weighted inputs and a threshold activation function — no hidden layers. An **MLP** adds one or more hidden layers between input and output, with non-linear (sigmoid / ReLU) activations.

- **Architecture difference:** perceptron has only the input → output mapping; MLP has input → hidden → output, allowing layered non-linear composition.
- **Learning problem best suited to:** the perceptron can only learn **linearly separable** decision problems (e.g. logical AND, OR — but *not* XOR). MLPs are universal function approximators and so handle non-linear classification and regression (e.g. XOR, image classification, function approximation).

---

#### Part (f)(ii) [2 marks] — Self-Organising Map vs MLP

See [[concepts/self-organising-maps]].

A **Self-Organising Map (SOM / Kohonen network)** is a 2D grid of neurons, each carrying a weight vector of the same dimensionality as the input. There is **no layered feedforward structure**, no activation functions, no backpropagation; the "output" is the position of the Best Matching Unit on the grid.

- **Architecture difference:** SOM is a single competitive layer organised as a 2D lattice; MLP is a layered feedforward network with weighted sums and non-linear activations.
- **Learning problem best suited to:** SOMs perform **unsupervised** topology-preserving dimensionality reduction, clustering, and visualisation of unlabelled high-dimensional data. MLPs perform **supervised** classification or regression where labelled targets are available.

---

#### Part (f)(iii) [2 marks] — Recurrent neural network vs MLP

A **Recurrent Neural Network (RNN)** has **feedback connections**: outputs (or hidden states) at one time step are fed back as inputs at the next step, giving the network an internal memory of past inputs. An MLP is **strictly feedforward** — information flows in one direction only with no cycles, and each input is processed independently of any other input.

- **Architecture difference:** RNN has cycles / recurrent connections (the same weights are applied at every time step to combine current input with previous hidden state); MLP has no cycles.
- **Learning problem best suited to:** RNNs are suited to **sequential / temporal data** where each output depends on history — speech recognition, language modelling, time-series prediction, handwriting recognition. MLPs are suited to **static**, fixed-size inputs where temporal order does not matter (e.g. tabular classification).

### Answer

#### Part (a) [15 marks]

See [[concepts/genetic-algorithms]], [[concepts/evolutionary-algorithms]], [[concepts/representations]].

**Problem and motivation:**

Wloch and Bentley (2004) apply a Genetic Algorithm to optimise the setup of a Formula One racing car within a detailed race simulator. A real F1 setup involves many tightly interacting design parameters (suspension stiffness, wing angles, gear ratios, brake balance, tyre choice, fuel load, pit-stop strategy). The interactions are non-linear and the search space is high-dimensional, so manual tuning by engineers — though guided by experience — cannot explore the full space efficiently. The paper proposes the GA as an automated optimiser that can find non-obvious, race-circuit-specific setups.

**Representation:**

Each chromosome encodes the configurable parameters of the car as a fixed-length vector. The encoding is **mixed**: real-valued genes for continuous parameters (suspension, ride height, gear ratios, aero settings) and integer genes for discrete choices (tyre compound per stint, pit-stop lap, brake bias). This is a **direct encoding** — each gene corresponds straightforwardly to a tunable physical parameter.

**Fitness function:**

Each candidate setup is evaluated by running the full F1 race simulator using those parameter values and measuring the **race time** (total lap time across the race, including pit stops). Fitness = $-\text{race time}$ (or equivalently the GA minimises race time). The fitness evaluation is *expensive* — one simulator run per individual per generation — which is why a heuristic search like a GA is appropriate (it does not require gradients and amortises evaluation cost across the population).

**Algorithm (standard generational GA):**

1. Initialise a population of $N$ random parameter configurations within feasible ranges.
2. Evaluate each individual by running the race simulator and recording race time.
3. Selection: tournament or rank-based selection of parents.
4. Variation:
   - **Crossover** between parent vectors (uniform or multi-point on the parameter vector).
   - **Mutation:** Gaussian noise added to real-valued genes; random replacement (within range) for integer genes.
5. Replace the population with the offspring; retain the best individual by **elitism** to prevent loss of the best solution.
6. Repeat for a fixed number of generations or until improvement plateaus.
7. Return the best setup found.

**Experimental setup:**

The simulator is run on different circuits (each circuit has a different geometry / cornering profile, so the optimum setup differs). The GA is run multiple times with different random seeds. Comparisons are made against an engineer-tuned baseline setup.

**Experimental findings (briefly):**

- The GA consistently produced setups with **faster race times** than the manually tuned baseline across the simulated circuits.
- It discovered **non-intuitive trade-offs and circuit-specific configurations** — combinations of tyre choice, pit-stop timing, and aero/suspension settings that human engineers had not converged on.
- Different circuits required substantially different optimal setups, demonstrating that the GA correctly tailored solutions to circuit geometry rather than producing one-size-fits-all configurations.
- Results were robust across multiple runs of the GA, indicating the solutions were genuinely good and not artefacts of one stochastic run.

**Significance:** the paper demonstrates that EAs are practical for **expensive, mixed real-world engineering design problems** where fitness evaluation requires a full simulation and where the search space is high-dimensional and non-linear. It is one of the canonical Exeter case studies for "EA applied to engineering optimisation".

#### Part (b)(i)A — Permutation mutation [2 marks]

See [[concepts/crossover-mutation]], [[concepts/representations]].

**Swap mutation** (also called pairwise swap). With probability $p_{\text{mut}}$, pick two random positions $i$ and $j$ in the permutation and exchange the values at those positions. Other valid alternatives are **insertion** (remove an element and re-insert it at a different position) and **inversion** (reverse a randomly chosen sub-sequence). All three preserve the permutation property — every element appears exactly once — which **simple single-gene replacement would break** by producing duplicates.

Example: `(D,E,G,J,A,C,B,F,I,H)` → swap positions 2 and 5 → `(D,A,G,J,E,C,B,F,I,H)`.

---

#### Part (b)(i)B — K-ary mutation [2 marks]

**Single-gene random replacement** (the standard k-ary mutation). With probability $p_{\text{mut}}$, pick one gene at random and replace its current value with a different random value drawn uniformly from the legal range $\{0, 1, \ldots, k-1\}$. (For binary encoding, $k=2$, this reduces to **bit-flip mutation**.) A multi-gene variant applies the same procedure to $m$ randomly chosen genes per mutation event.

Example (k=10): `352872` → mutate position 2 → `312872`.

---

#### Part (b)(i)C — Continuous mutation [2 marks]

**Additive Gaussian mutation** (single-gene). With probability $p_{\text{mut}}$, pick one gene at random and add zero-mean Gaussian noise:
$$x_i \leftarrow x_i + \delta, \quad \delta \sim \mathcal{N}(0, \sigma^2)$$

The standard deviation $\sigma$ controls the step size: small $\sigma$ produces fine local refinement (exploitation), large $\sigma$ produces larger jumps (exploration). A **vector mutation** variant adds an independent Gaussian to every gene simultaneously, $\mathbf{x} \leftarrow \mathbf{x} + \boldsymbol{\delta}$ with $\boldsymbol{\delta} \in \mathbb{R}^L$.

---

#### Part (b)(ii) [5 marks]

See [[concepts/selection]], [[concepts/fitness-landscapes]].

**How to identify early (premature) convergence:**

1. **Best fitness plateaus early.** The best-of-generation fitness stops improving long before the planned generation budget is exhausted, and the final best fitness is poor (you have not solved the problem).
2. **Population diversity collapses.** The average pairwise distance between chromosomes (Hamming distance for discrete encodings, Euclidean for continuous) drops sharply, and the standard deviation of fitness values across the population approaches zero — every individual is essentially the same solution.
3. **Best and mean fitness converge.** A widening gap between best and mean fitness indicates healthy diversity; the gap collapsing to zero indicates the whole population has clustered on a single (likely local) optimum.

**Parameter changes to prevent it:**

| Change | Reason |
|--------|--------|
| **Decrease tournament size $t$** (e.g. from 5 → 2) or use weaker rank-based selection | Lower selection pressure; weaker individuals get a chance to reproduce, preserving genetic diversity. |
| **Increase mutation rate** $p_{\text{mut}}$ (or increase Gaussian step size $\sigma$) | Stronger exploration; injects new genetic material that crossover alone cannot create. |
| **Increase population size $N$** | More starting diversity and more parallel exploration of different basins of attraction. |
| **Introduce/strengthen diversity mechanisms** — e.g. fitness sharing, crowding, random immigrants, restart-on-stagnation | Explicitly forces the algorithm to spread across the search space. |
| **Reduce elitism** (don't carry too many elites unchanged) | Heavy elitism magnifies selection pressure and accelerates convergence. |

The general principle: shift the exploration–exploitation balance back toward exploration.

---

#### Part (b)(iii)A — Mutation rate = 0.0 [2 marks]

With mutation switched off, no new genetic material can enter the population — every gene value in the final population must already exist somewhere in the initial population (the **gene pool is fixed**). Crossover can only shuffle the existing alleles between individuals; it cannot introduce values that no parent carries. The EA degenerates into a search over recombinations of the initial alleles, so:

- **Premature convergence** is very likely — the population converges to the best recombination of the initial alleles, which is almost never the global optimum.
- The algorithm cannot recover diversity once selection has driven a gene to a single value across the whole population (gene loss).
- For complex problems the achievable fitness is bounded by what the initial random population happens to contain.

---

#### Part (b)(iii)B — Crossover rate = 0.0 [2 marks]

With crossover switched off, the EA reduces to a **parallel hillclimber / mutation-only EA**: each offspring is just a mutated copy of a parent, and selection biases the parents toward fitter ones. The algorithm loses its ability to **combine good partial solutions from different parents**, which is one of the main theoretical justifications for crossover (building-block hypothesis / schema theorem).

- The search proceeds more slowly because progress is made one mutation at a time.
- For problems with strongly modular / decomposable structure (where good solutions can be assembled from good sub-solutions), performance is *much* worse than with crossover.
- For problems where crossover would be disruptive anyway (e.g. unstructured rugged landscapes), mutation-only EAs can still work — it is essentially a population-based stochastic hillclimber — but it is rarely the most efficient choice.

### Answer

#### Part (a)(i) [4 marks]

See [[concepts/multi-objective-optimization]].

A **Pareto front** for a given population in a multi-objective EA is the set of all **non-dominated solutions** in that population. A solution $\mathbf{a}$ is said to **dominate** another solution $\mathbf{b}$ ($\mathbf{a} \succ \mathbf{b}$) if and only if:

- $\mathbf{a}$ is at least as good as $\mathbf{b}$ on *every* objective, AND
- $\mathbf{a}$ is strictly better than $\mathbf{b}$ on *at least one* objective.

A solution is **non-dominated** if no other solution in the population dominates it. The set of all such non-dominated solutions is the Pareto front (an *approximation* to the true Pareto front during a run; an approximation of the true Pareto front of the problem at convergence).

Geometrically, when plotted in objective space the Pareto front forms a **trade-off curve** (2 objectives) or surface ($\geq 3$ objectives): you cannot improve on one objective without worsening another. These are the rational solutions a decision-maker should choose between.

---

#### Part (a)(ii) [6 marks]

The two desirable properties of a Pareto front approximation are **convergence** and **diversity (spread)**:

| Desirable property | What it means | Why it matters for the end user |
|---------------------|---------------|--------------------------------|
| **Convergence (closeness to the true front)** | The solutions returned lie as close as possible to the true Pareto front of the problem — they are genuinely optimal trade-offs, not dominated by some unfound solution. | The decision-maker is choosing among the *best possible* trade-offs; no recommendation is wasted on a solution that is obviously sub-optimal. |
| **Spread / coverage** | The solutions span the **whole extent** of the Pareto front, including the extreme solutions at each end (cheapest vs highest performing, etc.). | The decision-maker can see the full range of options; any reasonable preference (cost-focused, performance-focused, balanced) is represented and choosable. |
| **Even spacing along the front** | Solutions are roughly evenly distributed in objective space, not clustered in one region. | No region of the trade-off space is under-represented. The decision-maker can pick a trade-off at any preference level — e.g. a moderately priced moderate-performance solution — without finding a "gap" in the offered options. |

**Why these properties enable better decisions:**

If the front is **well converged but poorly spread**, the user only sees options in one part of the trade-off space (e.g. only cheap/low-power solutions); valid preferences outside that region are unsupported. If the front is **well spread but poorly converged**, all the offered solutions are individually sub-optimal — the user is choosing between mediocre options. Only a front that is both close to the true front *and* evenly spread across its full extent gives the user a complete, optimal set of trade-offs from which to make an informed choice based on their own preferences. Modern MOEAs (e.g. NSGA-II via crowding distance, MOPSO via archive management) are explicitly designed to deliver both properties simultaneously. See [[concepts/nsga-ii]], [[concepts/mopso]].

---

#### Part (a)(iii) [5 marks]

See [[comparisons/single-vs-multi-objective]], [[concepts/multi-objective-optimization]].

The weighted-sum approach combines $M$ objectives into a single scalar $F(\mathbf{x}) = \sum_{m=1}^{M} w_m f_m(\mathbf{x})$ and runs a single-objective EA repeatedly with different weight vectors to trace out the Pareto front. A proper multi-objective optimisation algorithm (e.g. NSGA-II, MOPSO) is preferable for several reasons:

1. **One run vs many runs.** A MOEA approximates the entire Pareto front in a *single* run. Weighted-sum requires $k$ separate runs (one per desired Pareto point), each of which evaluates the population and operators afresh — much more expensive computationally.
2. **Cannot find non-convex regions of the Pareto front.** Weighted-sum scalarisation can *only* discover Pareto-optimal solutions on the **convex hull** of the front. Points on concave / non-convex regions are mathematically unreachable as the optimum of any weighted sum — no choice of $w_m$ will return them. A true MOEA, which uses dominance rather than linear scalarisation, has no such restriction.
3. **Weights are hard to choose a priori.** The mapping from weights to Pareto-front position is non-linear and problem-dependent; a uniform sweep of weight values typically produces a heavily *non-uniform* coverage of the front (solutions clump in some regions, gaps in others). Achieving an even spread requires knowing the front's shape — which is what you wanted the algorithm to tell you.
4. **No simultaneous diversity preservation.** Each weighted-sum run converges to a single point; nothing in the algorithm encourages those points collectively to span the front. MOEAs explicitly maintain diversity in objective space using crowding distance, niching or archive management.
5. **Sensitive to objective scaling.** Different objectives may be measured in incomparable units (£ vs MW); choosing meaningful weights requires normalisation that itself depends on the unknown ranges of the objectives.

For these reasons MOEAs (NSGA-II, MOPSO, SPEA2 etc.) are the standard tool for multi-objective optimisation.

---

#### Part (b)(i) [5 marks]

See [[concepts/swarm-intelligence]], [[comparisons/aco-vs-pso]], [[comparisons/ea-vs-swarm]].

PSO and ACO are grouped together as **Swarm Intelligence (SI)** because they share the following defining properties:

1. **Population of simple agents.** Both algorithms maintain a population (swarm) of many simple, identical, low-capability agents — particles in PSO, ants in ACO. No single agent on its own can solve the problem.
2. **Local rules, no central controller.** Each agent follows a simple local update rule (PSO velocity update; ACO transition rule) with no centralised authority directing the search. Every agent is equal — there is no leader or master.
3. **Emergent collective intelligence.** The ability to find good optima is an **emergent property** of the swarm — it arises bottom-up from many agent interactions, not from any individual's reasoning. The whole is more than the sum of its parts.
4. **Indirect / shared communication.** Information is shared across the swarm via a collective structure rather than direct pairwise messaging: ACO uses **pheromone trails (stigmergy)** in the environment; PSO uses the **global best position (gbest)**. Each agent's update depends on this shared swarm-level information.
5. **Positive + negative feedback loops.** Both balance reinforcement of good solutions (positive feedback — pheromone deposition / pull toward gbest) with diversity-preserving mechanisms (negative feedback — pheromone evaporation / stochastic velocity terms $z_1, z_2$).
6. **Biological inspiration from collective animal behaviour.** ACO is inspired by ant foraging (Deneubourg double-bridge experiment); PSO is inspired by bird/fish flocking (Reynolds' Boids).

These shared features — simple agents, decentralised control, emergent intelligence, and shared-environment communication — define the SI framework.

---

#### Part (b)(ii) [5 marks]

| Dimension | PSO | ACO |
|-----------|-----|-----|
| **Problem domain** | Native **continuous** real-valued optimisation. Discrete extensions exist (binary PSO) but are added-on. | Native **discrete / combinatorial** optimisation — solutions constructed as paths through a graph. |
| **Solution representation** | Each particle holds a **complete** real-valued solution vector (position) at every iteration. | Each ant **constructs** its solution incrementally, one decision at a time, by walking a construction graph. |
| **Communication mechanism** | **Direct**: $\text{gbest}$ is broadcast to every particle. Particles also keep a *personal* memory (pbest). | **Indirect (stigmergic)**: ants do not communicate directly; they leave pheromone in a shared matrix and read pheromone deposited by others. |
| **Memory structure** | Per-particle pbest + global gbest. Each particle remembers its own best. | Shared pheromone matrix $\tau_{ij}$ on graph edges. No per-ant memory between iterations (except the current tour). |
| **Update mechanism** | Velocity update combining inertia + cognitive (pbest) + social (gbest) terms; position then updated by adding velocity. | Probabilistic transition rule using $\tau_{ij}^\alpha \eta_{ij}^\beta$ to choose the next node; pheromone update (evaporation + deposition) at end of iteration. |
| **Heuristic incorporation** | Not native; not standard in basic PSO. | Native via $\eta_{ij}$ in the transition rule (e.g. $1/d_{ij}$ for TSP). |
| **Crossover** | None. | None. |
| **Key parameters** | $c_1$, $c_2$, swarm size, inertia. | $\alpha$, $\beta$, $\rho$, $Q$, number of ants. |

Together these differences show that although PSO and ACO share the *SI philosophy*, they are mechanistically distinct algorithms tailored to different problem types (continuous vs combinatorial).

#### Part (b)(iii) [5 marks]

**Convergence indicators for PSO (particles):**

1. **Normalised swarm radius approaches zero.** Compute the average distance of all particles from the swarm centroid (or from gbest) and normalise by the range of the search space. As the swarm converges, this radius shrinks toward 0 — particles cluster tightly around gbest.
2. **Velocities approach zero.** The average magnitude of particle velocities $\|\mathbf{v}_i\|$ drops toward zero — particles are no longer moving, so further iterations cannot change positions.
3. **gbest stagnates.** The global best fitness has not improved for $K$ consecutive iterations (a "no-improvement" counter).
4. **pbest distances → 0.** Every particle's $\|\mathbf{x}_i - \mathbf{p}_i\|$ becomes small — particles are essentially sitting at their own personal bests, which themselves cluster around gbest.

**Convergence indicators for ACO (ants):**

1. **Pheromone concentration on a few edges.** The pheromone matrix becomes highly **uneven** — a small set of edges (a single tour) accumulates almost all the pheromone, while the rest evaporate to near $\tau_{\min}$. A measure such as the **entropy of the pheromone matrix** falls toward zero (or, equivalently, the **branching factor** at each node drops to ≈ 1).
2. **All ants build the same (or nearly the same) tour.** Tour similarity across ants — measured e.g. by the average pairwise edge overlap or the standard deviation of tour lengths — approaches its maximum (or zero standard deviation).
3. **Best tour length stagnates.** The best tour found has not improved for $K$ iterations.
4. **Transition probabilities become deterministic.** For each node, one outgoing edge has probability $\approx 1$ in the transition rule; further iterations produce the same tour with very high probability.

**Common indicator for both:** *no improvement in best fitness over $K$ iterations* is the simplest and most widely used stopping signal. The structural indicators (swarm radius / pheromone concentration) confirm that the population itself has lost diversity and so further iterations are unlikely to help.

## Question 4 — EA for Neural Network Weights & Self-Organising Maps

**(a) Traditionally, perceptrons learn through the use of a learning algorithm (such as the delta rule), however, evolutionary algorithms (EAs) can also be used to optimise the weights within a perceptron.**

**(i) Describe how you would use an evolutionary algorithm to optimise the weights in a perceptron, paying particular attention to the representation, selection process, variation operators, fitness function, parameter settings and experimental design used to test your approach.**

**(10 marks)**

**(ii) The EA could also be used to optimise the weights of a multi-layer perceptron (MLP). Explain how you would change the EA representation to optimise the weights of an MLP and discuss any differences you would expect to see in algorithm behaviour as a result.**

**(5 marks)**

**(b) *Self-organising maps* (also known as *Kohonen networks*) are neural networks that are able to learn structure from data using unsupervised learning.**

**(i) Explain what is meant by the word *unsupervised* in this context and how *unsupervised learning* differs from *supervised learning*.**

**(5 marks)**

**(ii) Show, with the aid of equations, how a self-organising map learns from an input pattern presented to it.**

**(5 marks)**

**(iii) Two variables involved in the learning process in a self-organising map will change over time. Explain what these variables are, how they change, and the effect this has on the learning process.**

**(5 marks)**

**(Total 30 marks)**

### Answer

#### Part (a)(i) [10 marks]

See [[concepts/neural-networks]], [[concepts/evolutionary-algorithms]].

**Goal:** evolve the connection weights of a single-layer perceptron (one neuron with $n$ inputs) to fit a labelled training set, as an alternative to gradient-based delta-rule training.

**Representation.** Encode the perceptron's free parameters as a **real-valued chromosome**:
$$\mathbf{w} = (w_1, w_2, \ldots, w_n, \theta)$$
where $w_i$ are input weights and $\theta$ is the threshold (bias). Chromosome length $= n + 1$ (or $n$ if threshold fixed). This is a **direct encoding** — each gene corresponds to a tunable weight.

**Fitness function.** Evaluate the perceptron on a training set of $(\mathbf{x}_i, t_i)$ pairs. For classification:
$$\text{fitness} = \text{classification accuracy} = \frac{\text{# correct}}{N}$$
For regression / continuous outputs, use the **negative Mean Absolute Error** (so that higher fitness is better):
$$\text{fitness} = -\text{MAE} = -\frac{1}{N}\sum_{i=1}^{N} |y_i^{\text{pred}} - t_i|$$
Higher fitness ↔ better weights.

**Selection.** Use **tournament selection** with $t = 2$ to $5$. It is efficient, robust to fitness scaling, immune to superfit dominance, and gives tunable selection pressure via $t$.

**Variation operators.**
- **Mutation:** additive Gaussian mutation. With probability $p_{\text{mut}}$ pick a random weight and add $\delta \sim \mathcal{N}(0, \sigma^2)$. Small $\sigma$ → local refinement (exploitation); occasional larger jumps → exploration.
- **Crossover:** uniform crossover (or single-point) between two parent weight vectors with probability $p_{\text{cross}}$. Recombines good partial weight sets from two parents.

**Parameter settings (typical).**

| Parameter | Typical value |
|-----------|---------------|
| Population size $N$ | 50–100 |
| Mutation rate $p_{\text{mut}}$ | 0.05–0.1 per gene |
| Crossover rate $p_{\text{cross}}$ | 0.7–0.9 |
| Mutation step size $\sigma$ | 0.01–0.1 (often self-adaptive) |
| Generations | 100–500, with early-stopping when fitness plateaus |
| Elitism | retain best 1–2 individuals each generation |
| Tournament size $t$ | 2–5 |

**Experimental design.**

1. **Train/validation/test split.** Hold out an unseen test set (e.g. 70 / 15 / 15) so that final performance is evaluated on data the EA has never seen. Use the validation set for early stopping and hyper-parameter selection.
2. **Cross-validation** ($k$-fold, e.g. $k=10$) to guard against overfitting and report a more robust performance estimate.
3. **Multiple runs.** EAs are stochastic — run the algorithm e.g. 30 times with different random seeds; report **mean and standard deviation** of test fitness.
4. **Baseline comparison.** Compare against the **delta rule / backpropagation** trained perceptron on the same data and folds, and a random-weights baseline.
5. **Statistical significance.** Use a non-parametric test (Wilcoxon signed-rank, or Mann–Whitney $U$) to determine whether any observed difference between EA and baseline is statistically significant.
6. **Generalisation check.** Inspect train vs test fitness over the run; if the gap grows, overfitting is occurring and stopping criteria / regularisation should be revisited.

#### Part (a)(ii) [5 marks]

**Representation change for an MLP.**

An MLP with layer sizes $n_0, n_1, n_2, \ldots, n_L$ (input, hidden(s), output) has total weight count
$$W = \sum_{l=1}^{L} (n_{l-1} + 1) \cdot n_l$$
(the $+1$ accounts for a bias per neuron). All these weights are concatenated into a **single longer real-valued chromosome** of length $W$. For example, a 2-input, 3-hidden, 1-output MLP has $(2+1)\cdot 3 + (3+1)\cdot 1 = 9 + 4 = 13$ weights. The encoding is *structurally identical* to the perceptron case (real vector + Gaussian mutation + crossover) but much **longer**, and the MLP architecture (number of layers / neurons) must be fixed in advance — the EA optimises only the weights, not the structure.

**Expected differences in algorithm behaviour.**

1. **Much higher-dimensional search space.** Many more weights → exponentially larger landscape with many more local optima, saddle points and plateaus. Convergence is slower and premature convergence is more likely.
2. **High epistasis.** Weights in different layers interact in complex non-linear ways — the effect of mutating one weight depends on the values of many others. The landscape is much more rugged than the perceptron's, which reduces the effectiveness of operator-by-operator search.
3. **Crossover may be disruptive.** Uniform crossover on a long chromosome can break apart **co-adapted groups of weights** in the same layer; layer-wise (block) crossover or even disabling crossover can help.
4. **Increased computational budget.** Larger population, more generations, and possibly self-adaptive mutation step sizes are typically needed to make progress.
5. **EA is less efficient than backpropagation here.** For differentiable MLPs, backprop computes exact gradients and converges far more quickly per evaluation. EAs become preferable when activation functions are **non-differentiable** (e.g. step functions, spiking neurons — see [[concepts/spiking-neural-networks]]), when the **architecture itself** is to be evolved (neuroevolution / NEAT), or when fitness is noisy/non-differentiable.

---

#### Part (b)(i) [5 marks]

See [[concepts/self-organising-maps]], [[sources/lecture19-som]].

**Unsupervised learning** means the training data consists of inputs only, with **no labelled targets**. The learning algorithm has no externally provided "correct answer" against which to measure error — instead, it must discover **regularities, clusters or structure** *intrinsic to the data* using only the inputs themselves.

**Differences from supervised learning:**

| Aspect | Supervised | Unsupervised |
|--------|-----------|--------------|
| **Training data** | Input + target output pairs $(\mathbf{x}_i, t_i)$ | Inputs only $\mathbf{x}_i$ |
| **Goal** | Learn a mapping $\mathbf{x} \to t$ that predicts targets for new inputs | Discover structure / regularities / clusters in the input space |
| **Error signal** | Explicit difference between predicted output and target (drives weight updates) | No external error — weights driven by competition / similarity / data statistics |
| **Training method (example)** | Backpropagation, delta rule | Competitive learning, $k$-means, SOM |
| **Typical use** | Classification, regression, function approximation | Clustering, dimensionality reduction, visualisation, anomaly detection |
| **Example algorithm** | MLP trained with backprop | SOM, $k$-means, autoencoder |

In a SOM, no target output is presented with each input pattern; the map self-organises so that nearby map units come to respond to similar input patterns, purely from the statistics of the inputs.

---

#### Part (b)(ii) [5 marks]

See [[concepts/self-organising-maps]].

A SOM consists of a 2D grid of neurons, each with a weight vector $\mathbf{w}_i \in \mathbb{R}^d$ of the same dimensionality as the input. When an input pattern $\mathbf{x}$ is presented, learning proceeds in three stages:

**1. Competition — find the Best Matching Unit (BMU).**

The BMU is the neuron whose weight vector is closest (Euclidean) to $\mathbf{x}$:
$$\text{BMU} = \arg\min_i \|\mathbf{x} - \mathbf{w}_i\|$$

**2. Cooperation — define a neighbourhood around the BMU.**

A neighbourhood function $h(i, \text{BMU}, t)$ centred on the BMU determines how strongly each neuron $i$ co-operates with the BMU. A typical choice is a **Gaussian over the grid distance**:
$$h(i, \text{BMU}, t) = \exp\!\left(-\frac{d(i, \text{BMU})^2}{2\sigma(t)^2}\right)$$
where $d(i, \text{BMU})$ is the (lattice) distance on the 2D grid and $\sigma(t)$ is the current neighbourhood radius. The BMU itself receives $h = 1$; further neurons receive progressively smaller $h$.

**3. Adaptation — update weights.**

The BMU and its neighbours move their weight vectors *toward* the input:
$$\mathbf{w}_i(t+1) = \mathbf{w}_i(t) + \eta(t) \cdot h(i, \text{BMU}, t) \cdot (\mathbf{x} - \mathbf{w}_i(t))$$

where $\eta(t)$ is the (time-dependent) learning rate. The BMU moves the most (since $h = 1$); neighbours move proportionally less according to $h$. Neurons far from the BMU receive almost no update.

After many such presentations the map self-organises: similar inputs activate (and are mapped to) nearby grid positions, producing the topology-preserving 2D representation of the input space.

---

#### Part (b)(iii) [5 marks]

The two variables that **change over time** during SOM training are:

**Variable 1 — Learning rate $\eta(t)$.**

The learning rate $\eta(t)$ scales how much each weight vector is moved toward the input on each presentation. It is initialised to a relatively large value (e.g. $\eta(0) \approx 0.5$) and **decays monotonically toward zero** over training (typically exponential or linear decay, $\eta(t) = \eta_0 \exp(-t/\tau)$ or $\eta_0 (1 - t/T)$).

**Effect on learning:**
- **Early on (large $\eta$):** weights make large adjustments. This is the **ordering phase** — the global topology of the map self-organises rapidly; neurons sort themselves into approximately correct positions.
- **Later (small $\eta$):** weights make only fine adjustments. This is the **convergence / fine-tuning phase** — the map refines without destabilising the gross topology learned earlier.
- Without this decay the map would never settle — late-stage updates would continually undo what had already been learned.

**Variable 2 — Neighbourhood radius $\sigma(t)$.**

The width of the neighbourhood function $h(i, \text{BMU}, t)$ — equivalently, the standard deviation $\sigma(t)$ of the Gaussian — is initialised wide (covering a large fraction of the grid, e.g. half the map size) and **shrinks monotonically toward a small value** (often just the BMU itself) as training progresses.

**Effect on learning:**
- **Early (wide $\sigma$):** many neurons in a large neighbourhood are updated together with the BMU. This produces **large-scale topological organisation** — neighbouring grid units learn to respond to similar inputs, and the global topology of the input space is mapped onto the grid.
- **Later (narrow $\sigma$):** only the BMU and a small handful of immediate neighbours are updated. The map enters a **fine-tuning phase** where individual neurons specialise to capture finer features without disturbing distant regions.
- Without the shrinking neighbourhood, either the map could never coarsely self-organise (if $\sigma$ starts small) or it could never refine local detail (if $\sigma$ stays large).

**Joint effect.** The simultaneous decay of $\eta(t)$ and $\sigma(t)$ realises a classic **annealing schedule**: training begins broad and aggressive (global ordering), and ends narrow and gentle (local fine-tuning). This two-phase trajectory — ordering then convergence — is what enables the SOM to learn a stable topology-preserving map.
