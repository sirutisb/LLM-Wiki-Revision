### Answer

#### Part (a) [7 marks]

**PSO pseudocode:**

```
INPUTS:
  N           = swarm size                            (typical: 20-50)
  c1          = cognitive (personal) coefficient      (typical: 2.0)
  c2          = social (global) coefficient           (typical: 2.0)
  v_max       = velocity clamp                        (typical: 10-20% of search range)
  T_max       = maximum number of iterations          (typical: 100-10000)
  D           = dimension of search space             (problem-specific)

1. For each particle i = 1..N:
     initialise position  x_i  uniformly at random in the search space
     initialise velocity  v_i  small random vector
     set  pbest_i = x_i
2. Set  gbest = argmax_i f(pbest_i)   (or argmin for minimisation)
3. For t = 1..T_max:
     For each particle i = 1..N:
       For each dimension j = 1..D:
         z1, z2 ~ U(0,1)
         v_ij <- v_ij + c1 * z1 * (pbest_ij - x_ij)
                       + c2 * z2 * (gbest_j   - x_ij)
         clamp v_ij to [-v_max, v_max]
         x_ij <- x_ij + v_ij
       Evaluate f(x_i)
       If f(x_i) is better than f(pbest_i):  pbest_i <- x_i
       If f(x_i) is better than f(gbest):    gbest   <- x_i
4. Return gbest
```

Velocity has three components: **inertia** ($v_{ij}$), **cognitive pull** toward pbest, and **social pull** toward gbest. The random multipliers $z_1, z_2 \sim U(0,1)$ inject stochasticity each step so the swarm does not collapse prematurely.

**Termination alternatives:** maximum iterations, target fitness reached, no improvement for $k$ iterations, or normalised swarm radius $\approx 0$.

See [[pso]].

#### Part (b)(i) [4 marks]

A mutation operator perturbs a parent chromosome to produce a new candidate. It must be designed so that:

1. **Validity** — the mutated chromosome must still represent a feasible solution. The valid set of chromosomes depends entirely on the encoding (permutation, binary, real, tree, etc.); an operator that ignores this can output a chromosome that is not a valid member of the encoding's space.
2. **Locality / step size** — mutation defines the *neighbourhood* and hence the [[fitness-landscapes|fitness landscape]]. For a real-valued vector, "a small change" means adding a small Gaussian; for a binary string it means flipping a single bit; for a tree (GP) it means replacing one subtree. Applying a real-valued perturbation to a binary chromosome does not produce a small change — it does not even produce a chromosome.
3. **Smoothness of phenotype mapping** — if a single-gene mutation changes the phenotype drastically, the landscape is rugged and the EA cannot exploit local structure. The right operator keeps similar genotypes mapping to similar phenotypes.
4. **No bias toward infeasible regions** — for direct encodings of constrained problems, a careless operator can push almost every offspring into the infeasible region, wasting evaluation budget.

In short: encoding, mutation operator, and landscape are designed together. Choosing the operator is choosing the landscape. See [[representations]], [[crossover-mutation]].

---

#### Part (b)(ii) [3 marks]

**Example: applying single-gene random-replacement mutation to a permutation encoding (e.g. TSP).**

A TSP tour over $n$ cities is encoded as a permutation, e.g. `(2, 0, 6, 7, 5, 1, 4, 3)`. "Random replacement" mutation picks one gene and replaces it with a random city, e.g. picks position 3 and changes the `6` to a `4`, giving `(2, 0, 4, 7, 5, 1, 4, 3)`.

**Why this is poor:**

- The result is **not a permutation** — city `4` now appears twice and city `6` has been dropped. The chromosome represents an invalid tour (a city visited twice, another never visited).
- Every such mutation produces an infeasible solution; the EA either spends its budget evaluating garbage or relies on costly repair.
- The correct operator for permutations is the **swap** operator (or inversion / insertion), which exchanges two positions and is guaranteed to yield another valid permutation.

This illustrates the general rule: standard k-ary mutation breaks permutation validity — see [[crossover-mutation]].

#### Part (c) [6 marks]

Craig Reynolds (1987) modelled flocking as the emergent behaviour of homogeneous agents called **boids**, each following three simple local rules over its omni-directional neighbourhood of nearby flockmates:

1. **Separation** — *steer to avoid crowding local flockmates*. Each boid moves away from neighbours that are too close, preventing collisions and maintaining personal space.
2. **Alignment** — *steer toward the average heading of local flockmates*. Each boid adjusts its direction to match the mean velocity of its neighbours, synchronising movement direction across the flock.
3. **Cohesion** — *steer toward the average position of local flockmates*. Each boid moves toward the centroid of its neighbours, keeping the group together.

With just these three rules, large-scale flocking behaviours emerge: spontaneous polarisation, synchronised turns, flash expansion when startled, flock merging and splitting — all with **no leader and no global plan**.

See [[flocking-boids]].

#### Part (d)(i) [4 marks]

| Symbol | Meaning |
|--------|---------|
| $J_i^k$ | The set of cities **still allowed** for ant $k$ when it is at city $i$ — i.e. cities not yet on its current tour memory. This enforces the permutation constraint of TSP. |
| $\tau_{ij}(t)$ | The **pheromone** concentration on edge $(i, j)$ at iteration $t$. Encodes the collective experience of all ants: edges that belonged to good tours accumulate more pheromone. |
| $\eta_{ij}(t)$ | The **heuristic value** for edge $(i, j)$. For TSP, $\eta_{ij} = 1/d_{ij}$ where $d_{ij}$ is the distance — a short edge gives a large $\eta$. (Subscript $t$ allows for time-varying heuristics; for static TSP $\eta$ does not depend on $t$.) |
| $\alpha$ | The exponent controlling the **relative weight of pheromone** in the choice rule. |
| $\beta$ | The exponent controlling the **relative weight of the heuristic** (distance) in the choice rule. |

The denominator normalises the numerator over all currently allowed cities, so the rule defines a probability distribution that sums to 1.

See [[ant-colony-optimization]].

---

#### Part (d)(ii) [2 marks]

- **Effect of $\alpha$** — controls how strongly ants follow the **collective memory** (pheromone). Large $\alpha$ → ants concentrate on edges with established trails, exploiting current best routes but risking premature convergence to a suboptimal tour. Small $\alpha$ (e.g. $\alpha = 0$) → pheromone is ignored, the search becomes a greedy nearest-neighbour heuristic without learning.
- **Effect of $\beta$** — controls how strongly ants follow the **problem heuristic** (short edges). Large $\beta$ → strong greedy bias toward short edges, fast initial convergence but reduced ability to deviate from a locally short but globally suboptimal tour. Small $\beta$ (e.g. $\beta = 0$) → distance is ignored; ants choose purely by pheromone, ignoring domain knowledge.

The pair $(\alpha, \beta)$ thus tunes the **exploration–exploitation balance** between learned trail-following and heuristic greediness. Typical values: $\alpha = 1$, $\beta = 2$ to $5$.

#### Part (e)(ii) [4 marks]

Annotate the diagram from (i) as follows:

- **Under-fit region** — the region to the **left** of the minimum of the generalisation-error curve. Here both errors are high; the model is too simple to capture the underlying pattern. Adding capacity (more neurons, more layers, longer training) would reduce both errors.

#### Part (f) [6 marks]

Conway's Game of Life (1970) is a 2D cellular automaton with binary cell states (alive / dead) and the **Moore neighbourhood** (the 8 cells surrounding each cell). At each timestep every cell synchronously applies one of three rules based on its current state and the number of live neighbours:

- **Birth** — *triggered when a **dead** cell has **exactly 3** live neighbours*. In the next step that cell becomes alive (analogue of reproduction).
- **Death** — *triggered when a **live** cell has **fewer than 2** live neighbours (under-population / isolation) or **more than 3** live neighbours (over-population / overcrowding)*. In the next step the cell becomes dead.
- **Stasis (survival / unchanged)** — *triggered in any other case*. Specifically: (i) a live cell with **exactly 2 or 3** live neighbours stays alive (survival); (ii) a dead cell with **any number of live neighbours other than 3** stays dead.

These three deterministic local rules produce remarkably complex emergent behaviour — gliders, oscillators, spaceships, glider guns — and the system is Turing-complete. See [[cellular-automata]].

### Answer

#### Part (a) [6 marks]

**Generational EA with elitism, tournament selection, crossover + mutation:**

```
INPUTS:
  popsize    = population size                          (typical: 50-500)
  p_cross    = crossover probability                    (typical: 0.6-0.95)
  p_mut      = mutation probability per chromosome      (typical: 0.01-0.1)
  t          = tournament size                          (typical: 2-5)
  elite_k    = number of elites preserved each gen      (typical: 1-2)
  G_max      = maximum number of generations            (typical: 100-10000)

1. Initialise population P of  popsize  random valid solutions; evaluate fitness
2. For g = 1..G_max:
     a. Carry the best  elite_k  individuals into G' unchanged
     b. While |G'| < popsize:
          parent1 <- tournament_select(P, t)
          parent2 <- tournament_select(P, t)
          With probability p_cross: child1, child2 <- crossover(parent1, parent2)
          else: child1, child2 <- copy(parent1, parent2)
          With probability p_mut:  child1 <- mutate(child1)
          With probability p_mut:  child2 <- mutate(child2)
          Evaluate child1, child2; add to G'
     c. P <- G'
     d. If stopping criterion met (best unchanged for N generations,
        budget exhausted, target fitness reached): break
3. Return best individual ever found
```

Components are representation-specific: **selection** (tournament, rank, roulette — [[selection]]), **crossover & mutation** (encoding-specific — [[crossover-mutation]]), **replacement** (generational with elitism shown above; steady-state alternative replaces only the weakest). See [[evolutionary-algorithms]].

#### Part (b) [6 marks]

| Aspect | Genetic Algorithm (GA) | Genetic Programming (GP) |
|--------|------------------------|---------------------------|
| **Generic EA loop** | Same: init → select → vary → evaluate → replace | Same |
| **Chromosome** | Fixed-length linear vector (binary, integer, real, permutation) | Variable-length **tree** of function and terminal nodes |
| **What is being evolved** | Solution **parameters** (weights, coordinates, choices) | A **program / expression** that computes the solution |
| **Selection** | Tournament / rank / roulette | Same (no difference) |
| **Crossover** | 1-point, 2-point, uniform crossover on vector | **Subtree crossover**: swap a random subtree from each parent |
| **Mutation** | Bit-flip, single-gene replacement, Gaussian noise | **Subtree mutation**: replace a random subtree with a new random subtree |
| **Fitness evaluation** | Direct formula on the chromosome | Must **run the program** on a set of training cases; sum of errors |
| **Compute cost per evaluation** | Cheap | Often expensive (program execution + many test cases) |
| **Typical output** | A parameter vector | A symbolic expression / program |
| **Canonical example** | TSP, knapsack, antenna weights, NN weight tuning | Symbolic regression (rediscover $x^2 + x + 1$), antenna geometry, evolved controllers |
| **Special issue** | Encoding–operator mismatch (e.g. permutations) | **Bloat** — trees grow large; need depth limits or parsimony pressure |

**Similarities (the EA core is identical):** both are population-based, both use selection + variation + replacement, both rely on a fitness function, both inherit problems like premature convergence and the No Free Lunch theorem.

**Differences:** GA evolves a parameter vector for a *fixed* problem structure; GP evolves the *structure itself* (the algorithm). GP requires more compute and needs Koza's five preparatory steps (terminal set, function set, fitness measure, run parameters, termination criterion). See [[genetic-algorithms]], [[genetic-programming]].

#### Part (c) [6 marks]

**Direct representation:** the chromosome **explicitly stores the decision variables** of the problem. The mapping from genotype to phenotype is trivial — typically the identity (or a simple reshape).

**Indirect representation:** the chromosome stores the **parameters of a constructive heuristic** that builds a solution at decode time. The genotype-to-phenotype mapping is non-trivial: a decoder runs, often using domain knowledge, to translate the gene string into an actual candidate.

| | Direct | Indirect |
|--|--------|---------|
| Genotype | A variable of the problem | A heuristic parameter |
| Invalid solutions in search space | Common (must penalise) | Rare (decoder builds feasible) |
| Search space size | Larger | Smaller |
| Landscape | Usually smoother | Can be rugged |
| Domain knowledge | Not required | Required (in the decoder) |
| Decode speed | Fast (identity) | Slower (run the decoder) |

**Example — direct (exam timetabling):** chromosome = $(s_1, s_2, \ldots, s_n)$ where $s_i$ is the **time slot directly assigned to exam $i$**. Mutation flips one slot. Many chromosomes correspond to clashing schedules and need a fitness penalty.

**Example — indirect (exam timetabling):** chromosome = $(k_1, k_2, \ldots, k_n)$ where $k_i$ means "place exam $i$ in the **$k_i$-th currently clash-free slot**". A decoder iterates over exams and consults the partial schedule to choose the $k_i$-th admissible slot. Mutation changes one index. Most decoded schedules are feasible by construction — the search space is much smaller and concentrated on the feasible region.

Another canonical pair: TSP can be encoded directly as a permutation of cities, or indirectly as a sequence of choices into a nearest-neighbour heuristic. See [[representations]].

#### Part (d) [6 marks]

**Why EAs produce different solutions on different seeds:**

EAs are **stochastic** algorithms. Sources of randomness in every run include:

1. **Random initialisation** of the population — the starting points cover different regions of the landscape on every seed.
2. **Stochastic selection** — tournament draws, roulette spins, and random tiebreaks all depend on the PRNG.
3. **Stochastic variation** — crossover points, mutation positions, mutation magnitudes are sampled randomly.
4. **Stochastic replacement** in some variants.

The trajectory through the search space therefore differs across seeds, ending in different (local) optima. Two runs starting in different basins of attraction can converge to genuinely different solutions of comparable quality, or one may find a much better optimum than the other.

**Why this matters for problems of high combinatorial complexity:**

On NP-hard problems the search space is exponentially large (e.g. $n!$ for TSP, $2^n$ for subset selection) and the fitness landscape is multimodal — many local optima of similar quality and a few much-better global optima. No deterministic polynomial-time algorithm can be relied upon to find the global optimum. Properties that follow:

- **Multiple independent runs sample the landscape from different starting points**, dramatically increasing the chance that at least one run reaches the global optimum (or a near-optimal basin).
- **The best-of-$N$-runs strategy** is standard practice precisely because variability is a feature, not a bug.
- **Different runs reveal alternative high-quality solutions** that may be preferred for different operational reasons (a single "the answer" is rarely well-defined for combinatorial design problems).
- **Statistical reporting** is therefore essential: report mean ± std and use significance tests (Mann–Whitney U, Wilcoxon) when comparing algorithms — see Sörensen's metaphor critique ([[metaheuristics-critique]]).

In short: stochasticity is what allows EAs to escape local optima on hard landscapes; the price is non-reproducibility per run, paid back by averaging or best-of-$N$ aggregation.

#### Part (e) [6 marks]

**Convergence** in an EA refers to the population becoming **concentrated** in a (small) region of the search space — typically around a single (local) optimum. Operationally, it shows up as: (i) the population's genetic diversity collapses, (ii) the best fitness in the population stops improving, (iii) selection produces very similar parents, and (iv) crossover effectively does nothing because parents are near-identical. A converged EA has settled on a solution and no longer searches.

**Premature / early convergence** is the failure mode where the population converges **too soon** — onto a **local** optimum rather than the global one — and then cannot escape because diversity is gone. Causes: very high selection pressure (large tournament size), too-low mutation rate, dominance by a super-fit individual under roulette wheel, very small population.

**Diagram** (best fitness vs generations):

```
Fitness
   ^
   |                                  ____________ global optimum
   |                                 /
   |                  __________    /
   |                 /          \  /                healthy run
   |                /            \/
   |               /
   |       _______
   |      /                  premature-convergence run (early plateau)
   |     /  
   |    /  
   |___/_________________________________________
   |                                                   -> Generations
```

- **Healthy run** (upper trajectory): fitness improves quickly, plateaus briefly, then improvements continue as crossover + mutation escape local optima, eventually reaching the global optimum.
- **Prematurely converged run** (lower trajectory): fitness improves quickly, hits a much **lower plateau**, and stays there forever — the population has collapsed onto a local optimum and there is no longer enough diversity for variation operators to escape.

**Mitigations:** reduce selection pressure (smaller tournament, rank-based selection), increase mutation rate, larger population, restart from the diverse population, or use niching/crowding to maintain diversity. See [[evolutionary-algorithms]], [[selection]].

### Answer

#### Part (a) [2 marks]

- **Pareto set:** the set of **non-dominated decision vectors** in the search space — i.e. all $\mathbf{x} \in \Omega$ such that no other feasible $\mathbf{x}'$ Pareto-dominates $\mathbf{x}$. It lives in **decision space**.
- **Pareto front:** the image of the Pareto set under the objective mapping — i.e. the set of objective vectors $\{ \mathbf{F}(\mathbf{x}) : \mathbf{x} \text{ in the Pareto set} \}$. It lives in **objective space** and traces out the optimal trade-off surface among the conflicting objectives.

See [[multi-objective-optimization]].

---

#### Part (b)(i) [2 marks]

In the ZDT family:

- **$g(x_2, \ldots, x_n)$ controls convergence.** It defines a scalar landscape over the auxiliary variables $x_2, \ldots, x_n$; the true Pareto front corresponds to $g = g^* = 1$ (the optimum of $g$). The harder $g$ is to minimise (multi-modality, deceptiveness, biased sampling), the harder it is for the algorithm to *reach* the true front.
- **$h(f_1, g)$ together with $f_1(x_1)$ controls diversity.** $f_1$ depends on $x_1$ and parameterises the position along the front; $h$ describes the **shape** of the front (convex, concave, disconnected). These determine whether solutions can be spread evenly along the front and which regions are intrinsically hard to populate.

So: $g \to$ convergence (vertical distance to the front), $f_1 \cdot h \to$ diversity (spread along the front).

---

#### Part (b)(ii) [3 marks] — Properties that make convergence difficult

1. **Multi-modality of $g$:** when $g$ has many local optima, the algorithm gets trapped at a $g$-local-optimum corresponding to a *dominated* approximate front far above the true Pareto front.
2. **Deceptive landscape:** the gradient of $g$ leads the search away from the global optimum (toward a misleading basin), so naive variation operators converge to suboptimal regions.
3. **Bias / non-uniform distribution of $g$:** if the genotype space maps non-uniformly to $g$ values (e.g. most random samples have very high $g$), the algorithm spends most of its budget far from $g = g^*$, slowing convergence.

Other valid points: many local Pareto fronts; flat plateaus in $g$; high decision-space dimensionality; rugged landscape with high epistasis among the auxiliary variables.

---

#### Part (b)(iii) [3 marks] — Properties that make diversity difficult

1. **Non-convex / concave front shape:** standard dominance-based MOEAs concentrate mass at the knees and extremes; uniform coverage of a concave region is harder.
2. **Disconnected Pareto front:** when the true front consists of disjoint segments, the algorithm may converge on just one segment and never explore the others.
3. **Biased / non-uniform mapping from $x_1$ to $f_1$:** if $f_1$ values are not evenly attainable from uniform $x_1$, evenly spaced solutions in genotype space produce uneven coverage along the front.

Other valid points: extreme aspect-ratio differences between objectives; very wide spread between extreme solutions making selection pressure for middle solutions weak; presence of many local front segments at intermediate $g$ values.

#### Part (d) [5 marks]

**Many-objective optimisation** refers to multi-objective problems with **four or more objectives** ($M \geq 4$). The standard MOEA toolset that works for $M = 2$ or $3$ (Pareto dominance + crowding distance) breaks down in this regime.

**Four reasons this is challenging:**

1. **Dominance resistance / loss of selection pressure.** As $M$ grows, the probability that one random solution dominates another shrinks rapidly — most pairs become mutually non-dominated. The whole population tends to land in Rank 0, so non-dominated sorting no longer discriminates, and there is no selection pressure to push solutions toward the front.
2. **Exponential growth in the number of solutions needed.** The Pareto front is now an $(M-1)$-dimensional manifold; covering it with an even point set requires a number of solutions that grows exponentially in $M$. A practical population (a few hundred) gives a hopelessly sparse approximation.
3. **Visualisation and decision-making are hard.** Humans cannot directly view a 4+ dimensional Pareto front. Parallel coordinate plots, scatter-plot matrices, and dimensionality-reduction techniques (e.g. self-organising maps) are needed, and the decision-maker has trouble identifying preferred trade-offs.
4. **Diversity measures degrade.** Crowding distance and similar local-density estimates become unreliable in high dimensions (curse of dimensionality — most points appear roughly equidistant), so the diversity tiebreaker stops working effectively.

**Algorithmic remedies:** reference-point based methods (NSGA-III), decomposition-based methods (MOEA/D), indicator-based selection using **hypervolume** or **average rank** ([[mopso]]) — these replace pure dominance with stronger discriminators.

See [[multi-objective-optimization]], [[nsga-ii]].

#### Part (e)(i) [6 marks]

The **hypervolume (HV)** of $S$ with reference point $\mathbf{z}^r = (9, 10)$ is the area of the region in objective space **dominated by at least one point of $S$ but dominated by $\mathbf{z}^r$** (i.e. with $f_1 \le 9$ and $f_2 \le 10$).

Sort the points by $f_1$ ascending: $x^1(1,9), x^2(3,7), x^3(5,4), x^4(7,2), x^5(8,1)$.

Slice the dominated region into **vertical strips** between successive $f_1$ values; in each strip the top edge is set by the smallest $f_2$ among points with $f_1$ at or to the left of the strip's left boundary.

| Strip ($f_1$ range) | Width | Dominating point (smallest $f_2$ at or left of left edge) | Top edge ($f_2$) | Height ($z^r_2 - f_2$) | Area |
|---|---|---|---|---|---|
| $[1, 3]$ | $3 - 1 = 2$ | $x^1$ with $f_2=9$ | $9$ | $10 - 9 = 1$ | $2 \times 1 = 2$ |
| $[3, 5]$ | $5 - 3 = 2$ | $x^2$ with $f_2=7$ | $7$ | $10 - 7 = 3$ | $2 \times 3 = 6$ |
| $[5, 7]$ | $7 - 5 = 2$ | $x^3$ with $f_2=4$ | $4$ | $10 - 4 = 6$ | $2 \times 6 = 12$ |
| $[7, 8]$ | $8 - 7 = 1$ | $x^4$ with $f_2=2$ | $2$ | $10 - 2 = 8$ | $1 \times 8 = 8$ |
| $[8, 9]$ | $9 - 8 = 1$ | $x^5$ with $f_2=1$ | $1$ | $10 - 1 = 9$ | $1 \times 9 = 9$ |

$$\boxed{HV(S) = 2 + 6 + 12 + 8 + 9 = 37 \text{ (square units)}}$$

#### Part (e)(ii) [3 marks]

In **SMS-EMOA**, environmental selection removes the solution with the **smallest exclusive hypervolume contribution** — the smallest area dominated by that solution alone (i.e. would disappear if the solution were removed).

For sorted $x^1 \ldots x^5$ with reference $\mathbf{z}^r = (9, 10)$, the exclusive contribution of each point is the rectangle bounded by its $f_2$, the $f_2$ of the next point in $f_2$-order (which here equals the previous point in $f_1$-order since all points form a Pareto-front), and its $f_1$ relative to its neighbour in $f_1$:

| Solution | Exclusive contribution |
|----------|------------------------|
| $x^1 (1,9)$ | $(f_1^2 - f_1^1) \times (z^r_2 - f_2^1) = (3-1)(10-9) = 2 \times 1 = 2$ |
| $x^2 (3,7)$ | $(f_1^3 - f_1^2) \times (f_2^1 - f_2^2) = (5-3)(9-7) = 2 \times 2 = 4$ |
| $x^3 (5,4)$ | $(f_1^4 - f_1^3) \times (f_2^2 - f_2^3) = (7-5)(7-4) = 2 \times 3 = 6$ |
| $x^4 (7,2)$ | $(f_1^5 - f_1^4) \times (f_2^3 - f_2^4) = (8-7)(4-2) = 1 \times 2 = 2$ |
| $x^5 (8,1)$ | $(z^r_1 - f_1^5) \times (f_2^4 - f_2^5) = (9-8)(2-1) = 1 \times 1 = 1$ |

The smallest contribution is **$x^5$ with HV-contribution $= 1$**.

**Therefore $x^5$ is removed.** It is the solution whose objective vector is "most redundant" given the rest of the set — removing it shrinks the dominated region the least, so the population retains the highest hypervolume after deletion. Intuitively $x^5(8,1)$ and $x^4(7,2)$ are very close in objective space, so $x^5$ adds little new dominated area on top of $x^4$.

See [[mopso]] (hypervolume contribution) and [[multi-objective-optimization]].

### Answer

#### Part (a) [15 marks]

**Note on syllabus relevance:** Sejnowski & Rosenberg's NETtalk paper is **not in the current ECM3412 syllabus** as a named reading. The answer below draws on general NIC knowledge of MLPs and the standard NETtalk experimental design. See **Out-of-syllabus topics** section below.

**Problem:** Map English orthography to phonetic output — given a word, produce the correct pronunciation (phonemes plus stress markers). This is hard because English spelling is irregular: the same letter is pronounced differently depending on context (e.g. "c" in "cat" vs "cell", final "e" in "make").

**Neural network approach — overall design:**

NETtalk is a **feed-forward multi-layer perceptron** (one hidden layer) trained by **back-propagation** on aligned text–phoneme pairs.

**Architecture:**

- **Input layer**: a sliding window of **7 consecutive letters** (3 letters of left context, 1 central letter, 3 letters of right context). Each letter slot is **one-hot encoded** over the 29 input symbols (26 letters + punctuation/space) — 7 × 29 = **203 input units**.
- **Hidden layer**: a single layer of approximately **80 hidden sigmoid units** (Sejnowski & Rosenberg experimented with 0–120 units; ~80 was the standard configuration).
- **Output layer**: **26 output units** encoding the phoneme of the central letter — 21 articulatory feature units (place of articulation, voicing, etc.) plus 5 units for stress and syllable boundary. The phoneme is read out by finding the closest output pattern to the activation vector (winner-take-all over phoneme templates).

**Representation (the key design choice):**

- **Local / one-hot encoding** for the letter inputs — exactly one unit active per letter slot.
- **Distributed / feature-based encoding** for the phoneme output — each phoneme is a vector of articulatory features, so phonemically similar phonemes share many output units.
- The 7-letter context window is the model's way of supplying enough surrounding text for the network to disambiguate context-dependent pronunciations (e.g. "the" → /ðə/ requires seeing the "h" and the "e").

**Training:**

- **Standard back-propagation** with a sigmoid activation. Weights initialised small and random.
- Training corpus: aligned text and phonetic transcription of a continuous source (a 1024-word phonetic transcription of a child's speech, and separately a 20,012-word dictionary).
- For each training example the window slides one letter at a time; the target is the phoneme of the central letter. Gradient descent updates all weights after every pattern.
- Several passes over the corpus.

**Experimental findings (briefly):**

1. **Rapid initial learning:** during the first few epochs the network distinguished vowels from consonants and learned the rule "blank → silence", producing distinguishable but inaccurate babbling.
2. **Continued learning of regularities:** with more training, common pronunciation rules emerged (e.g. final-e silent, "sh" → /ʃ/), and the network reached around **95% phoneme accuracy on the training set** and approximately **78% on held-out text** for the dictionary task.
3. **Generalisation:** the network correctly pronounced many novel words it had never seen, demonstrating that it had learned *rules* of pronunciation (distributed across the hidden layer's weights), not just memorised the training set.
4. **Graceful degradation:** removing a fraction of the hidden units or perturbing weights only mildly degraded performance, supporting the connectionist claim of distributed representation.
5. **Hidden-unit analysis:** clustering the hidden-layer activations revealed that the network had spontaneously discovered linguistic categories such as vowels/consonants and articulation features without ever being told them explicitly.
6. **Comparison with rule-based systems (DECtalk):** NETtalk did not exceed the engineered DECtalk system in accuracy, but it was striking because the same network learned the task from data alone, with no hand-coded phonological rules.

**Significance:** NETtalk was one of the first demonstrations that an MLP trained by back-propagation could learn a complex, rule-laden cognitive task and discover internal representations that recapitulated structure recognised by linguists. It became an exemplar of the **connectionist** approach to cognitive modelling.

See [[neural-networks]].

#### Part (b) [5 marks]

The truth table is the **logical OR** function: $y = 0$ only when both inputs are $0$; $y = 1$ in all other cases.

**Design: single-layer perceptron (McCulloch–Pitts neuron) with two inputs and a threshold (Heaviside step) activation.**

```
             w1 = 1
   x1 ----------------\
                       \
                        ( Σ , threshold θ = 1 ) ----> y
                       /
   x2 ----------------/
             w2 = 1
```

Computation:
$$y = \text{step}(w_1 x_1 + w_2 x_2 - \theta) = \begin{cases} 1 & \text{if } x_1 + x_2 \ge 1 \\ 0 & \text{otherwise} \end{cases}$$

**Validation against Table 1:**

| $x_1$ | $x_2$ | $w_1 x_1 + w_2 x_2$ | $\ge \theta = 1$? | $y$ predicted | $y$ target |
|-------|-------|---------------------|-------------------|---------------|------------|
| 0 | 0 | 0 | No | 0 | 0 ✓ |
| 0 | 1 | 1 | Yes | 1 | 1 ✓ |
| 1 | 0 | 1 | Yes | 1 | 1 ✓ |
| 1 | 1 | 2 | Yes | 1 | 1 ✓ |

All four rows match — the perceptron correctly implements logical OR. (Any equal positive weights and any threshold $\theta \in (0, 1]$ also work.)

#### Part (c)(i) [6 marks]

Using sigmoid activation $f(a) = 1/(1+e^{-a})$ with derivative $f'(a) = f(a)(1 - f(a))$, where $o_j$ denotes the output of neuron $j$ and $t_j$ its target:

**Output layer node $j$:** error is the difference between the target and the actual output, multiplied by the local gradient of the activation.

$$\boxed{\delta_j = o_j (1 - o_j)\,(t_j - o_j)}$$

This is the **delta rule** for the output layer.

**Hidden layer node $j$:** error is back-propagated from all downstream nodes $k$ in the next layer, weighted by the connection strengths $w_{kj}$ and modulated by the local gradient.

$$\boxed{\delta_j = o_j (1 - o_j)\,\sum_{k} \delta_k\, w_{kj}}$$

The sum runs over every node $k$ in the layer immediately after $j$. (In Figure 3 the hidden layer feeds into the output layer, so $k$ ranges over the output nodes 8 and 9.)

This recursive definition is the essence of **back-propagation**: the error at any hidden layer is a weighted combination of the errors at the layer above. See [[neural-networks]].

#### Part (c)(ii) [4 marks]

Figure 3: inputs $x_1, x_2, x_3$ (nodes 1–3), hidden $a_4, a_5, a_6, a_7$ (nodes 4–7), outputs $y_8, y_9$ (nodes 8–9). Hidden-to-output weights $w_{8j}, w_{9j}$ for $j \in \{4, 5, 6, 7\}$.

**$\delta_8$ (output node 8)** — use the output rule:

$$\boxed{\delta_8 = y_8 (1 - y_8)\,(t_8 - y_8)}$$

where $t_8$ is the target value for output 8 and $y_8$ is the actual sigmoid output of node 8.

**$\delta_4$ (hidden node 4)** — back-propagate from the output layer (nodes 8 and 9):

$$\boxed{\delta_4 = a_4 (1 - a_4)\,\big(\delta_8\, w_{84} + \delta_9\, w_{94}\big)}$$

The two output nodes 8 and 9 are the only successors of hidden node 4 in this architecture, so the sum in the hidden-layer formula contains just two terms. $w_{84}$ is the weight from hidden 4 to output 8 and $w_{94}$ is the weight from hidden 4 to output 9; $\delta_9$ is computed analogously to $\delta_8$.
