# Event streams

*The fundamental unit of a stream pipeline: a continuous, unbounded sequence of small, immutable, timestamped records describing things that happened.*

## Definition

An **event** is a small, self-contained, **immutable** record that captures a single occurrence — a user clicking, a sensor reading, a transaction completing. Events carry a **timestamp** of when they happened. An **event stream** is an unbounded sequence of such events flowing from one or more producers.

## Why it matters

Events are the raw material of [[stream-processing|stream processing]]. Their immutability is what makes replay-based fault tolerance possible: if a consumer crashes, you can re-read the events from the broker and reprocess them without losing data.

## Mechanism

Each event typically contains:

- **What happened** — a type / action code (e.g. `page_view`, `purchase`).
- **When it happened** — the event timestamp (see [[event-time-vs-processing-time]]).
- **Context** — IDs, metadata, payload (user ID, item ID, amount, etc.).

Encoding is flexible — JSON, Avro, Protobuf, plain text.

Events are produced by a **producer** (also called publisher or sender) and consumed by **consumers** (subscribers, recipients). The [[message-broker|message broker]] sits between them, decoupling the two sides.

## Trade-offs

- **Immutability** — events cannot be modified after emission. Corrections are new events. This simplifies reasoning but requires consumers to handle superseding events.
- **Ordering** — within a single partition/topic events are ordered by arrival; across partitions there is no global order guarantee.
- **Retention** — brokers like Kafka retain events for a configurable window, enabling replay. Traditional queues delete after delivery.

## Examples in the syllabus

- Stream Processing 2025 s. 8: events are immutable, timestamped, encoded as JSON/binary, generated by producers, processed by consumers.
- Fraud detection: each card transaction is an event; the stream processor checks patterns across recent events.
