# COM3021 May 2023 Exam

## Question 1 (25 marks)

**(a)** *Describe the difference between a fault and a failure in the context of systems availability. (5 marks)*
**Solution:**
A **fault** occurs when one specific component of the system behaves unexpectedly or deviates from its spec (e.g., a single server crashing). A **failure** occurs when the system as a whole stops providing its required service to the user. A reliable system anticipates faults and prevents them from causing a failure.
→ [[fault-vs-failure]], [[reliability]]

**(b)** *Describe three aspects of hardware faults that make them potentially less damaging to the system availability in comparison with software faults. (10 marks)*
**Solution:**
1. **Independence:** Hardware faults are typically independent and random (e.g., one disk crashing doesn't cause another to crash). Software faults often cause systematic, correlated failures across many nodes simultaneously.
2. **Mitigation via Redundancy:** Because hardware faults are uncorrelated, simple redundancy (adding more hardware, like RAID or extra cluster nodes) is usually sufficient to maintain availability. Software faults bypass hardware redundancy because the same bug exists on all redundant nodes.
3. **Predictability/Burn-in:** Hardware failure rates often follow predictable patterns (bathtub curve) and can be anticipated and proactively swapped, whereas software bugs are triggered unpredictably by specific edge-case inputs or unforeseen state interactions.
→ [[fault-vs-failure]]

**(c)** *When it comes to making decisions related to the design of data-intensive systems, explain the trade-off between availability and consistency, and how choices made based on this trade-off can affect the scalability of said system. (10 marks)*
**Solution:**
This trade-off is governed by the **CAP theorem**, which states that in the presence of a network partition (P), a system must choose between Availability (AP) and Consistency (CP). 
- **Consistency (CP):** Ensures every read receives the most recent write, but requires cross-node coordination (consensus), which increases latency and reduces availability if nodes cannot communicate. This limits scalability because coordinating state across many nodes adds overhead.
- **Availability (AP):** Ensures the system responds to every request, even if nodes are disconnected, by sacrificing strong consistency for eventual consistency. This greatly enhances scalability, as nodes can accept writes and serve reads independently without waiting for locks or consensus, allowing the system to scale horizontally much more effectively.
→ [[cap-theorem]], [[scalability]]

## Question 2 (15 marks)

**(a)** *In the context of document models, explain data locality, mentioning one advantage and one disadvantage of said model in comparison to a traditional relational database. (5 marks)*
**Solution:**
**Data locality** in document models means that all data associated with an entity is stored together in a single continuous string (like a JSON document).
- **Advantage:** Fast retrieval. Fetching the document requires only a single database lookup or disk read, avoiding the expensive, multi-table joins required in relational databases.
- **Disadvantage:** Inefficient partial updates. If the document is large, modifying or retrieving just a single small field may require loading or rewriting the entire document, whereas a relational database can update a single row efficiently.
→ [[document-model]]

**(b)** *In the context of distributed data architectures, explain two benefits and two drawbacks of combining replication and partitioning. (10 marks)*
**Solution:**
- **Benefits:**
  1. **Scalability:** Partitioning distributes the data and request load across many machines, allowing the system to handle datasets and throughput beyond a single machine's limits.
  2. **High Availability / Fault Tolerance:** Replication ensures that if a node holding a partition fails, another node holding a copy of that partition can take over, preventing data loss and downtime.
- **Drawbacks:**
  1. **Increased Complexity:** Combining both requires sophisticated cluster management to track which nodes hold which partition replicas, route requests correctly, and handle rebalancing and leader election.
  2. **Cross-Partition Transactions:** Ensuring consistency across operations that span multiple partitions while they are also being replicated requires complex protocols like Two-Phase Commit (2PC), which introduces high latency and overhead.
→ [[partitioning]], [[replication]]

## Question 3 (15 marks)

**(a)** *Regarding the programming model MapReduce, describe the workflow of a MapReduce job execution. (5 marks)*
**Solution:**
1. **Input Reader:** Reads data from a distributed filesystem and splits it into chunks.
2. **Map:** Mappers process chunks independently in parallel, generating intermediate `(key, value)` pairs.
3. **Shuffle & Sort:** The framework groups all intermediate values associated with the same key and routes them to the appropriate reducer.
4. **Reduce:** Reducers process the grouped values for each key to produce aggregated output.
5. **Output Writer:** The final output is written back to the distributed filesystem.
→ [[mapreduce]]

**(b)** *Regarding the programming model MapReduce, describe two benefits and two limitations of the MapReduce model. (8 marks)*
**Solution:**
- **Benefits:**
  1. **Fault Tolerance:** Built-in resilience; if a mapper or reducer fails, the framework automatically restarts the task on another node.
  2. **Scalability:** The programmer only writes map and reduce functions; the framework transparently handles distributed execution and scaling across thousands of nodes.
- **Limitations:**
  1. **High Latency:** MapReduce writes intermediate data to disk between stages to ensure fault tolerance, making it slow and unsuited for real-time or interactive stream processing.
  2. **Rigid Model:** Complex algorithms (like iterative machine learning, e.g., SGD) are difficult to implement because they require chaining multiple MapReduce jobs together, leading to massive disk I/O overhead.
→ [[mapreduce]]

**(c)** *Which component of the MapReduce framework is being described in the paragraph below? "This function is called once for every input record, and its job is to extract the key and value from the input record. For each input, it may generate any number of key-value pairs." (2 marks)*
**Solution:**
**Mapper** (or Map function).
→ [[mapreduce]]

## Question 4 (30 marks)

**(a)** *Explain the main difference between synchronous and asynchronous replication in a leader and followers architecture. (5 marks)*
**Solution:**
In **synchronous** replication, the leader waits for the follower(s) to acknowledge they have successfully written the data before confirming the write to the user. In **asynchronous** replication, the leader confirms the write to the user immediately after writing to its own local log, without waiting for the followers.
→ [[synchronous-vs-asynchronous-replication]]

**(b)** *In a leader and followers replication architecture, explain how read and write operations are served and performed by the replica nodes. (10 marks)*
**Solution:**
- **Writes:** All write operations must be sent exclusively to the **Leader** node. The leader records the write locally and then propagates the data change (via a replication log) to all **Follower** nodes.
- **Reads:** Read operations can be served by the Leader, but they are typically distributed across the **Followers** to scale read throughput. If a follower is lagging, reads may temporarily return slightly stale data (eventual consistency).
→ [[leader-follower-replication]]

**(c)** *Explain what would be a fully synchronous replication set and what would be a key disadvantage of this configuration. (15 marks)*
**Solution:**
A **fully synchronous replication set** means that every single follower in the cluster must acknowledge a write before the leader considers the write successful and responds to the client. 
- **Key Disadvantage:** **Severe Availability and Latency Issues.** If even a single follower node crashes, becomes slow, or suffers a network partition, the leader cannot complete any write operations. The entire system's write availability is bottlenecked by the slowest node, and the latency for every write is determined by the round-trip time to the most distant or slowest follower. 
→ [[synchronous-vs-asynchronous-replication]], [[leader-follower-replication]]

## Question 5 (15 marks)

```text
V = [2, 4, 1, 1, 2, 1, 1, 1, 3, 3, 1, 1]
ROW_INDEX = [0, 3, 5, 7, 9, 11, 12]
COL_INDEX = [0, 1, 4, 1, 5, 0, 2, 3, 4, 4, 5, 5]
```

**Solution:**
To reconstruct the matrix, we use `ROW_INDEX` to determine the start and end of values in `V` and `COL_INDEX` for each row. The number of rows is `len(ROW_INDEX) - 1 = 6`. The maximum column index is 5, meaning there are 6 columns.

- **Row 0:** elements from index 0 to 2 in `V` and `COL_INDEX`.
  Cols: 0, 1, 4 -> Values: 2, 4, 1
- **Row 1:** elements from index 3 to 4.
  Cols: 1, 5 -> Values: 1, 2
- **Row 2:** elements from index 5 to 6.
  Cols: 0, 2 -> Values: 1, 1
- **Row 3:** elements from index 7 to 8.
  Cols: 3, 4 -> Values: 1, 3
- **Row 4:** elements from index 9 to 10.
  Cols: 4, 5 -> Values: 3, 1
- **Row 5:** element at index 11.
  Col: 5 -> Value: 1

The reconstructed 6x6 dense matrix is:

```text
[
  [2, 4, 0, 0, 1, 0],
  [0, 1, 0, 0, 0, 2],
  [1, 0, 1, 0, 0, 0],
  [0, 0, 0, 1, 3, 0],
  [0, 0, 0, 0, 3, 1],
```
