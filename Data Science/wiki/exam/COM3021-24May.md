**(d)** *Reconstruct the matrix using the compressed sparse row (CSR) representation below and calculate its sparsity. (7 marks)*
```text
V = [1, 2, 3, 1, 1, 1, 3, 2, 2, 1]
ROW_INDEX = [0, 2, 3, 5, 7, 10]
COL_INDEX = [0, 2, 1, 2, 4, 0, 2, 1, 3, 4]
```
**Solution:**
The matrix has `len(ROW_INDEX) - 1 = 5` rows. The maximum column index is 4, so it has 5 columns. 
- **Row 0:** indices 0,1 in V -> Cols 0,2 -> Values 1, 2
- **Row 1:** index 2 in V -> Col 1 -> Value 3
- **Row 2:** indices 3,4 in V -> Cols 2,4 -> Values 1, 1
- **Row 3:** indices 5,6 in V -> Cols 0,2 -> Values 1, 3
- **Row 4:** indices 7,8,9 in V -> Cols 1,3,4 -> Values 2, 2, 1

Dense matrix:
```text
[
  [1, 0, 2, 0, 0],
  [0, 3, 0, 0, 0],
  [0, 0, 1, 0, 1],
  [1, 0, 3, 0, 0],
  [0, 2, 0, 2, 1]
]
```
**Sparsity:** Total elements = 5 * 5 = 25. Non-zero elements = 10. Zero elements = 15.
Sparsity = 15 / 25 = **0.6 (or 60%)**.
→ [[sparse-tensors]]

## Question 2 (45 marks)

**(a)** *Regarding the training process of the PikPokIT models behind its recommendation system, would you use full batch, mini-batch, online learning or a combination of them? Describe and justify your choice. (20 marks)*
**Solution:**
A **combination of batch (or mini-batch) and online learning** is required.
- **Online learning:** Since social media trends and user interests change by the minute, online learning should be used to update model parameters continuously as new user interactions arrive. This allows the system to adapt instantly to short-term trends.
- **Batch / Mini-batch learning:** Online learning alone can suffer from catastrophic forgetting and might struggle to learn deep, complex historical patterns. Periodically (e.g., nightly), a batch or mini-batch training job should run over the massive historical dataset of all user behaviors to provide a stable, highly accurate baseline model, which the online learner then fine-tunes during the day.
→ [[online-learning]], [[batch-vs-online-learning]]

**(b)** *What data processing approach will you use to suggest the next videos to be shown while a user is scrolling through their Home page? Justify your choice highlighting the advantages of your strategy for PikPok in comparison with other competing approaches. (10 marks)*
**Solution:**
**Stream Processing** should be used. 
As the user scrolls, their real-time actions (likes, view times) are ingested as an unbounded stream of events. A stream processing engine can consume these events, update the user's immediate context with ultra-low latency (milliseconds), and immediately suggest the next video. Compared to **batch processing**, which takes minutes to hours and would serve hopelessly stale recommendations, stream processing ensures maximum engagement by reacting instantly to the user's current session behavior.
→ [[stream-processing]]

**(c)** *During a meeting with your team, one of the data scientists in the team informed you of a problem in PikPokIT and suggested the usage of ADWIN as a solution. Q: What problem was being discussed? How was the problem manifesting itself? How can ADWIN help with that? (15 marks)*
**Solution:**
- **The Problem:** The team was discussing **Concept Drift**. In social media, the relationship between inputs (hashtags, user features) and the target variable (engagement/likes) changes rapidly as trends evolve.
- **Manifestation:** The problem manifests as a degradation in model accuracy over time, leading to a noticeable drop in user engagement metrics because the recommendations become stale or irrelevant.
- **How ADWIN helps:** ADWIN (ADaptive WINdowing) is a drift detection algorithm. It dynamically maintains a window of recent data. When the data is stable, the window grows, improving model confidence. If ADWIN detects a statistically significant change in the distribution of error rates (concept drift), it automatically shrinks the window by dropping the old, irrelevant data. This tells the system to discard outdated patterns and adapt to the new trend quickly.
→ [[concept-drift]], [[adwin]]

## Question 3 (15 marks)

**(a)** *In the design of distributed data systems like cloud-based storage solutions, explain (1) the data storage strategies employed to ensure data availability across multiple nodes or data centres, and (2) how these strategies impact the overall system’s scalability. (15 marks)*
**Solution:**
1. **Strategies for Availability:** Systems use **Replication**, where identical copies of the data are stored across multiple nodes or geographically separated data centers (e.g., leader-follower architecture). If one node or data center goes down, a replica can immediately take over, ensuring continuous data availability.
2. **Impact on Scalability:** To scale, systems use **Partitioning (Sharding)**, splitting the dataset into smaller chunks spread across different nodes to distribute the read/write load. While replication limits write scalability (since every write must be copied to all replicas), it significantly enhances **read scalability**, as read requests can be distributed among all available replicas. The combination of both allows the system to scale horizontally to massive sizes while remaining highly available.
→ [[replication]], [[partitioning]], [[scalability]]

## Question 4 (30 marks)

**(a)** *In the context of distributed data systems, explain in details the four A.C.I.D. properties. (15 marks)*
**Solution:**
- **Atomicity:** Transactions are "all-or-nothing". If a transaction consists of multiple steps, either all steps successfully complete, or the entire transaction is rolled back, leaving no partial updates.
- **Consistency:** Transactions must transition the database from one valid state to another, strictly adhering to all defined rules, constraints, and invariants (e.g., account balances cannot be negative).
- **Isolation:** Concurrent transactions execute without interfering with each other. The system ensures that the intermediate states of a transaction are invisible to other transactions until a commit occurs.
- **Durability:** Once a transaction is committed, its changes are permanent and will survive subsequent system crashes or power failures (usually achieved via writing to a Write-Ahead Log).
→ [[acid-properties]]

**(b)** *Explain the linearizability requirement, describing its trade-offs and how they are related to the trade-off underpinning the CAP theorem. (12 marks)*
**Solution:**
- **Linearizability:** This is the strongest consistency guarantee. It requires that a system behaves as if there is only a single copy of the data, and every operation takes effect atomically at a specific point in time. Once a write completes, all subsequent reads must return that new value (strong recency).
- **Trade-offs & CAP Theorem:** Providing linearizability requires intense coordination (consensus) between distributed nodes. According to the **CAP theorem**, if a network partition occurs (P), a linearizable system must refuse to serve requests (sacrificing Availability) to maintain this strong Consistency (making it a **CP** system). Therefore, the trade-off for linearizability is high latency and a severe loss of availability during network faults.
→ [[linearizability]], [[cap-theorem]]

**(c)** *In the context of a global social media platform with millions of users, explain and justify how would you approach the CAP trade-off. (3 marks)*
**Solution:**
I would choose an **AP (Available and Partition-tolerant)** approach, opting for **eventual consistency**. For a social media platform, high availability and low latency are critical for user experience; users can tolerate seeing slightly stale data (e.g., a delayed "like" count), but they will not tolerate the system being completely offline or sluggish during a partition.
→ [[cap-theorem]], [[eventual-consistency]]
