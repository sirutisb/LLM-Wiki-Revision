Parallel architectures
• Single instruction stream, single data stream (SISD) 
uniprocessor. 
• Single instruction stream, multiple data streams (SIMD)—
• The same instruction is executed by multiple processors using different data streams. Each processor 
has its own data memory (hence multiple data),
• Multiple instruction streams, multiple data streams (MIMD)
Each processor fetches its own instructions and operates on its own data. The processors are often off-
the-shelf microprocessors.

OpenMP
• OpenMP is one of the most widely used Application Program Interface (API) for shared 
memory programming using threads
•  Parallelism is implicit ie. a lot is abstracted from the programmer 
You specify which parts of the program you want to parallelise and the compiler produces a parallel 
executable

Distributed memory parallelism
• The data is distributed amongst the 
processes.
• Create smaller sub-problems - parallelism 
• Processes can only see their own data. 
• Communication between them is done via 
messages. 
A process can send data to another process

--- page 13 ---
Process communication
• Explicit messages between processes. 
• Messages are two-sided
they require a send and a receive. 
• To send a message must specify the data 
to send and the destination process. 
• The receive counterpart must match the 
sender process number.

--- page 14 ---
Message Passing Interface (MPI)
• Standard specification of a library that contains functions which the parallel copies of a 
distributed memory program can use to exchange messages with each other;
•  It addresses the parallel programming model based on message passing
•
 data from the address space of one process is moved into the address space of another process via 
a cooperative operations inside the processes;
•
Topology, synchronisation and communication functionality between a set of processes across 
nodes
• Language-independent with bindings for many different languages

--- page 15 ---
Message Passing Interface (MPI)
• Communicator
•
Objects connecting groups of processes in an MPI session
•
Give each process a unique identifier and arrange them in a topology
•
Consists of multiple MPI processes
•
A process can be associated with more than one communicator
•
The main/global communicator is called World (MPI_COMM_WORLD)

Message Passing Interface (MPI)
• Point-to-point communications; 
•
One process (the sender) posts a message 
containing a piece of data targeted at a 
certain other process; 
•
The other process (the receiver) expects the 
message; 
•
The operation is cooperative in the sense 
that both the sender and the receiver must 
play their part in the message exchange
