Batch learning
• Neural networks are trained over a series of epochs.
Each epoch will involve one forward pass and one backpropagation pass over the training data
In each epoch we can either
•
Compute the true gradient using all the available data
•
Full batch learning
•
Compute an approximation of the gradient
•
Mini-batch learning

--- page 7 ---
Mini-batch
• Alternatively, we can use more recent data 
for training (mini-batch)
It will be less computationally intensive
The older data won't affect the model as much
•
Incremental learning
• How often should a model be retrained?  
Daily, weekly, monthly?
• How much data should we use?
Week, month, year?
• Cons
More complexity added  in 
•
choice of the batch size
Convergence will be less stable depending on 
the data fluctuations in the batch
• Smaller the batch size, the greater the 
randomness

Online learning
• There are three types of data shifts that will cause model decay
• Covariate shifts
Changes in the input variables from the training data to the test data while the target variables are 
unchanged
•
Face recognition not being able to identify people wearing masks
• Prior probability shifts
Changes in the target variables while input variables are unchanged
•
A model that could estimate the probability of someone having flu based on their symptoms during the 
COVID pandemic
• Concept drifts
The relationship between input and output change over time in unforeseen ways
