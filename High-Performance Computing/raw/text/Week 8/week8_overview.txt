Manager-worker and task-based parallelism
4
• So far we have seen two ways to distribute work in parallel:
- Using OpenMP to parallelise loops
- Using domain decomposition with MPI 
• These are data parallel approaches: the processes/threads carry out the same 
operations on different data
• These are tried and tested methods for parallelising work in HPC applications but 
sometimes another approach is required

--- page 5 ---
Manager-worker and task-based parallelism
5
• Manager-worker and task-based parallelism are alternative ways of distributing work 
which allow for more flexibility
• Manager-worker: one process is a manager which hands out work to worker 
processes. A worker processes carries out a unit of work and request more work when 
the current work is completed.
• In task-based parallelism the workload is divided into tasks which are carried out by 
different processes/threads. This could involve different tasks being carried out on the 
same data.
