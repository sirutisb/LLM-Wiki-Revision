Introduction 
41
• Previously we have seen how OpenMP can parallelise loops and sections
• There are other cases where we need to express parallelism which is not described by 
loops with a predetermined trip count or number of sections:
- Recursive algorithms
- Traversing a linked list
• OpenMP 3.0 introduced tasks to handle these cases

--- page 3 ---
OpenMP tasks 
42
• One thread generates tasks and other threads execute the tasks
• After the generating thread has finished generating the tasks it may carry out some of 
the tasks
• A single construct is used to ensure that only one thread generates tasks
