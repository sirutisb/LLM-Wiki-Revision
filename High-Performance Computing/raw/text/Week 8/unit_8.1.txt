--- page 3 ---
• In a manager/worker algorithm we have a manager process which receives requests 
from worker processes
• We don’t know which worker will request work next so we need a way to receive a 
message from an unspecified source
• We can use the wildcards MPI_ANY_SOURCE to receive from an unspecified source
• Similarly MPI_ANY_TAG receives from an unspecified tag
• The rank and tag received from are returned in the status argument

--- page 7 ---
• We only need to collect results from the worker processes and not the manager 
process
• Instead of using the MPI_COMM_WORLD communicator we can define a new 
communicator which only includes the worker processes
• Define a group of processes which is a sub-set of MPI_COMM_WORLD and associate a 
new communicator with the group
• The new communicator can then be used instead of MPI_COMM_WORLD in MPI 
function calls

Typical process for creating a new communicator
15
1.Get a handle for the group associated with MPI_COMM_WORLD (MPI_Comm_group)  
2.Make a new group from the world group (e.g. MPI_Group_incl)  
3.Make a new communicator from the new group  
(MPI_Comm_create)  
4.Find rank in new communicator (MPI_Comm_rank)  
5.Do work and carry out communication  
6.Free the communicator and group (MPI_Comm_free and MPI_Group_free)
