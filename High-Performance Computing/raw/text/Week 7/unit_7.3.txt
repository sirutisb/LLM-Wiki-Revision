Performance degradation
31
• A number of factors can contribute to real-world parallel performance being less than 
the idealised peak performance: 
- Starvation: insufficient parallel work to keep the processors busy (insufficient parallel work, 
uneven distribution of work)
- Latency: time taken for information to travel from one part of the system to another (e.g. 
accessing memory, sending messages) 
- Overhead: work required in addition to the computation (e.g. starting and stopping OpenMP 
parallel regions)
- Waiting: multiple threads/processes accessing a shared resource (e.g. contention for memory 
or network bandwidth)
SAB: section 1.4.5

Interconnect
32
• An MPI program has to send messages to communicate and co-ordinate between 
processes
• Compute nodes are linked by an interconnect(network) which carries these messages
• Time spent communicating needs to be minimised to achieve high parallel efficiency 
and scale to large numbers of processors
• We want to understand the impact on parallel performance

Network topologies
35
• Number of “hops” between nodes affects latency
• Connecting every node to every other node (fully connected) would be ideal but is only 
practical for a small network
• A bus or ring topology is simple but does not provide enough connectivity for HPC
• In practice more sophisticated topologies are used which balance cost and 
performance

MPI process placement
39
• Communication overhead between MPI processes depends on where the processes 
are running
- MPI processes within the same compute node can communicate faster than process on 
different compute nodes
- The speed of MPI communication between processes on different compute nodes depends on 
the network connectivity between the nodes
• The placement of MPI processes can be optimised to reduce communication time 
(this paper for an example of domain decomposition rank re-ordering)

Latency and bandwidth
43
• A high performance interconnect has:
- Low latency: short delay sending small messages
- High bandwidth: high data transfer rate for large messages 
• Which one is more important depends on characteristics of the algorithm e.g. many 
small point-to-point communications vs. large collective communications
• May be able to aggregate smaller messages to reduce latency

Unit Summary
44
• MPI programs need to send messages and we want to minimise communication time
• Infiniband and ethernet are networking technologies used in commodity clusters
• In practice a number of factors affect the transmission time for a message (number of 
hops, blocking factor, other traffic)
• We can model the best case transmission time as a fixed latency plus a constant 
bandwidth
• Latency matters more for small messages; bandwidth matters more for larger 
messages
