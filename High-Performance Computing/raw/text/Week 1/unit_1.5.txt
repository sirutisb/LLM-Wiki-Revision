• HPC applications typically use compiled languages to maximise performance  
• Compilers parse the source code and generate an executable file 
• Compilers can make optimisations at compile time which would be too expensive for 
an interpreter to make at run time
• C, C++ and Fortran are the most common languages used in HPC

--- page 5 ---
• Parallelisation is typically too complex for the compiler to handle automatically and 
needs to be added by the programmer
• Parallel programming is often based on extensions to existing serial languages 
• OpenMP and MPI two widely used parallel programming methods in HPC which extend 
C, C++ and Fortran: 
- OpenMP: directives based parallelism for shared memory 
- MPI: Message Passing Interface for shared or distributed memory using function calls 
• Specific parallel programming languages exist (e.g. Chapel and Julia) but are not as 
widely used at present
