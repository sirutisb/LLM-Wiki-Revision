• Modern CPU architectures have memory controllers integrated into the CPU
• Two socket compute nodes are commonly used in HPC clusters (also multi-
socket servers)
• Accessing memory associated with the other controller is slower than 
accessing memory on the local controller
• Two socket nodes have NUMA characteristics

32

--- page 5 ---
• Physical memory is allocated the first time it is accessed (touched) rather 
than when it is allocated
• Typically physical memory is allocated on the first memory controller to 
touch a given memory page
• This is called a first touch allocation policy
• Accessing memory held on the other controller will take longer than 
accessing memory on the local controller 
First touch memory allocation   
33

--- page 6 ---
• NUMA is relevant to OpenMP where threads access shared memory. For example: 
- If we have a loop which initialises an array and the loop is not parallelised then thread 0 will 
touch all the memory first
- If this array is used in a second parallelised loop then threads running on the other socket will 
take longer to access this memory 
• Other effects may also be relevant (e.g. sharing cache, contention for memory 
bandwidth)
OpenMP and NUMA  
34

--- page 7 ---
• Understanding OpenMP performance in a NUMA environment is not 
straightforward
• One way to avoid this consideration is to combine MPI and OpenMP (hybrid 
parallelism)
• MPI processes have separate address spaces; OpenMP threads share an 
address space
• If we have one MPI process per socket then each MPI process starts 
OpenMP threads within a single socket so that OpenMP is not subject to 
NUMA effects
