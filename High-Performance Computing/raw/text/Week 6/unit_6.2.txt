• To understand performance we need to consider the characteristics of the algorithms 
we are using
• Different algorithms have a different ratio between the number of floating point 
operations required and the number of memory accesses required
• We can express this as an “arithmetic intensity” (also called “algorithmic intensity” or 
“operational intensity”)

• Different algorithms have different arithmetic intensities (floating point operations per 
byte of data)
• Lower arithmetic intensity means that the algorithms is limited by memory bandwidth 
(memory bound)
• Higher arithmetic intensity algorithms can make more efficient use of floating point 
hardware and are limited by the floating point performance (compute bound) 
• We can use the roof-line model to understand how the algorithm performs on a 
particular hardware platform
