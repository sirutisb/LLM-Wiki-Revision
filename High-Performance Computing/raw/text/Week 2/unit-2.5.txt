Parallelising loops
• In order to do useful work in parallel we need a way to 
distribute the workload between threads 
• One way to do this in OpenMP is to parallelise loops 
• Different threads carry out different iterations of the loop 
• We can parallelise a for loop from inside a parallel region: 
#pragma omp for 
• We can start a parallel region and parallelise a for loop: 
#pragma omp parallel for

--- page 5 ---
Order of loop iterations
• When the loop is parallelised the iterations will not take 
place in the order speciﬁed by the loop iterator 
• We can’t rely on the loop iterations taking place in any 
particular order 
• The results are stored in arrays which do hold the 
results in the order we require 
• If we want to print out the results of our calculation in 
order we will need a second sequential (not 
parallelised) loop

Unit summary
• We can parallelise a for loop in a parallel region with the 
directive: #pragma omp for 
• We can start a parallel region and parallelise a for loop 
with the directive: #pragma omp parallel for 
• In a parallel loop different threads carry out different loop 
iterations 
• Parallel loop iterations are not carried out in the order 
speciﬁed by the loop iterator 
• A sequential loop is required to write out results in order
