• We need to give the compiler a flag to tell it to use OpenMP directives to parallelise the 
executable 
• Compiler flags are not specified in the standard and differ between compilers e.g.
- GNU C compiler: gcc -fopenmp 
- Intel C compiler: icc -qopenmp

--- page 14 ---
• We can use the OMP_NUM_THREADS environment variable to control the number of 
threads forked in a parallel region e.g.
 
- export OMP_NUM_THREADS=4
• OMP_NUM_THREADS is one of the environment variables defined in the standard
• If you don’t specify the number of threads the default value is implementation defined 
(i.e. the standard doesn’t say what it has to be)
