NVIDIA Pascal architecture

--- page 2 ---
• Will use NVIDIA Pascal as an example of a GPU architecture designed for use in HPC
• Up to 60 “Streaming Multiprocessors” (SM) per GPU
- Up to 4 SMs may be disabled due to manufacturing defects
• Grouped in blocks of 10 (graphics processing cluster)
- Each graphics processing cluster includes all elements of the rendering pipeline - effectively an 
independent GPU

--- page 6 ---
• The “GigaThread Engine” schedules threads and handles context switching between 
threads (much cheaper context switching than a CPU)
• The thread engine schedules blocks of up to 1024 threads (coarsest division of 
workload) which execute on the same SM
- Threads within a block have access to shared memory, cache and can synchronise
• Thread blocks are subdivided into “warps” of up to 32 threads which execute on an 
individual Streaming Multiprocessor
- All threads in a warp execute the same instructions
